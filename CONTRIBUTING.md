# Engaging with Oblivion Theory

[日本語は下にあります。](#日本語)

This repository is the **public reading edition** of Oblivion Theory. It is a
curated archive of concept drafts, shared for reading, citation, and criticism.
It is not the internal research source, and it does not accept content pull
requests against the drafts themselves.

## What this repository invites

- **Criticism and counterexamples.** The drafts are unfinished. Pointed
  objections, broken steps, and missing assumptions are the most useful
  contributions.
- **Questions about claims.** If a claim label, definition, or derivation is
  unclear, ask. Confusion in the public edition is a defect in the public
  edition.
- **Pointers to prior or adjacent work.** References that confirm, refute,
  precede, or extend a draft are welcome.

## How to engage

- Open a **GitHub Issue** using one of the issue templates. Use the
  critique/question template for objections and the reference template for
  related work.
- Reference the specific paper and section (for example,
  `01_本編シリーズ｜Series/01_論文I_力としての忘却.md` §5.1) so the point is
  locatable.

## What is out of scope here

- **Content pull requests to the drafts.** Revision happens in a separate
  internal source. PRs that rewrite paper bodies will be closed with thanks;
  raise the underlying point as an Issue instead.
- **Internal ledgers and working metadata.** `*.meta.md` files and backstage
  process notes are intentionally excluded; the Public Edition Guard rejects
  them.

## How to cite

Use [`CITATION.cff`](CITATION.cff). Where a stable identifier is required, cite
the Zenodo concept DOI for the series timestamp declaration
(`10.5281/zenodo.19976038`). Treat every text here as a **concept draft** unless
a status label says otherwise.

## 日本語

このリポジトリは Oblivion Theory（忘却論）の**公開読者版**である。コンセプト草稿
を厳選して収めたアーカイブで、読解・引用・批判のために共有している。内部の研究正本
ではなく、草稿本体への content pull request は受け付けない。

### 歓迎するもの

- **批判と反例。** 草稿は未完成である。鋭い反論、破れている導出、欠けている前提の
  指摘が最も有用な貢献である。
- **主張への質問。** 主張ラベル・定義・導出が不明瞭なら問うてほしい。公開版での
  混乱は公開版の欠陥である。
- **先行・隣接研究の指摘。** 草稿を支持・反証・先行・拡張する参考文献を歓迎する。

### 関わり方

- **GitHub Issue** を、用意したテンプレートから開いてほしい。反論・質問は
  critique/question テンプレート、関連研究は reference テンプレートを使う。
- 対象の論文と節を明示してほしい（例:
  `01_本編シリーズ｜Series/01_論文I_力としての忘却.md` §5.1）。

### ここでの対象外

- **草稿本体への content pull request。** 改稿は別の内部正本で行う。本文を書き換える
  PR は感謝とともにクローズする。論点は Issue として挙げてほしい。
- **内部台帳・作業メタデータ。** `*.meta.md` や backstage の作業ノートは意図的に
  除外しており、Public Edition Guard が拒否する。

### 引用方法

[`CITATION.cff`](CITATION.cff) を使う。安定識別子が必要な場合は、シリーズの
タイムスタンプ宣言の Zenodo Concept DOI（`10.5281/zenodo.19976038`）を引用する。
ステータスラベルが別途示さない限り、ここの全テキストは**コンセプト草稿**として扱う。
