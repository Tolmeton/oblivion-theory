## 結論

Iteration 1 の `lambda_q^evt := |P_q|` は Tolmetes review R1 の通り、single canonical BF cell では `P_q != 0` の indicator に同型で、`Int_q in {-1,0,+1}` へ collapse する。Iteration 2 ではこれを採用候補から comparator へ降格した。

採用候補は **方針 F: Walsh primitive lattice normalization**:

`lambda_q^evt := 1`, `Int_q([F_q])(cell) := I_q(F_q)(cell)`.

根拠は、automath の `walshFlux` が `deltaSet` の boundary sum として整数値 cochain 上に定義されるため、primitive period lattice の生成元を `1` と読むのが自然なことにある [SOURCE: /tmp/automath_p10a/lean4/Omega/Core/WalshStokes.lean L27-L35]。これにより BF excess の整数係数 `E_q = |det(I-A_q)| - F_{2q-2}` を割り潰さず、`H^2(G_m; Z)` の中間 class を `0, 11, 55, 278, ...` として保持できる。

観測値 `k_4=0/k_5=1` は副次条件として照合した。方針 F の BF cell では `q=4: 0 mod 2`, `q=5: 11 mod 2 = 1` で一致する。一方 q=7 proxy は `278 mod 2 = 0` となり、Iteration 1 の normalize-to-one とは異なる。これは不一致ではなく、R1 を避けた結果として raw integer class の偶奇が見えている。
