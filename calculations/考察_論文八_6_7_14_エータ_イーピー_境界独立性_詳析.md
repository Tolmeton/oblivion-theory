## P-2 Decomposition

`η / E_p / boundary closure` の独立性を判断するには、共通上流である `Route B_min` を含めた 4 要素へ直交分割する必要がある。

| 要素 | 独立変数 | 役割 |
| :--- | :--- | :--- |
| `accumulation_core` | `α_E` | Euler path 上の累積忘却度そのものを与える |
| `scalar_open_bridge` | `η` | 開区間で `α_{\mathrm{III}}^E` と `α_E` を整合させる |
| `endpoint_closure` | `\bar{\eta}` | `\pm\infty` を `0,1` へ閉じる |
| `categorical_lift` | `E_p` | 経路順序を忘却濾過の縮退列へ持ち上げる |

MECE 判定:

- `accumulation_core` は path 上の量そのもの
- `scalar_open_bridge` は開区間での座標橋
- `endpoint_closure` は端点処理
- `categorical_lift` は圏論的意味付け

したがって、「3 つの独立性」を問うには、まず `Route B_min` を共通根として立てた 4 要素系で見る必要がある。

## P-3 Deep Dive

### 要素 1: `accumulation_core`

**SOURCE**: Paper VIII §6.7.14a-b は、`Route B_min` により `α_E` を Euler path 上の累積忘却度として定め、その導出から `α_E(\theta)=\sin^2(\theta/2)` を与える。  
**INFERENCE**: `η / boundary closure / E_p` の 3 者は、いずれもこの `α_E` を入力として使う。したがって flat な三者比較の前に、共通上流として `accumulation_core` を置かなければならない。

### 要素 2: `scalar_open_bridge`

**SOURCE**: Paper VIII §6.7.14c は

```text
η(α_III^E(θ)) = α_E(θ)
```

を与え、開区間 `(0,\pi)` 上で `α_{\mathrm{III}}^E` と `α_E` の可換性を立てる。  
**INFERENCE**: `η` は `α_E` を前提として初めて定義できる。ゆえに `η` は `accumulation_core` から独立ではない。  
ただし `η` 自体からは端点閉包も `E_p` も出ない。ここで得られるのはあくまで**開区間での scalar bridge** だけである。

### 要素 3: `endpoint_closure`

**SOURCE**: Paper VIII 命題 6.7.4c は `\bar\eta(-\infty)=0`, `\bar\eta(+\infty)=1` を与え、§6.7.14c は `α_{\mathrm{III}}^E(\theta)` が端点で `\pm\infty` に発散すると述べる。  
**INFERENCE**: したがって端点は `η` 単独では閉じない。`scalar_open_bridge` が開区間で働き、`endpoint_closure` が端点だけを閉じる。  
つまり `boundary closure` は `η` の従属概念ではなく、**同じ scalar branch の別レイヤー**である。  
独立性の判定としては、

- `η` から `boundary closure` は出ない
- `boundary closure` から `η` も出ない
- しかし両者は同じ scalar package を構成する

と切るのが最も正確である。

### 要素 4: `categorical_lift`

**SOURCE**: Paper VIII §6.7.14d は

```text
E_p(\theta):=C_{\alpha_E(\theta)}
```

を定め、`α_E` の単調増加と `(F4)` から order-preserving functor になると述べる。  
**INFERENCE**: `E_p` は `η` を経由せずに `α_E` から立つ。端点閉包も `E_p` の関手性には不要である。  
したがって `E_p` は scalar package からは**genuinely separate** な categorical layer である。

### 浮上次元

1. `η / boundary closure / E_p` を flat な 3 項として扱うのは不正確で、共通上流に `accumulation_core` がある。  
2. `η` と `boundary closure` は独立な追加公理ではなく、**scalar package の開区間層と端点層**である。  
3. `E_p` だけが scalar package から独立な categorical layer であり、ここに Euler path の categorical content がある。  
4. したがって C4 の次の未解決点は、「3 つの独立性」を問うことではなく、**scalar package と categorical lift の二層分離をどこまで圧縮できるか** を問うことへ言い換わる。

## P-4 Integration

再構成すると、現在の C4 は次の階層で読むのが正確である。

```text
Route B_min
  ↓
accumulation_core (α_E)
  ├─ scalar_open_bridge (η)
  ├─ endpoint_closure (η-bar)
  └─ categorical_lift (E_p)
```

この図式から分かることは 3 つだ。

1. `η` は `α_E` から独立ではない。  
2. `boundary closure` は `η` に従属しているのではなく、scalar branch の端点層として別に要る。  
3. `E_p` は scalar branch と別層にあり、C4 の genuinely separate な構成要素はむしろこちらである。

したがって、本文へ戻すなら最小版は次の 1 文で足りる。

```text
η / boundary closure は α_E を III 系と VIII 系の間で scalar に閉じる package であり、Euler path の圏論的内容はそこから独立に E_p が担う。
```

**再説明**: `η` は地図の座標変換、`boundary closure` は地図の端の処理、`E_p` はその地図上の道そのものだ。だから本当に別物として残るのは `E_p` で、`η` と `boundary closure` は同じ地図系の内部にある。
