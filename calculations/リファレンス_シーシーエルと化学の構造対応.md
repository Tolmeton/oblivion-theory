## 0. kernel

この話題の芯は一文で足りる。

> 49d は「何があるか」を数える分子式側の表現であり、CCL の本体は「何が何とどう結ばれるか」を記録する構造式側にある。次元設計の本体は、在庫表をどれだけ磨くかではなく、結合様式と混成をどう表現するかにある。

ここから 4 つの帰結が出る。

1. `49d = 分子式` は強く固定してよい  
2. `原子 = 演算子` という読みは誤りで、構造式面では `原子 = 意味トークン`、`結合 = 演算子` へ修正される  
3. 各演算子は単一 n-level に閉じず、スペクトル分解で読む方がよい  
4. Phase C は 49d を置き換えるより、49d が見落とす構造式面を読むための層である

## 2. 粒度の分離

この話題で最も混線しやすいのは、「原子」という語が 2 つの倍率で使われていることだ。

| 面 | 原子 | 結合 | 用途 |
|:---|:---|:---|:---|
| 基底面 | `U_arrow`, `U_compose` などの U パターン | テンソル積や合成 | Aletheia や特徴量空間の抽象基底を議論するとき |
| 構造式面 | `fn`, `pred`, `CTRL`, `.method`, `#`, `¥` などの意味トークン | `>>`, `~`, `*`, `&`, 縦棒族などの演算子 | 実際の CCL テキストを「何が何とどう繋がるか」で読むとき |

この 2 面を分けると、次の 2 つが両立する。

- `9 元素 = 9 基底パターン` は **抽象基底の話**
- `原子 = 意味トークン、結合 = 演算子` は **構造式読解の話**

両者は矛盾でなく、ズーム倍率の違いである。

## 3. 4層対応

| 層 | 化学像 | CCL での実体 | 保存するもの | 落とすもの |
|:---|:---|:---|:---|:---|
| Layer 1 | 元素 | U パターン / 意味トークンの族 | 最小単位の型 | 接続関係 |
| Layer 2 | 分子式 | 49d, テンソル積, カウントベクトル | 何が何個あるか | どう結ばれているか |
| Layer 3 | 構造式 | CCL テキスト, 演算子列, トポロジー | 接続順序と結合型 | 局所構造の機能圧縮 |
| Layer 4 | 官能基 | 頻出かつ弁別力のある部分パイプライン | 局所構造と機能 | 大域トポロジー |

この表の意味は単純である。

- Layer 2 で止まると、同じ在庫を持つ異なる構造を見分けにくい
- Layer 3 に入ると、「同じ材料で違う組み方」が見える
- Layer 4 に入ると、「どの部分構造が振る舞いを決めているか」が見える

## 4. 49d はなぜ分子式なのか

### 4.1 49d 監査

```text
D (演繹的): 20
H (混成):   17
A (アドホック): 8
E (経験的): 4
```

含意:

- 49d は純理論の結晶ではない
- しかしアドホック部が効いていたのは、間違った方法で構造を暗黙エンコードしていたからである

### 4.2 浄化実験

| 条件 | d | R@1 | R@10 |
|:---|---:|---:|---:|
| 49d baseline | 49 | 6.0% | 23.0% |
| 41d | 41 | 3.0% | 16.4% |
| 22d 演繹核 | 22 | 0.4% | 7.0% |
| 253d 2次テンソル積 | 253 | 2.0% | 11.0% |
| 453d 3次テンソル積 | 453 | 2.8% | 11.8% |
| 453d Z-first 3次 | 453 | 2.4% | 17.2% |

結論:

- 演繹核だけでは壊滅する
- テンソル積で次元を増やしても baseline 未達
- つまり、分子式を複雑化しても構造式にはならない

### 4.3 49d = 分子式 の意味

```text
化学:
  原子 = H, O, C
  分子式 = H2O
  構造式 = H-O-H
  示性式 = -OH

CCL:
  原子 = U パターン / 意味トークン
  分子式 = 49d / テンソル積 / カウント
  構造式 = CCL テキストそのもの
  示性式 = 部分パイプライン / 官能基
```

## 5. 結合次数と n-level

### 5.1 結合次数の対応

| 化学 | CCL 側の読み |
|:---|:---|
| 単結合 `σ` | n=1, `>>` |
| 二重結合 `σ+π` | n=1 + n=2, `>> + ~` |
| 三重結合 `σ+2π` | n=1 + n=2 + n=3, `>> + ~ + %` |
| 芳香環 / 非局在化 | n=1 + n=∞, `>> + ||` 型 |

### 5.2 コヒーレンス定理との対応

| n-level | 根拠 | 強制される中心演算子 | 読み |
|:---|:---|:---|:---|
| n=1 | Mac Lane | `>>`, `_`, `<<`, `&`, 縦棒族の一部, `\\` | 1-cell 合成の唯一性 |
| n=1.5 | 自由モノイダル | `{}`, `*`, `L:[]{}` | 合成を 1 つとして束ねる |
| n=2 | Street | `~`, `~*`, `~!`, `>*`, `!`, `F:[]{}` | 2-cell / 随伴 / モナド |
| n=3 | Kelly | `%` | 豊穣圏のテンソル積 |
| n=∞ | 随伴の普遍性 | `||`, `|>`, `<|`, `lim[]{}` | 非局在化、大域拘束 |
| Basis | Smithe Lens | `¥`, `#` | Get / Put, Markov Blanket 境界 |

### 5.3 1.5 の意味

`n=1.5` は曖昧な中間値でなく、「複数の射を 1 つの単位として扱う」ための層である。  
`{}` がここに現れるのは、結合律だけでは「どこまでが 1 単位か」を構文上で示せないからである。

## 6. スペクトル分解モデル

### 6.1 全単射仮説の放棄

旧仮定:

> 各演算子は正確に 1 つの n-level に帰属する

新仮定:

> 各演算子 X は複数の n-level に射影を持ち、支配的射影だけが主導層を与える

```text
X = Σ_i α_i · π_i(X)
```

ここで `α_i` は射影重み、`π_i(X)` は n-level `i` への射影である。

### 6.3 n=2 は混成の震源地

n=2 の演算子が他レベルとの混成を最も多く持つのは偶然ではない。

- n=2 は「射の間の射」である
- 下位の n=1 と上位の n=∞ を同時に参照しやすい
- そのため、平面拘束や二重結合的な振る舞いがここに集中する

これは化学で sp² 混成が平面を作ることと響き合う。

## 7. U⊗ 分解と Rank

### 7.1 主要演算子の U⊗ 分解

```text
>>       = U_arrow                                   Rank 1  純粋 s
{}       = U_compose                                 Rank 1  純粋 s
*        = U_compose ⊗ U_precision ⊗ U_sensory      Rank 3  sp3
~        = U_depth ⊗ U_adjoint                       Rank 2  sp
%        = U_compose ⊗ U_precision                   Rank 2  sp
lim[]{}  = U_adjoint ⊗ U_arrow ⊗ U_sensory          Rank 3  sp3
¥ / #    = U_sensory                                 Rank 1  純粋 s
```

### 7.2 Rank 分布

| Rank | 演算子数 | 割合 | 読み |
|:---|---:|---:|:---|
| 1 | 17 | 55% | 単一レベルにほぼ閉じる |
| 2 | 14 | 42% | 二レベル混成 |
| 3 | 2 | 6% | 三レベル混成 |

### 7.3 Rank は別軸

支配的 n-level と Rank を混同してはいけない。

| 概念 | 何を表すか | 例 |
|:---|:---|:---|
| 支配的 n-level | どのコヒーレンス層が主導するか | `~` は n=2、`%` は n=3 |
| Rank | いくつの独立 U 軸をまたぐか | `>>` は Rank1、`~` は Rank2、`*` は Rank3 |

だから、`%` が **n=3 支配的** でありながら **Rank2** を持つのは矛盾ではない。

### 7.4 Rank = 空間次元

| 化学 | CCL | Rank | 空間像 |
|:---|:---|---:|:---|
| 純粋 s | `>>` | 1 | 点 |
| sp | `~` | 2 | 直線 / 二軸拘束 |
| sp² | 平面拘束型の二レベル混成 | 2 | 平面 |
| sp³ | `*` | 3 | 立体 / 四面体 |

これは直観的にはかなり強いが、まだ推定帯にいる。

## 8. 2026-04-06 の批判更新

### 8.1 原子と結合の定義修正

旧:

> 演算子 = 原子

新:

> 原子 = 意味トークン  
> 結合 = 演算子

理由:

- 化学の原子は、それ自体で最小単位として存在する
- `>>` はそれ自体が内容を持つ単位ではなく、単位どうしを結ぶ作用だから

### 8.2 官能基の発見方法

旧:

> 官能基テンプレートを事前定義する

新:

1. CCL テキストから長さ 3-7 の部分列を抽出する
2. 頻度しきい値をかける
3. Recall@k などの弁別力で検定する
4. 弁別力のある部分列を官能基候補とみなす

### 8.3 構造式 v1 / v2

| 実験 | 結果 | 読み |
|:---|:---|:---|
| tensor3 | best `R@1=2.8%` | 分子式の複雑化であって構造式ではない |
| 構造式 v1 | 全条件で 49d 劣化 | ミクロすぎる特徴は `>>` に支配される |
| 構造式 v2 | `R@10=29.4%`, `+4.8pp` | トポロジーは群同定に効く |

### 8.4 3層統合と U_cos

| 層 | d | 単独 R@10 | 化学像 |
|:---|---:|---:|:---|
| L1 圏論 | 18 | 18.6% | 分子軌道 |
| L2 示性式 K=10 | 10 | 7.4% | 官能基 |
| L3 分子式 | 18 | 17.8% | 元素分析 |
| L1+L2+L3 | 46 | 20.8% | 三層統合 |

含意:

- 3 層全て揃って初めて最大弁別力が出る
- それでも 49d の 81.2% に留まる
- したがって失敗の本体は特徴量設計だけでなく、cosine 距離の忘却にもある

## 9. Lethe / Phase C への含意

1. Stage 1 の粗検索では 49d をまだ捨てなくてよい  
   分子式側の高速索引としては依然 useful。

2. Stage 2 以降で構造式面を別に読む必要がある  
   CCL テキスト、トポロジー、官能基、learned distance が必要。

3. 官能基は手で決め打ちしない  
   まず発見し、その後で性質を読む。

4. Phase C は 49d の否定ではなく 49d の盲点回収  
   coarse retrieval と structure-aware re-rank は役割が違う。

5. 真のボトルネックは U_count だけでなく U_cos  
   特徴量より距離関数の側に ceiling がある可能性が高い。
