# 結合次数 → CCL 演算子の強制性証明

**計算日**: 2026-04-04
**前提**: 計算_化学的同型_忘却の元素論.md §4 (結合 ↔ 演算子), Aletheia §2.1 (U フィルトレーション), Mac Lane coherence, Kelly enriched coherence
**目標**: 化学結合次数 k = Aletheia n-cell レベル n とするとき、CCL 演算子の**支配的射影**が n から強制される (coherence theorem) ことを示す。ただし各演算子は複数の n-level に射影を持つ (**スペクトル分解 / 混成軌道モデル**, §8)。

**地位**: 証明スケッチ → スペクトル分解モデルへの発展。形式的証明ではなく構造の同定。

## 0. 主張の精密化

**直感的主張** (ROM §4.1): σ結合=`>>`、π結合=`~`、テンソル結合=`%`

**精密化すべき問い**:
> 「CCL 演算子 X が n-cell レベル n から強制される」とはどういう意味か？

**定義**: 演算子 X が **n-cell レベルから強制される** とは:
1. X を持たない言語では、n-cell 構造を**表現できない** (表現力不足)
2. X より「弱い」演算子では、n-cell の**普遍性質** (universal property) を満たせない
3. X より「強い」演算子は、n より高いレベルの構造を前提とする (過剰)

つまり「X が n レベルで強制される」= **X は n-cell 構造の最小の構文的実現**。

## 1. n=1: `>>` の強制性

### 1.1 n=1 での新構造

圏の公理 (n=1):
- 対象の集まり
- 各 f: A → B に対して morfism (1-cell)
- 恒等射 id_A: A → A
- **合成**: f: A → B, g: B → C から g∘f: A → C
- 合成の結合律・単位律

**新しいのは合成のみ**。対象は「要素」に過ぎず (集合にある)、恒等射は合成の単位元。

### 1.2 合成の普遍性

圏論の中心的事実: **自由圏** (Free Category) は対象と射の有向グラフから生成される圏で、それ以外の関係は一切ない。自由圏における唯一の非自明な操作が**合成**。

Mac Lane のコヒーレンス定理 (モノイダル圏):
> 「全ての自然に書ける可換図式は可換である」

これを合成に適用すると: 合成の表現は**一意**である (結合律・単位律を満たす二項演算として)。

### 1.3 `>>` の強制

```
任意の言語 L が n=1 構造を表現しようとするとき:
  L は "A を B に変換する" を表現する演算子 X_L を持たなければならない。
  X_L が合成 (X_L(f, g) = g∘f または f∘g) を満たす唯一の演算子。
  CCL では X_L = `>>`。
```

**他の候補との比較**:
- `∘` (数学的合成): `>>` と同じ意味、方向が逆 (`f >> g` = `g ∘ f`)。**記法の自由のみ**
- `|>` (パイプ): `>>` と同型
- `then`: 自然言語的表記、意味は同じ

→ **`>>` は n=1 での唯一の本質的演算子** (記法の選択を除いて)

### 1.4 化学との対応確認

単結合 (σ): 原子間の単純な電子共有 = 方向のある 1 つの接続 = 1-cell (射)
`>>` = σ 結合 ✓

## 2. n=1.5: 木構造と `{}` の強制性

### 2.1 n=1.5 の新構造

Aletheia における n=1.5 は「合成律」— 射を集めるだけでなく**合成する**能力。

圏論的には: 合成の**結合律** `(h∘g)∘f = h∘(g∘f)` が保証する**括弧の自由**、および
自由モノイダル構造 (free monoidal structure) の生成。

### 2.2 木構造 + `{}` の強制

n=1.5 には 2 つの異なる側面がある:

**① 木構造は演算子ではなく制約**

合成の結合律が強制するのは「演算子の追加」ではなく「構造的制約」。

```
A >> B >> C  =  (A >> B) >> C  =  A >> (B >> C)    ← 結合律
```

→ n=1 の `>>` が**任意の深さで入れ子になれる**という構造的自由。新演算子は不要。

**② `{}` は n=1.5 から強制される**

しかし問題がある: 「合成可能な単位」を構文上で**区切る手段**がないと、
どこまでが一つの合成ブロックかを表現できない。

```
A >> I:[cond]{B >> C} >> D       ← {} がないと書けない
A >> (B >> C)                    ← 括弧 = {} の意味論
```

`{}` の強制性の論証:

1. 自由モノイダル圏の構造: モノイダル圏では、合成の単位元 `I` と結合 `⊗` によって
   「n 個の射を一つの射に束ねる」操作が定義される
2. これを**構文上で実現する**には、「この中は一つの原子的合成」を宣言する記号が必要
3. `{}` = スコープ境界 = **自由モノイダル圏の単位コンテナ** (unit container)
4. `{}` なしには: 制御構文 `I:[]{}`, `F:[]{}`, `W:[]{}` が書けない = n=1.5 の構造を表現できない

```
なぜ n=1 に `{}` がないか:
  n=1 では単独の射 (f: A → B) を扱う。括弧は不要 — 射はそれ自体で原子的。
なぜ n=1.5 に `{}` が必要か:
  複数の射を合成して「一つとして扱う」操作 (= 結合律の帰結) が n=1.5 の本質。
  この「一つとして扱う」の構文的実現が {}。
```

→ **`{}` は n=1.5 での新演算子として強制される** (合成の単位化が要求する)

### 2.3 化学との対応

炭素鎖 C-C-C は結合の連鎖 = `>>` の連続 → 木構造。
`{}` = 官能基の境界 = 「ここからここまでが一つの構造単位」= 有機化学における **括弧** (ベンゼン環表記など)。

```
CH₃-CH(-OH)-COOH  ←→  ¥ >> I:[¥ > 0]{#} >> {# >> # >> export}
         ^括弧的な構造    ^{} がなければ書けない
```

## 3. n=2: `~` の強制性 (核心)

### 3.1 n=2 での新構造

2-圏 (n=2) に現れる新要素:
- **2-cell** (自然変換 α: F ⇒ G): 関手と関手の間の射
- **水平合成** α ⊙ β: 2-cell の横方向の合成
- **垂直合成** α ∘₂ β: 2-cell の縦方向の合成
- **交換律**: (α ⊙ β) ∘₂ (γ ⊙ δ) = (α ∘₂ γ) ⊙ (β ∘₂ δ)

**n=1 にはない本質的な新要素**: 「関手の間の比較」。

### 3.2 自然変換の普遍性質

自然変換の中で最も「基本的」なものは何か？

**随伴** F ⊣ G は以下の普遍性質を持つ:
> 「圏 C から圏 D への関手 F と、D から C への関手 G が与えられたとき、F ⊣ G とは、任意の f: F(X) → Y と g: X → G(Y) が自然に 1-1 対応する関係」

随伴の普遍性質: **全ての自然変換は随伴の unit/counit から合成される**
(これは Street の coherence theorem の一形式)

### 3.3 `~` が 2-cell の最小実現である理由

**主張**: n=2 を表現するために最小限必要な演算子は「双方向性」を表す演算子 1 つ。

**証明スケッチ**:

1. n=2 の本質は「F と G が比較可能」
2. 比較の最小形式 = 随伴 F ⊣ G (unit η: Id ⇒ GF と counit ε: FG ⇒ Id)
3. 随伴の特徴: **左右対称だが非等式**。F から G に行けて、G から F にも行けるが F≠G
4. これを構文で表現する最小演算子 = 双方向接続 `~`

**形式的論証**:
```
n=1 のみの言語: A → B と B → C から A → C を作れる
n=2 を加えた言語: F: C → D と G: D → C の間の関係を表現できなければならない
その関係の最小形式 = 随伴 (η, ε の組)
随伴を 1 つの演算子で表現する = `~` (F ~ G = F ⊣ G または G ⊣ F)
```

**代替候補との比較**:
- 2 つの矢印 (η と ε を別々に書く): `~` の展開表示。`~` は η と ε の**組**を 1 演算子で束ねる
- 自然変換を書く演算子 (α のみ): 随伴より弱い表現力。普遍性質を持たない
- 等式 F = G: 同型射は特別な自然変換だが、一般の随伴を表現できない

→ **`~` は n=2 での唯一の本質的演算子**

### 3.4 化学との対応確認

二重結合 (σ + π):
- σ 結合 (回転自由) = n=1 の `>>` — 方向のある 1-cell
- π 結合 (平面拘束) = n=2 の `~` — **2-cell が加わることで「平面の拘束」が生まれる**

直感: 2-cell は「F と G の両方向の接続」→ 行って帰れる = 電子が π 軌道で上下に分布 = 平面拘束

二重結合の回転不可 = 2-cell の可換条件が**回転を固定する**: α_Y ∘ F(f) = G(f) ∘ α_X は図式の「形」を保存し、変形の自由を制限する。 ✓

## 4. n=3: `%` の強制性

### 4.1 n=3 での新構造

豊穣圏 (Enriched category, n=3):
- Hom(A,B) が**集合**ではなく**モノイダル圏 V のオブジェクト**
- 合成が Hom の間の写像ではなく: ∘: Hom(B,C) **⊗** Hom(A,B) → Hom(A,C)

**n=2 にはない本質的な新要素**: Hom-集合の内部構造 + その間の**テンソル積** ⊗。

### 4.2 テンソル積の強制

Kelly のコヒーレンス定理 (豊穣圏):
> 「豊穣圏 V の合成写像は、V のモノイダル積 ⊗ を経由しなければならない」

これが強制する:
- ⊗ なしには「2 つの射を組み合わせる」操作を定義できない
- ⊗ を表現する演算子が必須

CCL での `%`:
```
fn_1 % fn_2    ← fn_1 と fn_2 を並列に保持する (直積)
¥ % ¥          ← 2 つの外部引数の組 (タプル入力)
```

これは Hom(B,C) ⊗ Hom(A,B) の構文表現。

### 4.3 化学との対応確認

三重結合 (σ + 2π):
- σ + 1π = 二重結合 (n=2 まで)
- 2 本目の π = テンソル積の追加 = n=3

直感: 三重結合はさらに剛性が高い (回転完全不可) = n=3 の豊穣構造がさらに制約を加える ✓

## 5. Basis: `¥` と `#` の強制性

### 5.1 Lens の公理

Bayesian Lens (Smithe):
- Get: S → A (観測)
- Put: S × B → T (更新)

この 2 つは**独立な操作**で、どちらも省略できない:
- Get なし: 環境を観測できない (盲目)
- Put なし: 観測しても更新しない (静的)

### 5.2 `¥` と `#` の強制

```
¥ (yenmark): 外部引数 = Get の構文表現 (S → A の "A" 部分)
# (hashmark): 内部状態参照 (self/cls) = Put の方向 (S の更新)

Markov Blanket の境界:
  外側 (¥) = blanket の外から入る情報
  内側 (#) = blanket の内部状態
```

Lens の公理が強制する: Get/Put の区別を構文上で明示する**2 つの**演算子が必要。
`¥`/`#` はこの区別の最小実現。 ✓

## 6. コヒーレンス定理との接続: 強制性の形式的根拠

各レベルの「強制性」は以下のコヒーレンス定理に依拠する:

| n-level | コヒーレンス定理 | 強制する演算子 | 定理の内容 |
|:--------|:--------------|:-------------|:---------|
| n=1 | Mac Lane (1963) | `>>`, `_`, `<<`, `&`, `\|`, `\` | 全ての適切に型付けられた図式は可換 = 合成・積・余積の唯一性 |
| n=1.5 | — (自由モノイダル) | `{}`, `*`, `L:[]{}` | 結合律 → 木構造 (演算子不要) + 合成の単位化 → `{}` 強制 |
| n=2 | Street (1972) | `~`, `~*`, `~!`, `>*`, `!`, `F:[]{}` | 2-圏のコヒーレンス = 随伴が 2-cell の canonical generator |
| n=3 | Kelly (1982) | `%` | 豊穣圏のコヒーレンス = ⊗ が hom-set の合成に不可欠 |
| n=∞ | — (随伴の普遍性) | `\|\|`, `\|>`, `<\|`, `lim[]{}` | 随伴が全ての limit/colimit を生成 = 4 演算子が必要 |
| Basis | Smithe (2021) | `¥`, `#` | Bayesian Lens の Put/Get 分離 = 2 演算子の必要性 |
| Meta | — (η/ε 変換) | `+`, `-`, `^`, `√` | 自然変換 η:Id⟹T / ε:T⟹Id = 深度シフト演算子 |
| FEP | FEP (Friston) | `'`, `∂`, `∫`, `Σ` | 予測誤差の微積分 = FEP 専用演算子 |

**コヒーレンス定理が「強制」を意味する理由**:
コヒーレンス定理は「可換にしなければならない図式を全て可換にする最小の演算子集合」を特定する。この集合に含まれない演算子は不要 (他から導出可能)、含まれる演算子は除去不可能。

## 7. 全体の強制定理 (候補)

**定理候補 (Aletheia-CCL Spectral Correspondence)**:

> CCL の各演算子 X は、Aletheia フィルトレーションの n-level 空間への**スペクトル射影**
> `X = Σᵢ αᵢ · πᵢ(X)` を持つ。支配的射影 (max αᵢ) はコヒーレンス定理によって強制される。
>
> 支配的射影の階層:
> - n=1 (Mac Lane): `>>`, `_`, `<<`, `&`, `|`, `\` — 1-cell の合成・積・余積・双対
> - n=1.5 (自由モノイダル): `{}`, `*`, `L:[]{}` — 合成の単位化・積・ラムダ
> - n=2 (Street): `~`, `~*`, `~!`, `>*`, `!`, `F:[]{}` — 2-cell / 随伴 / モナド
> - n=3 (Kelly): `%` — 豊穣圏のテンソル積
> - n=∞ (随伴普遍性): `||`, `|>`, `<|`, `lim[]{}` — 随伴・極限
> - Basis (Smithe Lens): `¥`, `#` — Markov Blanket 境界の Get/Put
>
> ただし多くの演算子は二次射影を持ち、単一レベルに帰属しない (**混成軌道モデル**, §8)。

**結合次数 = n-level スペクトル (累積 + 混成)**:
- 単結合 (k=1) ↔ 純粋 n=1 (100% σ) ↔ `>>` のみ
- 二重結合 (k=2) ↔ n=1 + n=2 混成 (σ + π) ↔ `>>` + `~` (sp² 的)
- 三重結合 (k=3) ↔ n=1 + n=2 + n=3 混成 (σ + 2π) ↔ `>>` + `~` + `%` (sp 的)
- 芳香環 (k=1.5) ↔ n=1 + n=∞ が非局在化 ↔ `>>` + `||` (π 電子の共鳴 = 随伴の非局在化)
- **sp³ 結合** ↔ 3レベル以上に分散 ↔ `*` (n=1.5 + n=3 + Basis)

## 8. 全 CCL 演算子のスペクトル分解

### 8.0 パラダイム転換: 全単射仮説の放棄

当初の仮定「各演算子は正確に一つの n-level に帰属する」(全単射仮説) を**放棄**する。

**理由**: `*` は n=1.5 (モノイダル積) でもあり n=3 (豊穣内積) でもあり Basis (精度加重) でもある。
`I:[]{}` は n=1 (余積) でもあり n=2 (モナド的 fiber) でもある。
一演算子が複数の n-level に**射影を持つ**のが正しい姿。

**新モデル: スペクトル分解**

各演算子 X は n-level 空間への**射影** (projection) を持つ:

```
X = Σᵢ αᵢ · πᵢ(X)
```

ここで πᵢ は n-level i への射影、αᵢ はその重み (支配的射影 = 最大 α のレベル)。

**化学との対応: 混成軌道 (Hybridization)**

| 混成型 | 化学 | CCL | 意味 |
|:-------|:-----|:----|:-----|
| 純粋 s | 100% σ | `>>` = 100% n=1 | 単レベルに完全帰属 |
| sp³ | 25% s + 75% p | `*` = n=1.5 + n=3 + Basis | 三つのレベルに分散 |
| sp² | 33% s + 67% p | `I:[]{}` = n=1 + n=2 | 二つのレベルに分散 |
| sp | 50% s + 50% p | `~` = n=2 + n=∞ | 二つのレベルに強く帰属 |

→ 演算子の n-level 帰属は「どちらか」ではなく「どの割合で」。

### 8.2 スペクトル分解の構造的意味

**重要な発見: n=2 は「混成の震源地」**

n=2 レベルの演算子は他のレベルとの混成が最も多い。これは偶然ではない:
- n=2 (自然変換/2-cell) は n=1 (射) と n=∞ (随伴) の**間**に位置する
- 自然変換は「射の間の射」= 射のレベルを参照しながら、より高い構造を表現する
- だから n=2 演算子は「下 (n=1) への射影」と「上 (n=∞) への射影」を同時に持つ

**化学的等価**: sp² 混成炭素は平面構造を取る。2次元的な拘束が生じるのは、
σ (n=1) と π (n=2) の混成が**平面を定義する**から。

**`*` (内積) の三重混成は特に注目に値する**:
- n=1.5 の側面: 二つのモノイダル対象を一つに合成する構造的操作
- n=3 の側面: 豊穣 hom-set の内積 (精度付き合成)
- Basis の側面: FEP の精度加重 (π = V[ε]⁻¹)
- → `*` は CCL の中で最も「混成度が高い」= **sp³ 演算子**

### 8.3 49d への帰結: 分子式からスペクトル構造式へ

スペクトル分解モデルは 49d 問題に新しい方向を示す:

```
分子式 (49d):    演算子を数える          C₂H₆O
構造式:          演算子の接続を記録      CH₃-CH₂-OH
スペクトル構造式: 各結合の n-level 射影重み  CH₃-[100%σ]-CH₂-[60%σ+40%π]-OH
```

**Phase C エンコーディングへの示唆**:
1. 各演算子トークンに n-level 射影ベクトル (例: [n1=0.4, n1.5=0.35, n3=0.25]) を付与
2. 隣接演算子間の「結合型」を射影の重なりから計算
3. これが「構造式エンコーディング」の候補 — 49d (分子式) が失う結合パターン情報を回復

## 10. 射影重みの定量化: 公理依存性 → U⊗ テンソル分解

### 10.0 方法

### 10.1 独立性の解釈 (重要)

Aletheia §4 の独立性基準:
> U_a ⊗ U_b が定義されるのは「U_a と U_b が**認知的に独立**に発動しうる場合」

**n-cell tower の数学的前提関係** (U_arrow ≤ U_compose ≤ ... ≤ U_adjoint) と
**認知的独立性** (同時に「忘却」できるか) は**異なる順序**。

例: U_precision(n=3) と U_causal(n=4) は tower 上で比較可能 (3 ≤ 4) だが、
「精度を忘れる」と「因果を忘れる」は認知的に独立 → テンソル積可能。
(aletheia.md §4 B2 で実証済み)

→ CCL 演算子���分解でも、tower 上の順序に関わらず認知的独立性で判定する。

### 10.3 射影重みの導出

**均一モデル** (最も保守的): αᵢ = 1/Rank

```
Rank 1: α = 1.0 (100% 単一レベル)
Rank 2: α₁ = α₂ = 0.5 (50% ずつ)
Rank 3: α₁ = α₂ = α₃ = 0.33 (33% ずつ)
```

**検証**: §8.1 の直感的見積もりとの比較

| 演算子 | 均一モデル | §8.1 直感的見積もり | 乖離 |
|:-------|:----------|:------------------|:-----|
| `>>` | n=1: 100% | n=1: 100% | 一致 |
| `*` | n=1.5: 33%, n=3: 33%, Basis: 33% | n=1.5: 40%, n=3: 35%, Basis: 25% | 小 |
| `~` | n=2: 50%, n=∞: 50% | n=2: 65%, n=∞: 35% | **中** |
| `!` | n=1: 50%, n=2: 50% | n=2: 60%, n=1: 40% | 小 |
| `I:[]{}` | n=1: 50%, n=2: 50% | n=1: 50%, n=2: 50% | 一致 |

**`~` の乖離について**: 均一モデルは 50/50 を予測するが、直感は 65/35 (n=2 寄り)。
これは `~` の**操作的用法** (日常的に 2-cell 振動として使われ、明示的随伴宣言は `||` が担う) を反映。
均一モデルは構造的対称を仮定するが、CCL の使用パターンは非対称。

→ 均一モデルは**構造的下限**として有用。実際の射影重みは均一と直感の間にある。

### 10.5 化学との同型: Rank = 空間次元

| 化学 | CCL | Rank | 空間次元 |
|:-----|:----|:----:|:---------|

- Rank 3: 3 軸が張る空間内
