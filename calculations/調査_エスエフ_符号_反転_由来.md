## 0. 結論

最有力仮説は **H-GAP** である。今回の SOURCE と deterministic 計算が示したのは、Lean `collisionKernel5` が `S_5` exact moment sequence の companion ではなく、少なくとも `m=0..4` では **常に `-S_5(m+5)` を返す anti-companion** として振る舞う、という点である。  
この不整合は、`T1=S_5`, `T2=-S_5`, `T3=(-1)^m S_5`, `T4=S_5(m+k)` (`k=1,2`) のどれでも吸収されなかった。したがって、`q=5` anomaly を convention 差や単純な shift で処理する読みは落ちる。

`H-CONV` は **formal に棄却**できる。Lean 側も Python exact 側も、どちらも同じ signed Frobenius companion 規約

`last_row(p) = (-a_0, ..., -a_{d-1})`

に従っている。違っているのは規約ではなく、**入っている多項式そのもの**である。  
`H-BASIS` は、今回テストした backward orientation 族では支持されない。forward / backward_output / backward_input / backward_both のすべてで `S_5` sequence は不変で、回収される last row も Python exact companion `[-10, 20, 8, 11, 2]` のままだった。  
`H-D2` も単純な符号 mirror 説としては落ちる。`p_python(x)` は `-p_lean(-x)` に一致せず、固有値集合も `λ → -λ` の像になっていない。

したがって本稿時点の判断は次である。

> `q=5` sign flip anomaly の主因は **Lean 実装面の gap** であり、少なくとも現在の `collisionKernel5` は `momentSum 5` の exact recurrence verification を背負った kernel としては SOURCE 上で閉じていない。

**補遺** (2026-04-24 Claude 追記): H-BASIS の棄却は §5 で Fibonacci 対称性の構造的帰結として 86% → 95% に強化。H-GAP の内部粒度は §6 で 4 候補 (HGT-GRD 35% / HGT-INV 25% / HGT-Typo 20% / その他 20%) に分解。決定的判別の次手は §6.6 を参照。

`p(x) = x^5 + a_4 x^4 + a_3 x^3 + a_2 x^2 + a_1 x + a_0`

### 2.3 Phase 2 — H-BASIS check

`Fold.lean / Value.lean / Weight.lean` を読むと、forward orientation は

- `weight` が bit position `i` に `F_{i+2}` を載せる

### 2.4 Phase 3 — H-D2 check

#### 2.4.1 polynomial witness

`λ → -λ` だけで両者が対応するなら、`p_python(x)` は `-p_lean(-x)` と一致するはずである。  
だが実際には

- `-p_lean(-x) = x^5 - 2x^4 + 11x^3 - 8x^2 + 20x + 10`
- `p_python(x) = x^5 - 2x^4 - 11x^3 - 8x^2 - 20x + 10`

であり、`x^3` と `x` の係数が食い違う。  
よって pure `λ → -λ` mirror は不成立。

#### 2.4.2 eigenvalue witness

`numpy.linalg.eigvals` で比較すると、

- Lean spectrum: 2 組の複素共役対 + 小さい正実根
- Python spectrum: 大きい正実根 + 大きい負実根 + 1 組の複素共役対 + 小さい正実根

であり、形が違う。  
最良 permutation を取っても `a_i ≈ -b_{σ(i)}` の誤差は

- max abs error = `4.5518`
- sum abs error = `11.4625`

までしか下がらない。  
これは「第二層 orientation で固有値が一斉に反転している」という像からは遠い。

## §5 構造的補遺 — Phase 2 の Fibonacci 対称性

*追記: 2026-04-24, Claude*

Phase 2 の結果 (4 variant 全てで sequence prefix が不変、回収 recurrence も不変) は deterministic 数値検証として確定しているが、その**背後構造**は Fibonacci 対称性の帰結として理論的に予測可能である。本節は Codex §2.3 の結論を構造的論拠で補強する。

### 5.1 moment_sum の representation-invariance

`momentSum(q, m) = Σ x ∈ X m, fiberMultiplicity(x)^q`

ここで:

- `X m` = stable words of length `m` (No11 制約下の長さ `m` の語) の集合 [SOURCE: Value.lean]
- `fiberMultiplicity(x) = |{w ∈ Word m : Fold(w) = x}|` [SOURCE: Fold.lean]
- `Fold(w) = X.ofNat(m, weight(w))` [SOURCE: Fold.lean]

Input orientation 変更 (weight 計算で bit position `i` に `F_{m+1-i}` vs `F_{i+2}` を載せる) は、全 mask `w ∈ {0,1}^m` を舐めると Fibonacci index の単なる rename (bijection) であり、集合としての `weight` 像は不変。Output orientation 変更 (`X.ofNat` で Zeckendorf indices の highest→lowest 配置) も `X m` の labeling 変更であり、multiplicity 分布は **label-independent な集合としての cardinality** で決まる。

したがって `momentSum` の値は Input/Output orientation 変更に対して **representation-invariant**。

### 5.2 H-BASIS 棄却の構造的根拠

H-BASIS が意味を持つためには、orientation 変更が moment_sum の数値を変えなければならない。しかし:

- orientation 変更は `X m` の labeling 変更 (集合の自己同型)
- moment_sum は label-independent な集合量
- ⟹ orientation 変更は sequence を変えない
- ⟹ Berlekamp-Massey で回収される recurrence も変わらない
- ⟹ Lean row `[10, -20, -8, -11, -2]` に落ちる variant は存在し得ない

これは Codex の deterministic 結果 (4 variant 不変) の**構造的証明**であり、Phase 2 の結論を「試験済み orientation family で棄却」から「**理論的に orientation family では成立不能**」に **confidence 86% → 95%** へ押し上げる。

### 5.3 H-BASIS が潜在的に成立する条件

H-BASIS が「hidden transform」として成立するためには、orientation 以上の変更が必要:

- multiplicity を position-dependent weight で再定義: `Σ_x w(x) · multiplicity(x)^q`
- No11 制約を他の pattern 禁止に置換 (例: Paper III anti-copy セクターの排他律)
- `weight` の非 Fibonacci 化

これらは `momentSum` の定義書き換えで、**現在の formalization 上の q=5 anomaly の範囲外**。つまり H-BASIS が残るとしても、それは「Fold そのものを別種の fold に置き換えた場合」の話であり、今回の anomaly とは独立の問題として分離できる。
