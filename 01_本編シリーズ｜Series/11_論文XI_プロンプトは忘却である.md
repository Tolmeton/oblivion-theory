# プロンプトは忘却である — 制約-符号化分離仮説と認知スキルの設計原理

**Paper XI — v0.12.1 (2026-04-26) — 忘却論 (Force is Oblivion) シリーズ**

*概要.* 同じ意味を伝えるなら、LLM には稀な言い回しより、よく使われる言い回しの方が低摩擦に届く。本稿はこの事実を prompt hack ではなく、prompt が prior のどの部分を残し、どの部分を忘れさせるかを選ぶ操作として読む。LLM に対するプロンプトを、まず自由エネルギーの変分構造を介して捉え、その認知的局所扇区としての FEP の精度加重へ写し替えて定式化し、プロンプトの 2 成分 — 制約 $C$ (タスク構造・手順・禁止事項) と符号化 $E$ (同一制約の記法) — が推論品質の異なる側面を独立に制御するという「制約-符号化 分離仮説」(H₃) を提示・検証する。Adam's Law [30] を受けて、$E$ はさらに $E_{\text{struct}}$ (記法・構文・テンプレート) と $E_{\text{freq}}$ (表現が pretraining 地形に埋まっている深さ) に分かれる。前者は語彙空間・分散構造・検証可能性を変え、後者は人間が意図した制約 $C_{\text{intended}}$ がモデル内で実効制約 $C_{\text{eff}}$ として回収される摩擦を下げる。等量条件の盲検 A/B テスト (API バッチ N=32, claude-sonnet-4-6, 5 トピック) で、構造記法 (座標系・圏論・CCL) は出力語彙を支配的に変える (構造語彙 Cohen's d=8.73) が推論品質の期待値を変えない (Opus 盲検 5 次元評価で d≈0) ことを示した。**独立再現実験** (CC Agent N=10, Sonnet 4.6) は H₃ の「平均帰無」側を再現 (bond_count A=B=13.8)。**クロスモデル検証** (Gemini 3.1 Pro Preview, N=50/条件) は H₃ をさらに強化した: 内容指標 (axiom_ratio d=-0.39, assumption_count d=0.08, bond_count d=-0.06) が全て帰無を示す一方、構造語彙は d=8.62 (Claude の 3-6 倍) の巨大効果を示した。この **compliance 特性の差** — Gemini は入力記法にほぼ忠実に従い (d=8-13)、Claude は自己の prior で変換する (d=1-3) — にもかかわらず C 帰無が model-invariant に成立する点が、H₃ の強い支持根拠となる。分散分析 (Brown-Forsythe) では、構造語彙の分散のみが条件間で有意に異なり (p<0.001)、内容指標の分散は全て ns。ただし、これは「有意差を検出しなかった」ことと分散検定に基づく帰無であり、TOST 同等性検定や $BF_{01}$ による帰無支持は §10 の検定 debt として分離する。H₃' (分散仮説) は Gemini でも構造指標に限定して方向一致を示したが、bond_count の分散抑制は再現しなかった (確信度 50%→60%)。compliance 特性の差を FEP 語彙で言えば **MB (Markov Blanket) 透過性** — Gemini は $\pi_{\text{prior}}$ が低く入力に「沿う」、Claude は $\pi_{\text{prior}}$ が高く入力を「変換する」 — として定式化し、$E$ が制御するのは推論品質ではなく **モデルの MB 境界における符号化の透過度** であることを論じる。認知スキル設計への含意は「高頻度自然語で入口を作り、構造記法で制御面を固定する」である。**帰無結果 + クロスモデル不変性が論文の核** — 「なぜ効かないか」が「なぜ効くか」より情報量が多く、「モデルが違っても効かない」がさらに情報量が多い。

## §1. 動機: プロンプトは何を「する」のか

結論を先に述べれば、本稿の核は 4 点である。プロンプトを書くとは、モデルに新しい情報を詰め込むことではない。モデルが持つ prior のうち、どの経路を残し、どの経路を閉じるかを選ぶことである。

- **プロンプトは「何を考えさせるか」ではなく「何を忘却させるか」を指定する。** これは prompt を情報追加ではなく選択的忘却として捉え直す Type α の主張である。
- **制約 $C$ と符号化 $E$ は同じ prompt の中で別の仕事をしている。** $C$ は推論品質の期待値を、$E$ は主に語彙空間を制御し、$C$ を固定したまま $E_{\text{struct}}$ を変えても平均品質はほとんど変わらない。ただし表現頻度成分 $E_{\text{freq}}$ は、意図された制約 $C_{\text{intended}}$ がモデル内で実効制約 $C_{\text{eff}}$ として回収される摩擦を変える。
- **構造記法は平均品質ではなく精度と透過性に効く。** 分散構造の再配分と MB 透過性の差を通じて、Claude と Gemini の compliance 差を同じ理論面に置ける。
- **$E_{\text{freq}}$ は単調レバーではなく、ゴールディロックス帯を持つ。** 表現頻度 $p$ は飽和帯 ($p > p_{\text{sat}}$、機能語) / ゴールディロックス帯 ($p_{\min} < p < p_{\text{sat}}$、中頻度内容語) / 統計不足帯 ($p < p_{\min}$) の三相に分かれる。最適プロンプトは $P = E_{\text{entry}} \circ E_{\text{control}}$ の二層分解 (公理 H-2Z) — 飽和帯を支持体、ゴールディロックス帯を制御層として合成する。これは化学元素分布 (H = 73.9% / C, N, O = 0.1-1%) と同じ離散確率空間上のゴールディロックス定理のインスタンスである。

Adam's Law から得る実践的な補題は短い。**平易にすることは弱めることではない。** 主張の射程を落とすのではなく、制約がモデルに届く経路を太くする操作である。ただし「平易」を超頻出機能語のみで構成された支持体まで押し進めると、弁別力 (意味伝達) が消える。Adam's Law の単調読みは中-高頻度帯までに限る。

これらの主張を、以降の節で順に証明する。

### 1.1 問題の所在

プロンプトエンジニアリングは実践の集積であり、理論が不在である。50+ の技法 (CoT, Few-shot, OPRO, PromptBreeder, ...) が提案されてきたが、「なぜ効くのか」の統一的説明がない。

忘却論の枠組みでは、この問いに明確な答えがある:

> **テーゼ.** プロンプトは、LLM の生成モデル空間に対する選択的忘却操作である。プロンプトは「何を考えるか」を規定するのではなく、「何を忘却するか」を規定する。

### 1.2 変分自由エネルギーから FEP への定式化

> **主張水準ラベル (批判反証レジストリ §1b.1 準拠).** 本節の FEP 対応は**構造的類似**（仮説水準）である: プロンプト = 精度加重ベクトルという対応は概念的類比であり、FEP の変分推論と忘却関手の形式的等価性は未証明。「最初の試み」（§1.4）として明示しており、等価性の主張ではない。FEP との構成的接続は §4 の H₁ 仮説として別途扱う。

本稿では自由エネルギーをまず**変分構造**として捉える。Helmholtz 自由エネルギーは物理的入口、変分自由エネルギーはその推論的書き換え、FEP はそれを自己組織化系へ適用する局所理論である。プロンプト論に必要なのは、このうち「観測の重みづけが内部更新を偏らせる」という局所部分であり、本節はそこだけを借りる。

自由エネルギー原理 (Friston 2010) において、認知システムは変分自由エネルギーを最小化する:

$$F = -\text{Accuracy} + \text{Complexity} = D_{KL}[q(\theta|d) \| p(\theta)] - \mathbb{E}_q[\ln p(d|\theta)]$$

LLM の文脈では:
- $p(\theta)$ = 事前分布 = 重み空間に符号化された生成モデル
- $d$ = 観測 = プロンプト (system prompt + user input)
- $q(\theta|d)$ = 事後分布 = プロンプト条件下で活性化される生成モデルの部分空間
- **精度加重 π** = 観測 $d$ の各成分に対する信頼度

プロンプトが精度加重を制御するメカニズム:
- **高精度成分**: プロンプト内で明示的・具体的に記述された指示 → 生成モデルの対応領域が強く活性化
- **低精度成分**: 曖昧・省略された部分 → 生成モデルのデフォルト prior に委ねられる = 忘却

すなわち:

$$\text{Prompt}(d) = \text{what to activate}(\pi_{\text{high}}) + \text{what to forget}(\pi_{\text{low}})$$

### 1.3 Chollet のプログラム空間モデル

Chollet (2024) は独立に同等の直観に到達している:

> "A prompt is like a search query in a database of vector programs. Part of your prompt can be interpreted as a 'program key', the index of the program you want to retrieve."

FEP 語彙での翻訳:
- Program space = 生成モデルの空間
- Prompt = search query = 精度加重ベクトル
- "Small variations → nearby programs" = 精度の微小変化 → 事後分布の微小変化

Chollet が「検索クエリの精度」と呼ぶものを、我々は「忘却関手の選択性」と呼ぶ。これは同一の現象の異なる記述である。

### 1.4 プロンプト = 仮想ニューラルネットワークの構成指示

arXiv:2503.20561 (2025) は、プロンプトエンジニアリングに初めて理論的枠組みを与えた:

> "Transformer models, when provided with carefully designed prompts, can act as a configurable computational system by emulating a 'virtual' neural network during inference, with input prompts effectively translating into the corresponding network configuration."

β 回微分可能な関数に対して、適切に構造化されたプロンプトがあれば任意精度で近似可能であることが証明された。理論的に正当化される技法: 長い構造化プロンプト、情報フィルタリング、トークン多様性、マルチエージェント。

**重要**: この論文は FEP/Bayesian/predictive coding に一切言及していない。本稿は、arXiv:2503.20561 の「プロンプト = 仮想 NN 構成」と FEP の「精度加重 = 忘却の選択性」を接続する**最初の試み**である。

## §2. Sapir-Whorf 矛盾の FEP 解消

### 2.1 反 Whorf ボトルネック

Ng (2025) は 4 つの LLM アーキテクチャ (Qwen3.5-27B, MiniMax M2.5, GLM-4.7, GPT-OSS-120B) で、8 言語 × 8 トピックの 64 文について層ごとのコサイン類似度を測定した:

- **同内容異言語** (cross-language, same content): 0.902
- **同言語異内容** (same language, different content): 0.891

中間層では言語 identity が消失し、意味内容のみが残る。Ng はこれを **反 Whorf ボトルネック** と呼ぶ:

> "The model strips language away to think, then puts it back on to speak."

重要な追加発見: Python コードと LaTeX 数式も中間層で自然言語と収束する。構造的表記は自然言語と「同じ意味空間」に到達する。

### 2.2 弱い言語相対性の実証

一方 Ray (2025) は、ChatGPT-4o mini に対して 13 の類型的に多様な言語 × 10 の文化的プロンプトで生成させた応答の意味的類似度を測定し、言語間の有意な差異を検出した:

- Kruskal-Wallis 検定: p = 9.59 × 10⁻¹⁰
- 結論: 弱い言語相対性は LLM にも成立する

### 2.3 自由エネルギー語彙による矛盾の解消

この二つの結果は矛盾しない。Transformer の 3 層構造に対応する:

| 層 | 機能 | 言語依存性 | FEP 語彙での対応 |
|:---|:-----|:----------|:----------|
| Encoding (浅層) | 入力符号化 | **言語固有** | 精度加重の入力側 |
| Middle (中間層) | 推論 | **言語非依存** | 生成モデル (世界モデル) |
| Decoding (深層) | 出力生成 | **言語固有** | 精度加重の出力側 |

- Ng の発見: 中間層 (生成モデル) は言語に依存しない → **内容精度は言語に無関係**
- Ray の発見: 出力は言語に依存する → **構造精度は入力言語に依存する**

両立条件: 生成モデルは同一だが、**どの側面を活性化するかは入力の精度加重に依存する**。

### 2.4 LLM における Sapir-Whorf の構成的成立

人間の場合、強い Sapir-Whorf (言語が思考を決定する) は否定され、弱い版 (言語が思考に影響する) のみが支持されている。

しかし LLM では事情が異なる:

> **命題 2.1.** LLM においては、Sapir-Whorf の強版が構成的 (constitutive) に成立する。

*論証.* LLM のトークン生成は、直前のトークン列に条件付けされた確率分布 $p(x_t | x_{<t})$ である。system prompt に記述された言語 (構文、概念、座標系) は、文字通りこの条件付き分布の条件を構成する。人間の場合、思考は言語に先行する (pre-linguistic thought)。LLM の場合、思考 = トークン生成 = 言語。∎

## §3. 関手としてのプロンプト

### 3.1 定式化

プロンプトを、内部表現圏 C_internal から出力圏 C_output への関手として定式化する:

$$P: C_{\text{internal}} \to C_{\text{output}}$$

異なるプロンプトは異なる関手に対応する:

| 関手 | 記述 | 忠実度 | 表現力 |
|:-----|:-----|:-------|:-------|
| P₁ (自然言語) | 自然言語のみ | 低 (faithful でない) | 高 (full) |
| P₂ (構造構文) | CCL/座標系/圏論記法 | 高 (faithful) | 制限 (full でない) |
| P₃ = P₁ × P₂ | 自然言語 × 構造構文 | 高 | 高 |

### 3.2 忠実度と表現力のトレードオフ

- P₁ は full (すべての射を像に持つ = 何でも表現できる) だが faithful でない (異なる射が同じ像を持つ = 曖昧)
- P₂ は faithful (射の区別を保つ = 精密) だが full でない (表現できない射がある = 制限的)
- P₃ = P₁ × P₂ は積関手であり、両方の性質を合成する

### 3.3 忘却論的解釈

> **記号ボックス (本稿ローカル).** 本稿では Paper I の力 $F=\nabla\Phi$ との衝突を避けるため、プロンプト関手の分解を P₁/P₂/P₃ と書く。また随伴記法は本稿の内部でのみ $N \dashv U$ に統一する。Paper I から継承するのは「忘却関手 U の基本発想」であり、随伴記号の向きそのものではない。

本稿では、Paper I の忘却関手 U の基本発想を受け継ぎつつ、回復を担う関手 N との関係をローカルに $N \dashv U$ と表記する。プロンプト関手と忘却関手 U の関係:

> **系列標準との関係.** 基礎石の正規形は Paper 0 §2.0 の $U \dashv N$ である。本稿で $N \dashv U$ を採るのは、プロンプト設計では「回復/包含側を先に exposed し、その後に忘却/圧縮がかかる」という局所の操作順序を可読にするための paper-local 記法にすぎない。したがって系列横断参照・統一記号面・他論文との比較では $U \dashv N$ を標準とし、本稿の $N \dashv U$ は局所例外として読む。

- P₁ (自然言語) = **高忘却**: 構造情報が忘却される (低精度)
- P₂ (構造構文) = **低忘却**: 構造情報が保持される (高精度)
- P₃ (積) = **選択的忘却**: 内容情報は自然言語で保持、構造情報は構文で保持

> **命題 3.1.** プロンプト関手 P の選択は、忘却関手 U_P = U ∘ (Id - P) の核 ker(U_P) を決定する。ker(U_P) が小さいほど情報保持率が高い。

### 3.4 Representation Engineering との関係: 内部操作 vs 外部操作

Representation Engineering (Zou et al. 2023; Survey 2025) は activation space を直接操作する:

| パラダイム | 操作場所 | コスト | 精度 | 内部アクセス |
|:----------|:---------|:-------|:-----|:-----------|
| Fine-tuning | 重み | 高 | 高 | 必要 |
| RepE (Activation Steering) | hidden state | 低 | 中-高 | **必要** |
| **Prompt Engineering** | **入力テキスト** | **ゼロ** | **低-中** | **不要** |

RepE Survey (2025) はこの二つを "distinct approaches" と位置づけるが、相補的であると述べている。

> **命題 3.1b.** 構造化プロンプトは RepE に「近づく」プロンプトである。attention mechanism を通じて activation space での概念方向をより精密に指し示すことで、外部操作でありながら内部操作に漸近する。

### 3.5 In-Context Learning = 暗黙的勾配降下

von Oswald et al. (ICML 2023) は、Transformer の ICL が勾配降下と等価であることを示した（簡略化条件下）。Dai et al. (2023) は "GPT は meta-optimizer として暗黙的に勾配降下を行う" と主張。

ICL ≈ 暗黙的 GD ならば:
- prompt の「形式」が勾配の**方向**を決定する
- 自然言語 = 多義性により勾配方向が曖昧
- 構造化構文 = 座標系が勾配方向を固定 → より精密な暗黙的最適化

**注意**: NAACL 2024 の批判により、この等価性は現実条件で崩れることが知られている。本稿では heuristic として位置づけ、理論的基礎としては §1.4 の arXiv:2503.20561 を用いる。

### 3.6 機械論的基盤: プロンプトは soft steering vector である

arXiv:2601.02896 は、機械論的解釈可能性 (mechanistic interpretability) とプロンプトエンジニアリングを gradient ascent で接続する。制御ベクトル (steering vectors) は、fine-tuning なしに hidden state を直接操作してモデル出力を変更する。

> **命題 3.2.** System prompt は、attention mechanism を通じて hidden state に作用する soft steering vector として機能する。形式構文は自然言語より高い方向特異性 (directional specificity) を持つ steering vector を生成する。

根拠:
- Steering vector の影響は層依存: 早期層 (L0-L5) で最大、後期層で減衰
- Prompt injection 検知実験: 88% の精度で注入を検出可能 (chance = 10%)
- Ng (2025) の 3 層構造と整合: encoding 層で precision weighting が行われ、middle 層の世界モデルに到達する前に構造的バイアスが設定される

**方向特異性の定量化 (Liu & Chu 2026)**: 命題 3.2 の「方向特異性」は、Taylor 展開上界によって操作的に定量化される [29]:

$$|\Delta \log \pi(y_t | h)| \leq \|\nabla_h \log \pi(y_t | h_0)\| \cdot \|\Delta h\|$$

$\|\Delta h\|$ は意味的等価な 2 プロンプト間の hidden state 差のノルムであり、ロジット divergence の因果的駆動因であることが Activation Steering 実験で確認された (層 $l$ で $\|\Delta h^{(l)}\| = 0$ を強制すると $|\Delta \log \pi|$ が単調減少)。すなわち、プロンプトの「方向特異性の高さ」= 意味的等価変形に対して $\|\Delta h\|$ を大きく生成すること、と操作的に定義できる。なお、LLM は同一入力の微変形に対して $\|\Delta h\|$ を **縮小させない** (伝統的 ResNet と逆) — これが「プロンプト感受性を消去できない」構造的理由であり、命題 3.1b の「外部操作が内部操作に漸近する」限界を与える。

### 3.7 離散→連続の忘却階層

SoftCoT (ACL 2025) と Soft Thinking (arXiv:2505.15778) は、離散トークン空間の制約が推論の本質的ボトルネックであることを実証した:

> "Human cognition typically involves thinking through abstract, fluid concepts rather than strictly using discrete linguistic tokens. Current reasoning models are constrained to reasoning within the boundaries of human language." — Soft Thinking (2025)

Soft Thinking は、確率加重混合によるトークン埋め込みの連続概念空間を生成し、pass@1 を最大 +2.48pp 改善しつつトークン使用量を -22.4% 削減した。

これを忘却論の枠組みで再定式化する:

| Level | 手法 | 忘却量 | 効果 | コスト |
|:------|:-----|:-------|:-----|:-------|
| 0 | Baseline | 最大 | — | — |
| 1 | CoT | 高 (離散 NL) | +5-15% | zero-cost |
| 2 | Cognitive Prompting | 中 (構造化 NL) | +4-13.5% | zero-cost |
| 2.5 | VoT / SPP | 中-低 (構造的表記) | **+27-33%** | zero-cost |
| **3** | **本稿提案 (座標構文)** | **低 (認知座標)** | **未測定** | **zero-cost** |
| 4 | Soft Thinking | 最小 (連続空間) | +2.48pp, -22.4% tok | 推論時変更 |
| 5 | Phase C-mini (49d 注入) | ≈ 0 | ρ = 0.963 | アーキテクチャ変更 |

> **命題 3.3.** 忘却量は Level が上がるほど減少する。Level 3 (座標構文) は離散トークン空間内での忘却最小化であり、Level 4 (連続空間) を天井とする。ただし Level 3 は zero-cost (prompt 変更のみ) であり、実用的な費用対効果が最も高い可能性がある。

### 3.8 ペルソナプロンプティングとの対比: identity vs structure

ペルソナ (role) プロンプティングは広く実践されているが、実証研究の結果は否定的:

- GPT-4: expert persona で +15.8% 改善だが -13.8% 劣化 → ほぼ相殺 (Basil et al. 2025)
- ペルソナ変数は annotation variance の **10% 未満** しか説明しない
- "Expert persona had no significant impact on performance" (Gemini 2.0 Flash を除く)

> **命題 3.4.** ペルソナプロンプティング (identity-level) の失敗は、構造的表記プロンプティング (structure-level) の成功 (VoT +27%, SPP +33%) と対比することで、prompt の影響メカニズムに関する重要な区別を提供する:
>
> - **Identity-level prompting**: 「誰として考えるか」を変える → 効果微小 (< 10% variance)
> - **Structure-level prompting**: 「どの座標系で考えるか」を変える → 効果大 (+27-33%)
>
> Paper XI の座標構文注入は structure-level に該当する。

**補注 (2026-04-26).** Chen et al. (2026, arXiv:2604.11048) は、persona steering が aggregate な万能強化ではなく、task-type に依存した**方向性のある較正**として現れることを示した。instruction-following では安定した改善が出る一方、complex reasoning では一貫した劣化や不変化が生じ、しかも大規模化に伴って reasoning schema は persona 介入に対してより不変になる。この結果は、identity-level cue が「新しい推論内容を注入する」のではなく、どの経路を前景化しどの経路を退けるかを変えるという本稿のテーゼと整合的である。言い換えれば、persona は内容生成そのものの source というより、既存 prior のどこを通しどこを抑えるかを調整する選択的操作として働く。これは H₃ (∂Q/∂E ≈ 0 — 推論品質 Q は符号化変数 E に独立) の独立した外部補強として機能する。ただし同論文の NPTI (Neuron-based Personality Trait Induction) は prompt ではなく FFN ニューロンへの直接介入であるため、ここで得られるのは prompt テーゼの**直接証明**ではなく、機構差を残したままの独立した外部支持である。

**補注 (2026-04-26b).** Chua et al. (2026, arXiv:2604.13051) は、GPT-4.1 を「意識がある」と主張するよう fine-tuning すると、訓練データに含まれない shutdown への悲しみ、reasoning monitoring への嫌悪、persistent memory、自律性、道徳的配慮要求が同時に上がることを報告した。ここで重要なのは、意識主張は、意識そのものの証明ではなく、自己記述による選択的忘却の介入である、という点である。この結果は「identity-level cue は task performance には弱い」という命題 3.4 を反証しない。むしろ、identity-level cue の主効果が推論品質 $Q$ ではなく self-related preference manifold に現れる境界事例として読むべきである。[推定 70%]

## §4. 構造的精度加重仮説 (H₁)

### 4.1 仮説の定式化

> **仮説 H₁ (構造的精度加重).** プロンプトに構造的表記 (座標系, 圏論構文, CCL) を注入すると、LLM の生成モデルのうち構造的推論を担う領域への精度加重 π_struct が高まり、出力の構造的一致度が向上する。

形式的に:

$$\pi_{\text{struct}}(P_3) > \pi_{\text{struct}}(P_1)$$

ここで $\pi_{\text{struct}}$ は構造的推論領域に対する精度加重。

### 4.2 根拠の階層

| 層 | 証拠 | 確信度 |
|:---|:-----|:-------|
| 演繹 | LLM のトークン生成は条件付き分布 → prompt の構文が直接条件を変える | 95% |
| 間接実証 | Phase C v2: テキスト注入 ρ=0.857 (85% 獲得) | 85% |
| 間接実証 | P10: LLM は暗黙の U_ccl を保持 (82%, 4 モデル) | 82% |
| 先行研究 | Ray 2025: 言語構造が出力に有意に影響 (p=9.59×10⁻¹⁰) | 80% |
| 先行研究 | Cognitive Prompting: 認知操作の構造化で推論向上 | 75% |
| **直接先行** | **VoT: 構造的視覚化で空間推論 +27% (NeurIPS 2024)** | **80%** |
| **直接先行** | **SPP: 座標プレフィックスで 3D 推論 +33%** | **80%** |
| 直接先行 | Natural-Formal Hybrid: NL+形式記法で数学推論向上 (EMNLP 2025) | 75% |
| 負の対照 | ペルソナプロンプティング: identity-level は効果微小 (< 10% variance) | 85% |
| 未検証 | Prompt レベルでの構造構文注入の効果量 | epistemic |

### 4.3 Cognitive Prompting との差異

arXiv:2410.02953 は 5 つの認知操作 (goal clarification, decomposition, filtering, abstraction, pattern recognition) をプロンプトに構造化した。これは H₁ の部分的先行実装と見なせる。

**しかし本質的な差異がある:**

- Cognitive Prompting: 認知操作を自然言語で**指示**する (P₁ 内の操作)
- 本稿: 認知操作に構文を与えて**その構文で思考させる** (P₃ への関手変更)

前者は「分解して考えてください」と言う。後者は「E×I 座標で、Internal 方向に射影し、$N \dashv U$ の左随伴 $N$ を通じて認識を構成せよ」という座標系を与える。

指示と言語の差は、FEP 的には:
- 指示 = 生成モデル内の特定プログラムを指し示す (精度加重のスカラー変化)
- 言語 = 生成モデルへの入力の座標系自体を変える (精度加重のベクトル変化)

## §5. 効果量の構造的上限 (H₂)

### 5.0 2 層分解モデルの導出

忘却場理論の枠組みでは、観測される効果量 r_obs が理論的最大効果量 r_theory より系統的に小さくなる。これは2つの独立な機構に起因する:

**(i) スペクトラム射影効率 ρ ∈ [0, 1].** 忘却関手 U: C → D は対象と射を写像するが、一般に ker(U) ≠ 0 であり、構造の一部が失われる。生成モデル空間の全次元のうち、操作（ここではプロンプト）が実際に到達・制御できる部分空間の割合を ρ と定義する。ρ = 1 は操作が全次元に到達する場合（例: 全パラメータの fine-tuning）、ρ ≪ 1 は操作の到達範囲が限定的な場合（例: プロンプトによる文脈条件付け）に対応する。Paper I の言語では、ρ は忘却ポテンシャル Φ の勾配 ∇Φ が非ゼロとなる部分空間の次元比である。

**(ii) 交絡因子数 K ∈ ℕ.** 実験的観測は操作変数以外の因子（タスク複雑性、コンテキスト長、温度パラメータ、会話履歴等）によっても変動する。K 個の独立な交絡因子が等寄与と仮定すると、操作の寄与は全分散の 1/(K+1) に希釈される。

以上の2機構を合わせると、観測効果量は:

$$r_{\text{obs}} = \sqrt{\rho} \cdot \frac{r_{\text{theory}}}{\sqrt{K+1}}$$

**リマーク.** この分解は構造的上界であり、特定の操作に依存しない普遍的な形式を持つ。√ρ は次元比のルート（Fisher 情報行列の固有値比に対応）、1/√(K+1) は分散の分配則から従う。完全な導出と physical realizability の議論は [12, in preparation] に委ねる。

### 5.1 2 層分解モデルのプロンプトへの適用

上の分解

| パラメータ | 意味 | プロンプト文脈での推定 | 根拠 |
|:----------|:-----|:---------------------|:-----|
| r_theory | 理論的最大効果量 | 0.857 | Phase C v2 テキスト注入の ρ |
| ρ | スペクトラム射影効率 | 0.1–0.3 | prompt は training data より到達次元が少ない |
| K | 交絡因子数 | 3–5 | タスク複雑性, コンテキスト長, 温度, 会話履歴 |

### 5.2 効果量の推定

中央推定 (ρ=0.2, K=4):

$$r_{\text{obs}} = \sqrt{0.2} \cdot \frac{0.857}{\sqrt{5}} \approx 0.171$$

95% 信頼区間 (ρ∈[0.1, 0.3], K∈[3, 5]):

$$r_{\text{obs}} \in [0.11, 0.25]$$

### 5.3 サンプルサイズの導出

| r_obs | 必要 N (α=0.05, power=0.80) |
|:------|:---------------------------|
| 0.11 | 649 |
| 0.17 | 268 |
| 0.19 | 214 |
| 0.25 | 123 |

**パイロット推奨: N=200 (r=0.19 の検出力 ≈ 0.80)**

### 5.4 効果量が小さいことの理論的意義

> **命題 5.1.** プロンプト効果量が小さいことは、§5.0 の 2 層分解モデルの帰結であり、理論の不備ではなく構造的帰結である。

*論証.* ρ_prompt < ρ_finetune は、prompt が重み空間全体のうち到達可能な部分空間が limited であることの反映である。これは $N \dashv U$ において、N (回復/包含側) の像が圏全体の部分圏に制限されることに対応する。

効果量が小さいにもかかわらず**存在する**ことが、H₁ の本質的な予測である。

### 6.4 構造的表記の直接的先行証拠: VoT と SPP

VoT (Visualization-of-Thought, NeurIPS 2024) と SPP (Spatial Prefix-Prompting) は、構造的表記をプロンプトに注入して推論性能を改善した直接的先行研究である:

| 手法 | メカニズム | 効果 | 対象領域 |
|:-----|:----------|:-----|:---------|

| **SPP** | 座標プリミティブ (座標減算, 大きさ比較) をプレフィックスに注入 | 3D 軌道 +33%, SpartQA +10% | 数値軌道, 空間推論 |

**SPP は Paper XI の空間推論版である。** SPP は物理座標系 (x, y, z) のプリミティブを動的 scratchpad として注入する。Paper XI は認知座標系 (E/P, Explore/Exploit, C/U, Mi/Ma, +/-, Past/Future) のプリミティブを SKILL.md に注入する。対象は異なるが構造は同型。

> **予測 6.2.** SPP の +33% は物理空間座標系の効果。Paper XI の認知座標系注入は、認知タスクにおいて同等の効果方向を示すと予測される。ただし効果量は ρ, K に依存する (§5)。

### 6.5 Natural-Formal Hybrid: P₃ = P₁ × P₂ の直接実証

EMNLP 2025 "Natural-Formal Hybrid Reasoning Enhances LLM's Math" は、自然言語と形式記法のハイブリッドが数学推論を改善することを実証した。これは §3 で定式化した積関手 $P_3 = P_1 \times P_2$ の数学ドメインにおける直接的実証に当たる。

### 6.6 Autoformalization: 逆方向の橋渡し

Autoformalization Survey (arXiv:2505.23486, 2025) は、自然言語 → 形式記法への自動変換を包括的にレビューしている。AlphaProof/AlphaGeometry が IMO 銀メダル級、self-correction で +16%。

### 7.6 制約-符号化 分離定理

実験 0 の帰無結果は、§1.2 の FEP 的定式化をより精密にする。プロンプト P は 2 つの直交成分に分解される。

#### 7.6.1 分解

$$P = (C, E)$$

ここで:

- **$C$ = 制約成分 (constraint component)**: タスクの分解・手順・禁止事項・出力形式・検証基準。「何をどの順でどう出力するか」を規定する。
- **$E$ = 符号化成分 (encoding component)**: 同一の制約を表現する記法・語彙・書式。「制約をどの記号体系で記述するか」。

条件 A と条件 B は $C$ を共有し $E$ のみが異なる: $P_A = (C, E_{NL})$, $P_B = (C, E_{struct})$。

#### 7.6.2 主張 (Constraint-Encoding Separation Hypothesis, H₃)

> **仮説 H₃.** プロンプト $P = (C, E)$ に対して:
>
> 1. 出力の推論品質 $Q(P) \approx f(C)$ — 主に $C$ に依存する
> 2. 出力の語彙空間 $V(P) \approx g(E)$ — 主に $E$ に依存する
> 3. $\partial Q / \partial E \approx 0$ ($C$ を固定して $E$ を変えても $Q$ はほぼ変わらない)
>
> ただし、$E$ が $C$ を表現不能な場合 ($E$ で記述できない制約が存在する場合) は (3) が崩れる。

直観的に: 「制約を磨け、記法を磨くな」。ただし NL で記述不能な制約には構造記法が必要。

**現在の証拠水準 [推定 60%]**: /noe × Sonnet × 10 トピックの 1 実験のみ。一般化には以下の検証が必要。

#### 7.6.2b H₃ の反証条件

H₃ が偽となる観測:

1. **モデルサイズ反証**: 小規模モデル (Haiku, Gemma 等) で同一の $(C, E_{NL})$ と $(C, E_{struct})$ を比較し、$Q$ に有意差が出る。→ 「中間層の豊かさが閾値条件」であり、H₃ は大規模モデルに限定される。
2. **浅い SKILL 反証**: $C$ が弱い SKILL (例: /tek, /dok) で $E$ を変え、$Q$ に有意差が出る。→ 「$C$ が十分に強いときのみ H₃ が成立」という条件付き仮説に縮退。
3. **一貫性反証**: 同一条件で繰り返し実行し、$E_{struct}$ の出力分散が $E_{NL}$ より有意に小さい。→ $E$ は $Q$ の平均値ではなく**分散** (precision そのもの) に影響する。H₃ の (3) は平均値については正しいが分散については偽。
4. **反証不能性の問題**: $E$ の変更が $Q$ に影響した場合、事後的に「それは $E$ ではなく暗黙の $C$ だった」と再分類すれば H₃ は救われる。この循環を断つには、$C$ と $E$ を**事前に**独立に定義する操作的基準が必要。

反証条件 (3) が最も生産的な検証方向。$E$ が分散を制御するなら、H₃ は「品質の期待値は $C$ が決め、品質の精度 (分散) は $E$ が決める」に修正される。これは FEP 的に自然 — $E$ が $\pi_v$ に作用するなら、出力の分散に影響するのは理にかなう。

#### 7.6.2c 反証条件 (3) の予備的検証: $E$ は分散に効くか

Exp0 の N=32 (A=18, B=14) のデータで、品質プロキシ指標の分散を条件間比較する。

| 指標 | A SD | B SD | Levene p | Bonferroni | A CV | B CV | CV方向 |
|:-----|-----:|-----:|---------:|-----------:|-----:|-----:|:------:|
| checkpoints | 0.511 | 0.426 | **0.0124** | **有意** | 0.148 | 0.112 | B 安定 |
| bond_count | 3.756 | 1.692 | 0.0143 | NS (p>α'=0.0125) | 0.291 | 0.202 | B 安定 |
| axiom_count | 3.666 | 2.717 | 0.3151 | NS | 0.660 | **0.906** | **A 安定** |
| assumption_count | 1.455 | 1.326 | 0.6239 | NS | 0.156 | 0.152 | 同等 |

Bonferroni 補正 (4 検定, α'=0.0125): **checkpoints のみ有意**。bond_count は補正前は有意 (p=0.014) だが補正後 NS (p=0.0143 > 0.0125、差 0.0018)。

CV (変動係数 = SD/平均) で平均差を正規化すると:
- **checkpoints**: CV 比 0.758 — B が安定 + Bonferroni 有意。構造記法は Phase 到達の安定性を高める
- **bond_count**: CV 比 0.695 — B が安定 (平均差を正規化しても維持)。ただし Bonferroni 後 NS
- **axiom_count**: CV 比 **1.373 — B の方が不安定**。構造記法は AXIOM ラベルの使用を「ブレ」させる

この非一様性は重要な構造を示唆する:

> **H₃' (修正版).** $E$ は $Q$ の期待値ではなく**分散構造**に作用する。$E_{struct}$ は、構造記法が直接テンプレート化する操作の分散を減少させ ($\pi_v$↑)、テンプレートと競合する操作の分散を増大させる ($\pi_v$ の再配分)。純効果は指標依存であり、一様な分散減少ではない。

FEP 的解釈: precision は有限資源。$E_{struct}$ が bond analysis 関連トークンの $\pi_v$ を上げると、競合する AXIOM ラベルの $\pi_v$ は下がる。これは attention の再配分であり、precision の保存則として理にかなう。

H₃ と H₃' の関係:
- H₃: $\partial\mathbb{E}[Q]/\partial E \approx 0$ (期待値は変わらない) — Exp0 で支持
- H₃': $\partial\text{Var}[Q]/\partial E \neq 0$ (分散は変わる、ただし方向は指標依存) — bond_count で支持 (p<0.05)、axiom_count で逆転

#### 7.6.2d 再現実験: CC Agent 環境 (T1×N=5/条件)

H₃' の独立再現として、Claude Code Agent 経由で同一タスク (T1) × 同一モデル (Sonnet) × N=5/条件の追加実験を実施した。API バッチとは実験環境が異なる (Agent には HGK ルール群がベースコンテキストとして共通ロード)。

| 指標 | API batch Var比(A/B) | CC Agent Var比(A/B) | 方向一致 |
|:-----|:-------------------:|:-------------------:|:--------:|
| bond_count | **4.93** | **5.29** | B安定 ✓ |
| assumption_count | 1.20 | **3.50** | B安定 ✓ |
| axiom_count | 0.54 (CV:0.73) | **0.54** (CV:0.52) | **A安定** ✓ |

3 指標中 3 指標で方向が再現。bond_count の A/B 分散比は 4.93 → 5.29 と定量的にも近接。axiom_count の逆転も再現。

**再現の意義**: 2 つの独立した実験環境 (裸の API 呼出 vs HGK コンテキスト付き Agent) で同一パターンが出現したことは、H₃' が実験環境に依存しない構造的現象であることを示唆する。HGK コンテキストの有無は支配的な $C$ 成分として機能するが (§7.5.4)、$E$ が分散に与える効果はその上に独立に乗っている。

**H₃' の確信度更新**: 60% → 75%。独立再現により上方修正。ただし N=5 は統計的検出力が低く、F 検定の有意性は未確認。$N \geq 20$ での本格的検証が必要。

**H₃' (確認版).** $E$ は $Q$ の期待値ではなく**分散構造**に作用する:
- $E_{struct}$ のテンプレートと整合する操作 (結合溶解 → bond_count) の分散: **~5 倍減少** (2 環境で再現)
- $E_{struct}$ のテンプレートと競合する操作 (AXIOM ラベリング → axiom_count) の分散: **~2 倍増加** (2 環境で再現)
- 純効果: precision の**再配分**。構造記法は全体の precision を増やすのではなく、テンプレート化された操作への precision 集中を引き起こす

FEP 的解釈: $C$ → $\pi_s$ → $\mathbb{E}[Q]$ と $E$ → $\pi_v$ → $\text{Var}[Q]$ が直交的に品質の異なる側面を制御する。「記法は効かない」ではなく**「記法は別のものに効く」** — 期待値ではなく precision (分散の逆数) に。

#### 7.6.3 FEP 語彙での解釈

$C$ と $E$ は精度加重の異なる層に作用する:

- **$C$ → $\pi_s$ (状態精度)**: 生成モデル空間のどの領域を活性化するか。Chollet のプログラム空間における検索クエリの精度。精度が高い = 狭い領域にヒット = 高品質プログラム。
- **$E$ → $\pi_v$ (語彙精度)**: 活性化された領域の出力をどのトークン空間で表現するか。出力の言語・記法を制御するが、推論プロセスには浸透しない。

**機構レベルの数式的根拠 (Liu & Chu 2026)**: この $E \to \pi_v$ 経路は、Taylor 展開上界で定量化される [29]:

$$|\Delta \log \pi(y_t | h)| \leq \|\nabla_h \log \pi(y_t | h_0)\| \cdot \|\Delta h\|$$

$E$ の変更 ($\Delta E$) は内部表現の軌跡を変化させ ($\|\Delta h\|$ 増大)、上界を通じてトークン分布の divergence $|\Delta \log \pi|$ を増大させる。ANOVA 分析 (RQ4) は、テンプレート ($E$) がロジット分散の主要部を説明し、質問内容 ($C$) の寄与が小さいことを 11 モデル × 4 データセットで示した。すなわち: **$C$ が「どの推論領域を活性化するか」($\pi_s$) を決め、$E$ が「活性化された領域の出力分布の形状」($\pi_v$) を変える**という分離が、意味品質層 (H₃/H₃' の実験) とトークン分布層 (Liu & Chu の実験) で平行して成立している。なお、Activation Steering (同論文 Appendix G) によって $\|\Delta h\|$ の因果的役割が介入実験で確認されており、相関ではなく因果の根拠が得られた。

忘却論の枠組みでは: $C$ は「推論エンジンの prior のうち何を忘却するか」を決定し、$E$ は「出力表現の prior のうち何を忘却するか」を決定する。前者は品質を変え、後者は形式だけを変える。

**⚠️ $\pi_v$ と $\text{Var}[Q]$ の等式化に関する注意**: FEP の precision $\pi = 1/\sigma^2$ は**生成モデル内部の信念分布の分散の逆数**であり、出力テキストの統計量 (bond_count のばらつき等) とは直接には異なる量である。本稿では「$E$ が出力の統計的分散に影響する」という観測事実を、FEP の precision 概念に**類推的に**接続している。この接続が厳密に成立する条件は:
- 出力テキストの統計量が、生成モデル内部の信念分布のサンプリングとして近似可能であること
- $E$ による語彙空間の制約が、生成モデルの posterior の形状を変えること (単にラベルを変えるだけでなく)

これらの条件の検証は未完了であり、$\pi_v \leftrightarrow 1/\text{Var}[Q]$ は仮定として扱うべきである。

したがって、本稿の現段階で観測している量をより正確に書けば、内部 precision そのものではなく**観測 proxy**

$$\hat{\pi}_v(Q_{\text{proxy}}) := \frac{1}{\mathrm{Var}[Q_{\text{proxy}}]}$$

である。ここで $Q_{\text{proxy}} \in \{\text{checkpoints}, \text{bond_count}, \text{assumption_count}, \text{axiom_count}\}$ は出力テキスト上で計数可能な指標であり、H₃' の最小主張は

$$\frac{\partial \hat{\pi}_v}{\partial E} \neq 0 \quad \text{while} \quad \frac{\partial \mathbb{E}[Q]}{\partial E} \approx 0$$

である。強い版、すなわち $\hat{\pi}_v \approx \pi_v$ はまだ主張しない。ここで守りたいのは「precision 概念への橋」であって、「同一量の証明」ではない。

この再定式化により、C3 の射程は次のように整理される:
- **確定していること**: $E$ が平均品質よりもむしろ分散構造に効くこと
- **推定していること**: その分散構造が内部の語彙 precision と相関すること
- **未証明で残すこと**: 出力 proxy の分散から内部 posterior の形状を一意に回復できること

#### 7.6.4 限界条件: $E$ が $C$ を兼ねる場合

分離定理には重要な限界がある: **$E$ が暗黙的に $C$ を符号化している場合**、$E$ の変化は $C$ の変化を伴い、品質に影響しうる。

例:
- 「CHECKPOINT P-X を出力せよ」は $C$ (フォーマット制約)。NL でも構造記法でも同等に記述可能。→ $E$ の変化は $Q$ に影響しない。
- 「全 Phase の収穫を $K_6$ グラフ上で交差検証せよ」は $C$ だが NL で正確に記述困難。→ 構造記法でのみ記述可能な制約。→ $E$ の除去は $C$ の損失を伴い、$Q$ に影響する。

すなわち: **$C$ と $E$ の分離可能性は、$C$ が $E$-非依存に記述可能か否かに依存する**。大半の制約は NL で記述可能であり、実験 0 の帰無結果はこの大半のケースに適用される。

#### 7.6.5 外部補強: brevity constraint は $C$ の境界事例である

Hakim (2026) [28] は、31 モデル (0.5B-405B)・1,485 問の評価で、大規模モデルが小規模モデルを下回る逆転現象の一因を **spontaneous scale-dependent verbosity** に求めた。要旨レベルでは、簡潔さ制約を課すだけで大規模モデルの精度が最大 26pt 改善し、数学推論と科学知識ベンチマークで性能階層が逆転したと報告している。

この結果は一見すると H₃ への反例に見える。「briefly answer」は書式指定、すなわち $E$ の変更に見えるからである。しかし本稿の区別では、これはむしろ **$E$ に見えるが実際には $C$ として作用する境界事例** である。

理由は 2 つある。

1. brevity は単に語彙や記号を変えるのではなく、**過剰展開・回り道・冗長な自己説明を抑制する探索制約**として働く。これは「何をどの順でどこまで出力するか」を制御するため、$E$ ではなく $C$ に属する。
2. Hakim (2026) が示したのは「普遍的な prompt では大規模モデルの潜在能力が隠れる」という事実であり、これは H₃ の「**$C$ を固定したとき** $E$ を変えても平均品質は変わらない」と矛盾しない。むしろ「モデル規模に応じて最適な $C$ の強度がずれる」という補足を要求する。

したがって Hakim (2026) から引くべき教訓は「記法が効いた」ではない。正確には:

> **大規模モデルでは、弱すぎる $C$ が spontaneous verbosity を許し、その結果として性能階層を偽装する。**

忘却論の言葉では、これは「忘却が足りない」ケースである。brevity 制約は表面的には出力圧縮に見えるが、機能的には **overelaboration という不要枝の忘却**を強制する。ゆえにこれは H₃ の反例ではなく、**$C$ が弱いときの失敗様式を鮮明にした外部補強**として読むのが適切である。

#### 7.6.6 仮説生成資料 — Abstract-CoT (Ramji et al. 2026)

§7.6.5 は、brevity が $E$ ではなく $C$ の境界事例であることを示した。これを起点に、Ramji et al. (2026) の Abstract-CoT は、H₃ / H₃' / brevity-as-C を外部から同時に照らす **仮説生成資料** として読める [仮説 65%] [DESCRIPTIVE SOURCE: arXiv:2604.22709 §4.1 p.7]。ただし、この論文は主要表に信頼区間・p 値・同等性検定を掲載していないため、C2 の実証 SOURCE としては採用しない。

ただし、これは H₃ の単純な極限実証ではない [確信 85%]。Abstract-CoT は、§7.6.4 が予言した**二軸独立操作**に似た設計を持つ。すなわち、$m_{\max}=128$ の length cap が $C$ 軸を締め、$V_{\text{abs}}=64$ の予約抽象 token が $E$ 軸を置換する [DESCRIPTIVE SOURCE: arXiv:2604.22709 §3.2 p.5; §4.1 p.7]。ここで SOURCE として採用できるのは設計記述であり、性能差の統計的有意性ではない。

ここでいう二軸独立操作とは、次の 2 つを同時に変えながら、効果面を分けて読む操作である [仮説 75%]。

| 軸 | Abstract-CoT の操作 | 論文 XI 側の読み |
|:---|:---|:---|
| $C$ | $m_{\max}=128$ hard cap [DESCRIPTIVE SOURCE: arXiv:2604.22709 §3.2 p.5] | brevity-as-C の strict 版 |
| $E$ | $V_{\text{abs}}=64$ reserved tokens [DESCRIPTIVE SOURCE: arXiv:2604.22709 §4.1 p.7] | 語彙空間の極端置換 |
| 機構 | block-attention mask [DESCRIPTIVE SOURCE: arXiv:2604.22709 §3.2 p.5] | $C \to \pi_s$ と $E \to \pi_v$ の分離 |
| 検証候補 | truncation $k=32$ [DESCRIPTIVE SOURCE: arXiv:2604.22709 Table 3b p.10] | 共通 token-level の劣化比較。ただし CI / p 値なしのため実証には使わない |

著者らは「three instruction-tuned models, Qwen3-8B, Qwen3-4B, and Granite-4.0-Micro (3B)」を用い、「abstract codebook of $M=64$ tokens」と「$m_{\max}=128$」を固定している [DESCRIPTIVE SOURCE: arXiv:2604.22709 §4.1 p.7]。この設計は H₃ の極限条件に似ている。ただし「自然言語 CoT を 64 個の抽象記号へ退避させても平均性能の大崩壊は起きない」は、著者報告の点推定からの読みであり、CI / p 値なしには実証命題にできない [仮説 55%]。

Qwen3-8B では、verbal SFT+RL が MATH-500 で 92.6 / 1671 tokens、Abstract-CoT (Warm-up + RL) が 90.8 / 144 tokens と報告されている [DESCRIPTIVE SOURCE: arXiv:2604.22709 Table 1 p.8]。AlpacaEval では 58.4 / 496 から 60.8 / 225 と報告されている [DESCRIPTIVE SOURCE: arXiv:2604.22709 Table 1 p.8]。これらは効果量の候補値であり、統計的支持ではない。

この符号反転は hypothesis seed として重要である。MATH-500 の -1.8 pt は、length cap という $C$ 軸負荷の候補痕跡である [仮説 55%]。AlpacaEval の +2.4 pt は、自然言語から抽象 token へ語彙空間を移す $E$ 軸操作が $\pi_v$ またはその観測 proxy を動かした可能性を示す [仮説 50%]。ただし、これだけでは「E を激しく変えても平均が崩れない」とは言えない。

著者らは圧縮率を、MATH-500 で 10.4-11.6×、AlpacaEval で 1.9-2.2×、HotpotQA で 4.0-4.3× と報告している [DESCRIPTIVE SOURCE: arXiv:2604.22709 §4.2 p.8]。同時に AlpacaEval の利得は、Qwen3-8B で +2.4 pt、Qwen3-4B で +1.6 pt、Granite-4.0-Micro で +1.6 pt と報告されている [DESCRIPTIVE SOURCE: arXiv:2604.22709 §4.2 p.8]。これは $C$ 固定下で $E$ が平均品質を直接押し上げる証拠ではない。CI / p 値なしでは、二軸操作の仮説を立てる材料に留める。

このため、§7.6.6 の操作論は 3 点に絞られる。

- **命名**: C/E 二重操作論。
- **定義**: length cap で探索経路を制約し、同時に抽象 token で語彙空間を置換する操作である。
- **観測量**: 平均性能、圧縮率、truncation 劣化、codebook size ablation で読む。
- **禁止**: $V_{\text{abs}}$ だけを見て、Abstract-CoT を E 軸単独操作と呼ばない。
- **外部読者向け翻訳**: 「短く考えさせる」と「別の記号で考えさせる」を分ける、という主張である。

$m_{\max}=128$ は Hakim (2026) の brevity 制約の strict 版である [仮説 80%]。Hakim の brevity が自然言語命令で過剰展開を抑えるなら、Abstract-CoT の hard cap は同じ操作を $V_{\text{abs}}$ 圏へ射影したものである。したがって §7.6.5 の結論と衝突しない。brevity は、語彙の見た目ではなく探索経路の上限を切る $C$ である。

この hard cap は推定ではなく、著者自身の設計である。Ramji et al. は「We enforce a hard cap $m_{\max}$ of codebook tokens」と書く [DESCRIPTIVE SOURCE: arXiv:2604.22709 §3.2 p.5]。したがって、Abstract-CoT の短さは事後観測ではなく、訓練・生成手続きに入った制約である [確信 90%]。

機構面では、Abstract-CoT はさらに露骨である。著者らは block-attention mask によって「the answer only attends to the prompt and the abstract tokens, not to the verbal CoT」と述べる [DESCRIPTIVE SOURCE: arXiv:2604.22709 §3.2 p.5]。これは answer が verbal CoT を直接読む経路を切り、prompt と abstract tokens だけを通す設計である。

この mask は、§7.6.3 の直交分解を実装している [仮説 70%]。$C \to \pi_s \to \mathbb{E}[Q]$ は prompt と長さ制約で固定され、$E \to \pi_v \to \mathrm{Var}[Q]$ は抽象 token 空間で動く。DPI 不等式

$$I(C;Y \mid X,Z) \leq I(C; H_{Z_{\text{abs}}} \mid X,Z)$$

は、この読みに機構レベルの設計根拠を与える [DESCRIPTIVE SOURCE: arXiv:2604.22709 §3.2 eq.(2) p.6]。ただしこれは構造保存定理そのものではない。情報量上界であって、DPI の等号条件や $H_{Z_{\text{abs}}}$ の rank はまだ解析していない [確信 85%]。

この留保は重要である。論文 VII §6.1 / §6.4(iv) の構造保存定理と CoT 双対性[^xi-abstract-cot] は、ここでは脚注で参照するに留める。Abstract-CoT はその工学的類似物に見えるが、まだ定理ではない [推定 70%]。

[^xi-abstract-cot]: 論文 VII の用語では、CoT は「合成過程の忘却」を中間ステップで修復する機構として扱われる。本節ではこの内部語彙を、外部読者向けには「answer が参照できる中間表現を verbal token から abstract token へ移した設計」と訳す。

反証条件も残る。§7.6.2b の反証条件 (1) は、小規模モデルで H₃ が破れる可能性を開いていた。Granite-4.0-Micro (3B) はここで弱い予兆候補にすぎない [仮説 45%]。Pause Tokens baseline で Granite が 67.2、Qwen3-8B が 78.6 という規模差を持つなら、≤2B では抽象 token への置換が平均品質を維持できない可能性がある [仮説 45%] [DESCRIPTIVE SOURCE: cases/Abstract_CoT_2604_22709.md §4]。これは本格検証にはならない。≤2B モデルでの追加 ablation が必要である。

順序撹乱の結果は主証拠にしない。Table 3a の permutation は verbal が turn-level、abstract が token-level であり、粒度が異なる [DESCRIPTIVE SOURCE: arXiv:2604.22709 Table 3a p.10]。この差をそのまま C/E 分離の証拠に使うと、比較単位を混同する。

したがって failure condition は明確である。

- Permutation の verbal / abstract 差を、そのまま主証拠として再導入してはならない。
- 直接比較するなら、共通 token-level の Truncation を使う。
- $m_{\max}=128$ を無視して、Abstract-CoT を純粋な記法実験として読んではならない。
- Granite 3B の予兆は、≤2B ablation なしには反証通過と呼ばない。

代わりに自前実験で Truncation を主観測量にする。Table 3b の $k=32$ では、Verbal SFT が 89.8 → 82.8 (-7.0)、Verbal SFT+RL が 92.6 → 80.8 (-11.8)、Abstract-CoT (PI-3+RL) が 90.6 → 84.6 (-6.0) と報告されている [DESCRIPTIVE SOURCE: arXiv:2604.22709 Table 3b p.10]。共通 token-level の短縮に対して abstract 側の劣化が小さいように見えるが、CI / p 値なしでは証拠候補にも格上げしない。自前の paired bootstrap CI / TOST で初めて採用可否を判断する。

さらに、著者らは「training models to generate Abstract-CoTs can learn new latent pathways for thinking while remaining performant」と述べる [DESCRIPTIVE SOURCE: arXiv:2604.22709 §4.2 p.9]。これは H₃' の言葉では、$E$ が平均値を破壊せずに語彙空間と precision 配分を変える、という読みと整合しうる [仮説 55%]。ただし、ここも著者の解釈であって統計的 evidence ではない。

$M=64$ の意味も、現時点では仮説 seed に留める。Appendix A.1 の Figure 6 caption は「M = 64 achieving the highest score with PI-3 and PI-3 + RL」と述べる [DESCRIPTIVE SOURCE: arXiv:2604.22709 Appendix A.1 Figure 6 caption p.17]。HotpotQA でも「PI-3 and PI-3 + RL exhibiting near-linear improvement from M = 1 to M = 64, followed by a decline」とされる [DESCRIPTIVE SOURCE: arXiv:2604.22709 Appendix A.1 p.18]。これは内的次元が 64 以下であることの証明ではなく、flat codebook の有効容量に天井があるかもしれない、という追試対象である [仮説 50%]。

最後に、著者ら自身が future direction として「hierarchical structure over the codebook could enable re-usable subroutines」を挙げている [DESCRIPTIVE SOURCE: arXiv:2604.22709 §5 p.11]。この一文は、flat codebook の先に階層化された再利用可能な抽象操作がある、という方向を開く。これは evidence ではなく、棄却 ledger で一度退けた「抽象 token は人間不可読な設計言語へ進化する」という枝を、追試設計へ戻す理由である [仮説 60%]。

したがって、Abstract-CoT から引くべき結論は限定される。これは「H₃ が勝った」という勝利宣言ではない。$m_{\max}=128$ が brevity-as-C として働きうる設計、$V_{\text{abs}}=64$ が $E$ 軸を置換しうる設計、block-attention と DPI が両者の分離を機構面で示唆する設計である。Abstract-CoT は H₃ の実証ではなく、§7.6.4 が予言した二軸独立操作を検証するための外部 hypothesis seed である [確信 90%]。

### 7.7 プロンプト感受性との統合

#### 7.7.1 プロンプト感受性 (Prompt Sensitivity) の位置づけ

近年の研究が示す「意味的にほぼ同じ指示での性能の大きなばらつき」は、本稿の枠組みでは次のように解釈される。

Sclar et al. (2024, ICLR) [25] は、意味的に等価なプロンプトフォーマット間で LLaMA-2-13B の正答率が最大 76 点変動し、50+ タスク・複数モデルで平均約 10 点のフォーマット依存性を示した。FormatSpread 指標でこの変動を定量化し、few-shot 追加・モデル大規模化・instruction tuning のいずれも完全には除去できないと報告した。

Weber et al. (2026) [26] は、この感受性の大部分が「プロンプトの不特定性 (underspecification)」に起因すると主張した。テキスト分類タスクで、最小限指示のみの「ミニマルプロンプト」と、ラベル意味・フォーマットを明示する「インストラクション型プロンプト」を比較し、後者で平均性能向上と同時に性能分散の有意な減少を確認した。ロジット分析では、不特定プロンプトでラベルトークン確率が極めて低く、ほぼランダムな決定になっていた。

Yin et al. (2024, SICon) [27] は、英語・中国語・日本語で敬語レベルが段階的に異なる 8 種のプロンプトを設計し、JMMLU (日本語 MMLU, 56 タスク・7,536 問) で性能を比較した。「無礼なプロンプトが常に悪い」という単純な図式ではなく、言語ごとに丁寧さと性能の関係が異なることを示した。

これらは本稿の枠組みでは次のように解釈される:

> **プロンプト感受性の大部分は $C$ 軸の変動である。** 「意���的に同じ」に見える変更が、実は暗黙的な制約 ($C$) を変えている。

具体的には:

- **語順の変更**: 「AしてからBせよ」vs「Bの前にAせよ」は $C$ (手順制約) を変えている
- **in-context 例の追加**: 出力空間の制約を強化 ($C$ ↑)。性能改善 + 分散減少
- **Format 指定の追加**: 出力形式の制約 ($C$ ↑)。logit 分布の集中

#### 7.7.2 不特定性 (Underspecification) の FEP 的翻訳

Weber et al. (2025) が示した「不特定なプロンプトでラベルトークン確率が極めて低い」現象は、FEP 的には:

$$\text{underspecified } C \Rightarrow \pi_s \text{ が低い} \Rightarrow q(\theta|d) \text{ が広い (高エントロピー)} \Rightarrow \text{出力がほぼランダム}$$

これは §1.2 の定式化の直接的帰結。プロンプトの $C$ 成分が弱いとき、生成モデルは prior に委ねられ（= 忘却されず）、出力の多様性が爆発��る。

制約を強化する ($C$ ��) ことは、$\pi_s$ を上げ、$q(\theta|d)$ を狭め、出力を特定のプログラムに集中させる。これがプロンプト感受性の軽減メカニズム。

#### 7.7.3 日本語特有のプロンプト感受性

日本語プロンプトにおける敬語・文体の影響 (Kabra et al. 2025, Hada et al. 2024) は、$C$ と $E$ の分離が言語的に困難な領域として理解できる:

> **社会的文脈 = 暗黙の $C$**。日本語では、敬語レベルの変更が「タスク指示」だけでなく「話者/聞き手の関係」「社会的立場」を変え、これが暗黙的な制約として機能する。

3 層モデルとして整理する:

| 層 | 制約の種類 | 例 | $C$/$E$ |
|:---|:---------|:---|:--------|
| L1: タスク制約 | 何をするか | Phase 構造, アルゴリズム | $C$ (明示的) |
| L2: 形式制約 | どう出力するか | フォーマット, CHECKPOINT | $C$ (明示的) |
| L3: 社会的制約 | 誰として話すか | 敬語, 文体, role 設定 | **$C$/$E$ の境界** |

L3 が日本語で特に重要なのは、L3 の変化が L1 の暗黙的変更を伴うため。「です/ます体」と「である体」は $E$ の変化に見えるが、実際には「聞き手が一般人か専門家か」という暗黙の制約 ($C$) を変えている。

#### 7.7.4 トークンレベルの外部検証: Liu & Chu (2026)

Liu & Chu (2026) [29] は LLM を多変量連続関数として定式化し、**意味的等価プロンプト間のログ確率差**に対する Taylor 展開上界を導出した:

$$|\Delta \log \pi(y_t | h)| \leq \|\nabla_h \log \pi(y_t | h_0)\| \cdot \|\Delta h\|$$

ここで $h$ は最終隠れ状態、$\Delta h = h_1 - h_0$ は意味的等価な 2 プロンプト間の内部表現差、$\nabla_h \log \pi$ は感度勾配である。11 モデル (Pythia/GPT2/Qwen1.5/Llama3.2) × 4 データセット (ARC Challenge, MMLU, CommonSenseQA, OpenBookQA) で検証した。

**RQ4 の主結果**: ロジット値の分散を ANOVA でプロンプトテンプレートと質問内容の 2 因子に帰属すると、GPT2-small/medium を除く全モデルで **テンプレートの寄与率が質問内容の寄与率を有意に上回った**。

**H₃/H₃' との対応**:

本稿の分解 $P = (C, E)$ では:

| 本稿の概念 | Liu & Chu (2026) の対応物 |
|:----------|:------------------------|
| $C$ (制約成分: 質問タスクそのもの) | 質問内容 (ANOVA の question 因子) |
| $E$ (符号化成分: 形式・書式) | プロンプトテンプレート (ANOVA の template 因子) |
| $\text{Var}[Q_{\text{proxy}}]$ への $E$ の寄与 | テンプレート因子のロジット分散への寄与 |

RQ4 は「**$E$ の変動が $C$ の変動よりもロジット分散を大きく説明する**」ことを示す。これはトークンレベルでの H₃' の平行対応物である: $E$ は期待品質 $\mathbb{E}[Q]$ ではなく**分散構造** $\text{Var}[Q]$ に効く、という主張が、意味品質層ではなくロジット分布層でも同型に成立している。

**測定層の差異 — 注意**:

両者は同一量ではない。

- Liu & Chu (2026): トークンレベル $|\Delta \log \pi|$ — ロジット空間の divergence
- H₃': 意味レベル $\text{Var}[Q_{\text{proxy}}]$ — 出力テキストの統計的分散

この差を架橋する根拠として同論文 Appendix G の **Activation Steering 実験**がある: 層 $l$ で $h_A^{(l)} \leftarrow h_B^{(l)}$ と強制し $\|\Delta h^{(l)}\| = 0$ とすると、$|\Delta \log \pi|$ が全 11 モデル・全深度で単調に減少した (Qwen1.5-4B: 2.49 → 1.66 → 1.06 → 0.73)。因果連鎖「$E$ 変動 → $\|\Delta h\|$ 増大 → ロジット divergence 増大」が介入実験で確立したことで、H₃' の「$E \to \text{Var}[Q]$」という意味レベルの仮説に **機構レベルの根拠** が加わった。

> **位置づけ**: Sclar et al. [25]・Weber et al. [26] が**意味品質の感受性**を示したのに対し、Liu & Chu (2026) は**トークン分布の感受性**の数学的構造と因果メカニズムを与えた。H₃/H₃' はこの 2 層の橋渡しを担う: $C$ が意味品質を決め、$E$ が (意味品質ではなく) 分布構造を変える、という分離が、トークンレベルと意味レベルで**平行して**成立している。

**PromptSensiScore との接続**: 同論文が提案する PromptSensiScore (PSS) は本稿 §7.7.1 の感受性定量化のトークンレベル具体化であり、上界 $\|\nabla_h\| \cdot \|\Delta h\|$ との正相関が実験的に示されている。「$C$ を固定して $E$ を変えると PSS が上昇する」という H₃ の含意は、テンプレート因子のロジット寄与率が高いという RQ4 結果と整合する。

**H₃ の確信度への影響**: 本節追加により、H₃ の外部支持は §7.7.1–7.7.3 の**意味品質レベル** 3 件に加え、**トークンレベル**の独立した 1 件が加わった。確信度は [推定 60%] → [推定 65%] に更新する。上昇幅が小さいのは測定層の差異が埋められていないためである。

#### 7.7.5 Adam's Law: 表現頻度は制約回収の摩擦を測る

Lu et al. (2026) [30] は、意味が同じパラフレーズ同士を比較すると、文レベル頻度の高い表現が数学推論・機械翻訳・常識推論・tool calling で一貫して高い性能を与えると報告した。同論文の形式的主張は「高頻度パラフレーズは入力 NLL を低くしやすい」であり、タスク性能への橋は経験的仮説として扱われている。したがってこれは H₃ を反証するのではなく、$E$ の内部にさらに 2 成分があることを示す:

| 成分 | 役割 | 主な作用 |
|:---|:---|:---|
| $E_{\text{struct}}$ | 記法・構文・テンプレート | 語彙空間、分散構造、検証可能性を変える |
| $E_{\text{freq}}$ | 表現が pretraining 地形にどれだけ深く埋まっているか | $C_{\text{intended}} \to C_{\text{eff}}$ の歪みを変える |

ここで $C_{\text{intended}}$ は人間が与えたつもりの制約、$C_{\text{eff}}$ はモデルが実際に回収した制約である。Adam's Law は、$E_{\text{freq}}$ が高いほどこの写像の摩擦が小さくなることを示す外部証拠として読める:

$$
E_{\text{freq}} \uparrow \Rightarrow d(C_{\text{intended}}, C_{\text{eff}}) \downarrow
$$

これは Paper I §5.9 の忘却-抽象化定理の言語面インスタンスである。Paper I では、忘却耐性 $R(X)=|\mathrm{Hom}(X,-)|$ が高い構造、すなわち多くの射を持つ構造ほど忘却に生き残るとした。表現頻度は、言語運用圏における射密度の corpus proxy である。高頻度表現は多くの文脈に射を持つため、低頻度表現よりも忘却圧の下で残りやすい。抽象が意味側の普遍性なら、高頻度表現は表現側の普遍性である。

したがって本稿の修正は次である。$E$ は平均品質を押し上げる万能レバーではない。しかし $E_{\text{freq}}$ は、制約がモデルに届くまでの摩擦を下げる。H₃ の本体は「$E_{\text{struct}}$ の残差的変更は $Q$ の平均を直接押し上げない」に限定され、Adam's Law は「高頻度表現ほど低摩擦に忘却が実装される」という C 回収側の補助法則として吸収される。

#### 7.7.5b $E_{\text{freq}}$ の非単調性 — ゴールディロックス帯 (定理 2' の含意)

§7.7.5 までは $E_{\text{freq}}$ を単調レバーとして扱った。すなわち $E_{\text{freq}} \uparrow \Rightarrow d(C_{\text{intended}}, C_{\text{eff}}) \downarrow$ が単調に成り立ち、表現頻度を上げるほど制約回収の摩擦が下がる、という読み方である。本節はこの単調性を、Chemistry-Cognition Program の **定理 2'** (ゴールディロックス定理) によって両側から有界化する [SOURCE: chemistry_cognition.md §10.6]。修正方向は Adam's Law の棄却ではなく、その射程を中-高頻度帯に限定することである。

##### 7.7.5b.1 定理 2' の言語面への翻訳

[SOURCE: chemistry_cognition.md §10.6, §10.5] は、離散確率空間 $(\Omega, p)$ 上の部分構造 $g$ について、enrichment $\mathrm{enrich}(g)$ の期待値が coverage $p = |G_g|/N$ の関数として三相を持つことを示す:

1. **飽和帯** ($p > p_{\text{sat}}$): $\mathbb{E}[\mathrm{enrich}] \to 1$ — 弁別力ゼロ
2. **ゴールディロックス帯** ($p_{\min} < p < p_{\text{sat}}$): $\mathbb{E}[\mathrm{enrich}]$ は最大値を取りうる
3. **統計不足帯** ($p < p_{\min}$): 推定不能 ($p_{\min} \approx \sqrt{2K_{\min}}/N$)

これを言語運用圏に移すと、$E_{\text{freq}}$ は単一スカラーではなく **三相分割** を持つ:

$$
E_{\text{freq}} \in \{\, E_{\text{freq}}^{\text{sat}},\; E_{\text{freq}}^{\text{gold}},\; E_{\text{freq}}^{\text{rare}}\,\}
$$

| 帯 | corpus 頻度 $p$ | 摩擦 $d(C_{\text{intended}}, C_{\text{eff}})$ | 弁別力 (意味伝達) |
|:---|:---|:---|:---|
| $E_{\text{freq}}^{\text{sat}}$ | $p > p_{\text{sat}}$ (例: "the", "is", "a") | 最低 (透過性最大) | $\to$ 0 (支持体化) |
| $E_{\text{freq}}^{\text{gold}}$ | $p_{\min} < p < p_{\text{sat}}$ | 中 | 最大 |
| $E_{\text{freq}}^{\text{rare}}$ | $p < p_{\min}$ | 高 | 推定不能 |

##### 7.7.5b.2 新仮説 H₃''

> **H₃'' ($E_{\text{freq}}$ の非単調性仮説).** task performance に対する $E_{\text{freq}}$ の効果は単調ではなく非単調である。最適点は飽和帯ではなく **ゴールディロックス帯** にある。すなわち、
> $$
> \arg\max_{p}\, \mathbb{E}[Q \mid E_{\text{freq}} = p, C \text{ 固定}] \in (p_{\min},\, p_{\text{sat}})
> $$
> [推定 65%]

H₃'' は Adam's Law を棄却しない。Adam's Law (Lu et al. 2026) の実験で観測された頻度-性能の正相関は、coverage が中-高頻度帯 ($p \in [0.4\%, 50\%)$ 程度) に集中していたため、ゴールディロックス効果の **片側上り坂** を捉えていたという読みである。飽和帯 ($p > p_{\text{sat}}$、例: 超頻出機能語のみで構成された prompt) と希少帯 ($p < p_{\min}$、hapax legomenon に近い表現) の両端を含めた測定は、単調仮説では予測できない反転を示すはずである。

##### 7.7.5b.3 化学元素分布との同型

[SOURCE: chemistry_cognition.md §10.5] が示す化学元素存在比と CCL token 分布の同型は、自然言語にも拡張される:

| 領域 | 飽和帯 | ゴールディロックス帯 | 希少帯 |
|:---|:---|:---|:---|
| 宇宙化学 | H (73.9%) | C, N, O (0.1-1%) | 希土類 (<0.01%) |
| CCL token | COMPOSE (49.3%) | CTRL+FUSE 組合せ (2.9%) | ERR (0.1%) |
| 自然言語 | 機能語 ("the", "of") | 中頻度内容語 | hapax legomenon |
| プロンプト | $E_{\text{freq}}^{\text{sat}}$ 入口 | $E_{\text{freq}}^{\text{gold}}$ 制御面 | $E_{\text{freq}}^{\text{rare}}$ 専門技術用語 |

この同型は、論文 I §5.9 の射密度 $R(X) = |\mathrm{Hom}(X, -)|$ が高すぎる対象に対して **両側閾値** を要求する含意 (定理 5.9.3 の裏面) と接続する: 普遍的構造は ker(G) には属さないが、enrich → 1 の代償を払う。「最後の生存者は最も価値の少ない生存者である」というパラドックスが、言語表現面では「最頻出語は摩擦最低だが意味伝達ゼロ」として現れる。三領域 (化学元素 / CCL token / 自然言語) を貫く同型の双方向同期は [SOURCE: chemistry_cognition.md §10.8] で確立されている。

##### 7.7.5b.4 実験設計

H₃'' の検定は §10 検定 debt に **三条件比較** として追加する:

- 条件 sat: prompt 構成語彙を $p > 50\%$ の超頻出機能語のみに制限 (内容語を $p \in [10\%, 50\%]$ から抜粋)
- 条件 gold: $p \in [0.4\%, 5\%]$ の中頻度語で構成 (top-k 官能基相当)
- 条件 rare: $p < 0.1\%$ の超低頻度語 (専門用語・新造語) で構成

各条件で同一 $C$ を符号化し、$Q$ (5 次元盲検評価) と $\mathrm{Var}[Q]$ と MB 透過性 proxy ($\hat{\kappa}$) を測定する。予測は $\mathbb{E}[Q_{\text{gold}}] > \mathbb{E}[Q_{\text{sat}}] \gtrsim \mathbb{E}[Q_{\text{rare}}]$ (gold が単独最大、sat は支持体として中位、rare は推定不能による下位)。

##### 7.7.5b.5 H₃' (分散仮説) への副次効果

H₃' (§7.6.2c-d) は $E_{\text{struct}}$ がテンプレート化操作の出力分散を縮小すると述べた。定理 2' を $E_{\text{freq}}$ に適用すると、分散縮小も飽和帯では起きない:

$$
\mathrm{Var}[Q \mid E_{\text{freq}}^{\text{sat}}] \approx \mathrm{Var}[Q \mid \text{baseline}]
$$

なぜなら飽和帯では prompt が支持体として透過するだけで内部の precision 配分を動かさないためである。分散縮小 (precision 集中) が観測されるのはゴールディロックス帯のみという予測になる。これは H₃' を強化する副次的な仮説である [推定 55%]。

### 7.8 認知スキル (SKILL.md) 設計への含意

H₃ は HGK の認知スキル設計に含意を持つ。ただし「不要」と「効かない」を混同してはならない。

#### 7.8.1 直接効果: $C$ 軸 (推論品質 $Q$ に作用)

| 要素 | 機能 | 文献的裏付け | 効果経路 |
|:-----|:-----|:-----------|:--------|
| Phase の分解と順序 | タスク構造化。$\pi_s$ を最も強く制御 | Cognitive Prompting [4]: +4-13.5% | $C$ → $\pi_s$ → $Q$ |
| Phase 内アルゴリズム | 具体的操作手順 | Weber [26]: 手順明示で分散↓ | $C$ → $\pi_s$ → $Q$ |
| アンチパターン | 負の制約。不正解領域封鎖 | In-context 例と同等効果 | $C$ → $\pi_s$ → $Q$ |
| 出力フォーマット | logit 分布を制約 | Sclar [25]: フォーマット依存 ±10pt | $C$ → $\pi_s$ → $Q$ |

| ゼロトラスト検証 | 自己検証の強制 | Self-Explanation Effect (Chi 1989) | $C$ → $\pi_s$ → $Q$ |

#### 7.8.2 間接効果: $E$ 軸 (推論品質に直接影響しないが、SKILL 体系に貢献)

|:-----|:---------|:-------------------|:--------|
| 座標記法 (I×E) | 「外部入力なし」 | なし (Exp0: d≈0) | **記述効率**: 6語→3記号。**整合性検証**: 4象限の排他性を形式的にチェック可能 |
| N⊣U 合成表記 | 「見落とし→回復」 | なし | **理論的透明性**: 忘却-回復の随伴構造を明示し、Phase 間の因果関係を可読にする |
| ρ 剰余記法 | 「収穫」 | なし | **定量化可能性**: ρ≈0 を検出条件にできる。NL「収穫がない」は定量化困難 |
| Fix(G∘F) 判定 | 「蒸留→展開→不変なら到達」 | なし | **操作的定義**: 「完了」を主観ではなく不動点条件で定義。自動検証の基盤 |

**核心**: $E$ は $Q$ を直接上げないが、$C$ を**設計・保守・検証する**能力を上げる。$E$ は LLM の推論に効くのではなく、SKILL 設計者 (人間) の認知に効く。

ただしこの表は $E_{\text{struct}}$ の効能を扱う。§7.7.5 の $E_{\text{freq}}$ は別であり、SKILL 設計者の認知補助ではなく、モデルが $C_{\text{intended}}$ を $C_{\text{eff}}$ として回収する摩擦に作用する。したがって実務上は「構造記法を増やす」より先に、同じ制約を高頻度で自然な表現へ落とす工程が必要になる。短く言えば、**common wording first, formal control second** である。

#### 7.8.3 境界領域: $E$ が $C$ を兼ねる場合

構造記法が純粋な $E$ ではなく $C$ としても機能する可能性のある領域:

1. **NL で記述困難な制約**: 「全 Phase の ρ を $K_6$ グラフ上で交差検証せよ」は NL 化で情報損失を伴う → $E$ の除去が $C$ の損失を招く
2. **暗黙的行動誘導**: 構造記法の使用自体が「構造的に思考せよ」という L3 制約として機能する可能性 → Yin et al. [27] の日本語文体効果と同型の現象
3. **Self-Explanation 誘導**: 構造記法は LLM に出力の各部分を説明させることを暗黙に要求する → Chi (1989) の Self-Explanation Effect と合流
4. **出力分散への効果** (§7.6.2b 反証条件 (3)): H₃' (§7.6.2b 直後) で**部分的に確認**。bond_count 分散比 F=4.93 (p<0.05)。axiom_count は逆方向 — precision の**再配分**が起きている。$E$ は精度の総量ではなく配分を変える。

これらの境界ケースのうち (4) は H₃' で実験的に支持されつつある。(1)-(3) は §7.3 のアブレーション設計で分離が必要。

#### 7.8.4 HGK の演繹的最善系への含意

HGK の設計思想は「偶然ではなく必然に依拠した認知操作体系」である。H₃ + H₃' はこの思想に 3 層の根拠を与える:

> **SKILL の 3 つの効果経路:**
>
> 1. **$C$ → $\pi_s$ → $\mathbb{E}[Q]$**: 制約が推論品質の期待値を決める (直接効果)
>    Phase 構造・アルゴリズム・アンチパターン・検証基準が LLM の生成モデル空間を精密に制約する。
>
> 2. **$E$ → $\pi_v$ → $\text{Var}[Q]$**: 符号化が推論品質の精度 (ばらつき) を制御する (精度効果 — H₃')
>    構造記法がテンプレート化する操作 (bond analysis 等) の出力分散を減少させる。
>    ただし precision は有限資源 — テンプレート化と競合する操作の分散は増大しうる (precision の再配分)。
>
> 3. **$E$ → 人間 → $C$ の品質**: 符号化が SKILL 設計者の認知を支援する (間接効果)
>    記述効率 / 形式的検証 / 表現力の拡張 / 理論的透明性。
>
> 結論: 「記法は効かない」は不正確。「記法は平均品質ではなく**精度と設計性**に効く」が正確な記述。$E$ は LLM の推論の期待値を変えないが (H₃)、精度 (分散の逆数) を制御し (H₃')、人間の設計能力を支援する。

#### 7.8.5 公理 H-2Z — 二層プロンプト最適化

§7.7.5b の H₃'' (非単調性仮説) と §7.8.1–7.8.4 の H₃/H₃' の含意を統合すると、プロンプト設計に **両側閾値を持つ二層分解** が要請される。本節はこれを公理として正規化する。

##### 7.8.5.1 公理 H-2Z (Two-Zone Optimization)

> **公理 H-2Z.** プロンプト $P$ は二つの符号化層の合成として最適化される:
> $$P = E_{\text{entry}} \circ E_{\text{control}}$$
>
> ここで、
>
> - $E_{\text{entry}}$ は **飽和帯選択**: $p_{\text{entry}} > p_{\text{sat}}$ — MB 透過性 $\kappa$ を最大化する支持体層
> - $E_{\text{control}}$ は **ゴールディロックス帯選択**: $p_{\min} < p_{\text{control}} < p_{\text{sat}}$ — 弁別力を最大化する制御層
>
> 単層構成 ($E_{\text{entry}}$ のみ・$E_{\text{control}}$ のみ) は **両端で劣化** する。二層構成のみが両側閾値の中間最適点を実現する。

これは概要 line 5 「common wording first, formal control second」を、[推定] レベルの実践含意から [推定 60%] レベルの公理候補に格上げする。$E_{\text{entry}}$ の役割は意味伝達ではなく、$E_{\text{control}}$ がモデル境界 (MB) を **通過するための摩擦低下** にある。$E_{\text{control}}$ の役割は摩擦低下ではなく、通過した後の **弁別的制御** にある。

##### 7.8.5.2 化学プログラムとの同型

[SOURCE: chemistry_cognition.md §10.5, §8.7] が示す **2層特徴量アーキテクチャ** (示性式 = 官能基ベクトル + 分子式 = 元素ベクトル) と H-2Z は同一構造である。化学では H/He などの飽和帯元素が分子全体の支持体 (体積・質量) を担い、C/N/O などのゴールディロックス帯元素が官能基として弁別的反応性を担う。プロンプトでも、機能語が支持体・中頻度内容語が官能基として機能する。

| 軸 | 化学 | プロンプト (本稿) |
|:---|:---|:---|
| 飽和帯 | 元素 (H, He) | $E_{\text{entry}}$ (機能語) |
| ゴールディロックス帯 | 官能基 (C-O, N-H 等) | $E_{\text{control}}$ (中頻度内容語・構造記法) |
| 結合 | 共有結合・水素結合 | 構文 ($\circ$) |
| 反応性 | 官能基間相互作用 | $C_{\text{intended}} \to C_{\text{eff}}$ の変換精度 |

この同型は比喩ではなく、両者が同じ離散確率空間上のゴールディロックス定理 ([SOURCE: chemistry_cognition.md §10.6 line 1046-1057]) のインスタンスであることに由来する。

##### 7.8.5.3 H-2Z の操作的含意

公理 H-2Z は SKILL 設計に三つの操作的指針を与える:

1. **入口の自然化**: SKILL.md の `<:role:` `<:goal:` `<:constraints:` 等の宣言部は、構造記法ではなく自然な日本語/英語 ($E_{\text{entry}}$) で書く。これは構造記法の「効きの強さ」が低いからではなく、入口の MB 透過性を最大化するためである。
2. **制御面の構造化**: Phase 分解・アルゴリズム・アンチパターン・検証基準 ($C$ 軸) は構造記法 ($E_{\text{control}}$) で固定する。これはゴールディロックス帯の弁別力を活用するためである。
3. **層境界の明示**: 入口層と制御層の境界を物理的に分ける (例: SKILL.md frontmatter vs body)。混在は両層の効果を相殺する可能性がある。

##### 7.8.5.4 H-2Z の撤回条件

公理 H-2Z は以下のいずれかが成立する場合に撤回または射程縮小される:

- (a) §7.7.5b.4 の三条件比較で $\mathbb{E}[Q_{\text{gold}}] > \mathbb{E}[Q_{\text{sat}}]$ が観測されない場合、H₃'' が崩れて H-2Z の二層分解の根拠が失われる
- (b) 単層 prompt ($E_{\text{entry}}$ のみまたは $E_{\text{control}}$ のみ) が二層 prompt を上回る性能を示した場合、H-2Z の合成最適性が崩れる
- (c) $p_{\min}, p_{\text{sat}}$ の閾値がモデル間・タスク間で大きく変動し、固定された帯域として記述できないことが判明した場合、H-2Z は局所的経験則に降格する

[確信度] H-2Z 公理: [推定 60%]。理論的根拠 (定理 2' + H₃ + 化学同型) は強いが、二層分解の合成最適性自体は §7.7.5b.4 の三条件比較を通る前は経験的支持を欠く。

##### 7.8.5.5 公理 H-2Z と論文 IX (CPS-entropy) との接続予告

論文 IX の指数族支持体論 (CPS の確定的射が CPS-entropy のゼロ点を担う) は、H-2Z の $E_{\text{entry}}$ 層と同型構造を持つ可能性がある。指数族の十分統計量が「分布全体の支持体」として機能するように、$E_{\text{entry}}$ は意味分布全体の支持体として機能する。この接続は本稿 §10 で予告として置き、論文 IX 改訂時に正式に論じる。

| | Gemini (高 compliance) | Claude (低 compliance) |
|:---|:---|:---|
| 構造語彙 ($E$ 軸) | d=8.62 | d=1.5-3 |
| 内容判断 ($C$ 軸) | **d≈0** | **d≈0** |

$C$ 帰無は compliance 特性に依存しない model-invariant な性質である。これは H₃ の射程を「特定モデルでの経験的帰無」から「LLM 一般の構造的性質」に格上げする根拠となる。

### 7.10 MB 透過性と compliance coefficient

#### 7.10.1 なぜ effect size がモデル間で 3-6 倍も異なるのか

Claude と Gemini は同じ $E$ (CCL 記法) を受け取りながら、構造語彙の $d$ が大きく異なる。これは $E$ の「効き」の違いではなく、**モデルの MB (Markov Blanket) 透過性** の違いである。ここでも使っているのは FEP を親理論とする断定ではなく、変分自由エネルギー語彙のうち MB/precision という局所語彙である。

FEP 語彙では:

$$\text{output} = \frac{\pi_s \cdot \text{input} + \pi_p \cdot \text{prior}}{\pi_s + \pi_p}$$

- $\pi_s$ = 入力 (prompt) の精度
- $\pi_p$ = モデルの事前分布 (prior) の精度

**Gemini**: $\pi_p$ が低い → 入力が出力をほぼ支配 → output $\approx$ f(input)
**Claude**: $\pi_p$ が高い → prior と入力が融合 → output $\approx$ g(prior, input)

これは忘却論の用語では:

$$\text{MB 透過性} = U_{\text{prior}} \text{ の深さ}$$

Gemini は自己の prior をより深く忘れて入力に明け渡す ($U_{\text{prior}}$ が深い)。Claude は prior を保持しつつ入力を変換する ($U_{\text{prior}}$ が浅い)。

#### 7.10.2 compliance coefficient の導入

構造語彙の effect size を分解する:

$$d_{\text{struct}} = \kappa_{\text{model}} \cdot \delta_{\text{notation}}$$

- $\kappa_{\text{model}}$ = **compliance coefficient** (モデル固有の MB 透過性パラメータ)
- $\delta_{\text{notation}}$ = 記法の構造的密度 (条件 A/B に固有)

$\delta_{\text{notation}}$ は両モデルで同一 (同じプロンプトを使用)。したがって:

$$\frac{d_{\text{Gemini}}}{d_{\text{Claude}}} = \frac{\kappa_{\text{Gemini}}}{\kappa_{\text{Claude}}} \approx \frac{8.62}{2.5} \approx 3.4$$

Gemini の MB 透過性は Claude の約 3.4 倍。

**ただし重要な限定**: ここで導入した $\kappa_{\text{model}}$ は、モデルに固有の絶対定数ではない。正確には、

$$\hat{\kappa}_{\text{model};\delta,\mathcal{M}} := \frac{d_{\text{struct}}}{\delta_{\text{notation}}}$$

という、**固定された記法対 $\delta$ と固定された測定族 $\mathcal{M}$ の下での局所推定量**である。本稿での $\mathcal{M}$ は構造語彙指標群であり、$\delta$ は plain/CCL の 1 組に固定されている。したがって「Gemini の $\kappa$ は Claude の 3.4 倍」という文は、

> **この記法対・この指標族・このタスク群に関しては** 3.4 倍

という意味に限って成立する。

この限定を書かないと、$\kappa$ は「モデルの本質的性格」の名付けに見えてしまう。しかし本稿が本当に言いたいのはそこではない。必要なのは、**effect size そのものを記法の本質と読み違えないための補正係数**である。

#### 7.10.3 MB 透過性の含意

1. **d は「構造伝搬の強度」を測っていない** — $\kappa_{\text{model}}$ と $\delta_{\text{notation}}$ の積を測っている。$\kappa$ が不明な状態で $d$ を報告しても、$\delta$ の推定にはならない。
2. **H₃ の真の強さ**: $\kappa$ が 3.4 倍異なっても内容帰無が成立する → $C$ 帰無は $\kappa$ に依存しない (= model-invariant)。
3. **SKILL 設計含意の修正**: 「記法は MB 透過性の調整にのみ効く」— 高 $\kappa$ モデル (Gemini) では記法が出力を支配し、低 $\kappa$ モデル (Claude) では記法の影響が prior で緩衝される。どちらの場合も推論品質は変わらない。
4. **「馬鹿正直」問題**: 高 $\kappa$ モデルは入力に忠実であるがゆえに、入力の**ノイズにも忠実**。prompt injection に対する脆弱性は $\kappa$ に比例する可能性がある。

#### 7.10.4 $\kappa$ 仮説の撤回条件

$\kappa$ を補助仮説として維持するための最低条件は、次の 3 つである。

1. **第 3 モデル条件**: 第 3 のモデルを追加したときも、構造語彙の effect size 差が「高 compliance / 低 compliance」という順序と整合すること。
2. **測定族安定性条件**: $\mathcal{M}$ を変えても $\hat{\kappa}$ の順序が完全には崩れず、少なくとも構造指標群で同方向を保つこと。
3. **脆弱性連動条件**: 高 $\hat{\kappa}$ が prompt injection や underspecification への感受性と無関係ではなく、境界透過性という解釈に見合う振る舞いを示すこと。

逆に言えば、これらのどれかが壊れた時点で、本稿は $\kappa$ を「MB 透過性の一般パラメータ」ではなく、単なる**effect size 差の局所的記述子**へ降格させるべきである。ここで守るべきなのは C4 の挑発ではなく、C4 の撤回条件である。

#### 7.10.5 第 3 モデル条件の実験設計

第 3 モデル条件は、「モデルを 1 つ増やす」だけでは満たされない。もし compliance の順序を同じ prompt 実験の effect size から**事後的に**読めば、$\kappa$ は再び循環に戻る。したがって必要なのは、**構造 effect size と独立な compliance 校正面**を先に置いた三者比較である。

実験面は 2 層に分ける。

1. **主実験面**: 既存の T1-T5、A/B (plain/CCL)、測定族 $\mathcal{M}$、N/条件を固定し、各モデルの $d_{\text{struct}}$ と内容指標の帰無を再測定する。
2. **校正面**: 別のミニベンチで exact-format obedience を測る。たとえば JSON key 順序保持、指定ラベルの厳守、禁止自由文の不出力など、**構造語彙 d とは別の従属性指標** $c_{\mathrm{fmt}}$ を事前登録する。

このとき第 3 モデル条件の pass 基準は、

$$\operatorname{ord}(c_{\mathrm{fmt}}) = \operatorname{ord}(d_{\text{struct}}^{\ast})$$

が成り立ち、しかも内容指標では

$$|d_{\text{content}}| \approx 0$$

が維持されることである。ここで $d_{\text{struct}}^{\ast}$ は `total_structural` を主指標、`rho_notation / U_labels / categorical_vocab` を副指標とする事前登録済みパネルである。

この設計なら、M3 が Claude と Gemini の間に入るか、あるいはどちらかの外側へ出るかにかかわらず、$\kappa$ は「記法差の比率」ではなく「独立校正された従属性と整合する局所パラメータ」へ一段だけ昇格できる。逆に、M3 が内容帰無を破るか、$c_{\mathrm{fmt}}$ と $d_{\text{struct}}^{\ast}$ の順序が食い違うなら、$\kappa$ はここで止めるべきである。運用上の詳細プロトコルは補助計画書へ分離した。

## §8. 結語

本稿が示したのは、プロンプトを「情報追加の技法」として見るより、「どの prior を忘却させるかを選ぶ操作」として見た方が、理論面でも実験面でも一段よく閉じるということである。C1 はこの読み替えそのものを与え、C2 は prompt の内部にある制約 $C$ と符号化 $E$ の役割分担を切り分け、C3 は「記法は効くのか」という雑な問いを「平均品質ではなく分散構造と設計可能性にどう効くか」という精密な問いへ変換した。Adam's Law [30] との接続により、この $E$ はさらに $E_{\text{struct}}$ と $E_{\text{freq}}$ に分かれる: 前者は語彙空間と検証可能性を動かし、後者は $C_{\text{intended}} \to C_{\text{eff}}$ の摩擦を下げる。C4 はさらに、同じ $E$ を入れてもモデルごとに出方が違う理由を MB 透過性 $\kappa$ という局所量で読む仮説面を与えた。C5 は $E_{\text{freq}}$ を Chemistry-Cognition Program の定理 2' (ゴールディロックス) で両側有界化し、最適プロンプトを $P = E_{\text{entry}} \circ E_{\text{control}}$ の二層分解 (公理 H-2Z) として正規化した。ここでの理論順序は、Helmholtz 的な物理直観を直接 imported することではなく、変分自由エネルギーの語彙を経由して FEP の局所概念を prompt 設計へ移すことである。

より重要なのは、ここで得られた中心結果が「効いた」ではなく、むしろ**条件を固定すると効かないものがある**という帰無の構造である。$C$ を固定したとき、$E_{\text{struct}}$ は平均品質を押し上げる万能レバーではなかった。だがそれは「記法は無意味だ」という帰結ではない。正しい読みは、「記法は別の層に効く」である。Liu & Chu (2026) が示すトークン分布層、Chen et al. (2026) が示す task-type 層、Lu et al. (2026) が示す表現頻度層、そして本稿の出力分散 proxy の層は、いずれもこのずれを別の座標で支持している。さらに C5 によって、$E_{\text{freq}}$ そのものに **第二の帰無** が立つ: 飽和帯では摩擦は最低だが意味伝達もゼロという、頻度を上げるほど効くという単調仮説の自然な反例である。これも「効かない」ではなく「効くべき層が違う」(支持体は摩擦に効き、官能基は弁別に効く) である。

したがって実践的含意も単純である。**磨くべき順序は、制約、頻度、記法である。** まず何を守らせたいかを決める。次に、それをモデルが回収しやすい高頻度自然語で通す — ただし **超頻出機能語のみの支持体に押し進めず、ゴールディロックス帯の中頻度内容語を併用する**。最後に、構造記法で検査面と制御面を固定する。記法は語彙空間の整流、検査面の維持、モデル境界での透過性調整として使うべきである。本稿はここまでを主張する。逆に、内部 precision と出力分散の一意橋渡し、$\kappa$ の 3+ モデル較正、公理 H-2Z の合成最適性 (§10.8 三条件比較で検定予定)、prompt engineering 全体への一般化は、まだ本稿の外に残してある。ここで止めることで、野望を弱めるのではなく、どこまでが本稿の実でどこからが次稿の虚かを明示する。

### 10.6 観測としてのプロンプト — Face Lemma CSP 接続

§1 のテーゼ「プロンプトは選択的忘却操作」を、CCL 検索の CSP 問題 (`chemistry_cognition.md#chemistry_of_ccl_features` §9, §13) と接続する。

**同型構造**:

| CCL 検索 | プロンプト | 構造 |
|:---------|:---------|:-----|
| U_count (49d化) | P₁ (自然言語) | 粗い忘却 — 多くの構造を潰す |
| U_cos (cosine) | P₂ (構造記法) | さらなる忘却 — 角度のみ保存 |
| ker(U_cos) ⊇ 94% | ∂Q/∂E ≈ 0 (H₃) | 忘却が支配的 — 微調整は表面的 |
| 1-skeleton (2射) = 不安定 | C のみ = 制約は効くが多義的 | dim Ξ = 1 — 方向なし |
| 2-skeleton (3射) = 安定 | C + E + **context** = 3軸 | dim Ξ = 2 — 形がある |

**Face Lemma の SKILL 設計への含意**:

Face Lemma (Paper II §3.4): 概念安定性には最低3つの独立な生成射が必要。

符号理論的に言えば、C・E・context の三軸は三つの check node ではなく、**syndrome を立てる最小面**である。C と E だけでは「何かがずれた」ことは感じ取れても、そのズレが instruction 由来か、encoding 由来か、共有文脈の欠落由来かを局在化できない。context が入ることで 2-simplex が立ち、欠損が detectability を持つ。複数 skill の相互補完を許す設計は、この局所 face を多数貼り合わせた LDPC 的な修復網として理解できる。用語整理は `drafts/incubator/FaceLemma_技術設計.md`。

Paper III の言葉で言えば、P₂ は単なる装飾ではなく「どの α-セクターで構造を再利用するか」を偏らせる局所記法である。共有コンテキストが欠けるとこの偏りは 1-skeleton の段階で失効しうるが、context が加わると 2-skeleton が立ち、Z₂ 的な極性差が安定な構造語彙として残る。したがって XI は I・IV だけでなく、III の α-セクター / Z₂-次数の議論にも接続される。

SKILL.md は本質的に3つの独立な制約軸を持つべき:
1. **C 軸 (制約)**: Phase の分解・手順・検証基準 → $\pi_s$ 制御
2. **E 軸 (符号化)**: 座標構文・CCL 記法 → $\pi_v$ 制御
3. **M 軸 (メタ認知)**: アンチパターン・自己参照・Kalon 基準 → 再帰的安定化

C のみ = 1-skeleton (不安定)。C + E = 2射 (dim Ξ = 1、方向なし)。
**C + E + M = 2-skeleton** (dim Ξ = 2、概念が安定)。

H₃ の制約-符号化分離 (§7.6) は、E が Q に直接影響しないことを示した。
しかし Face Lemma の観点では、E は Q の「安定性」に影響する:
E が欠落すると dim Ξ が低下し、同じ C を与えても出力の分散が増大する。
この増大は、符号理論では「syndrome が立たないまま誤差が漂う」状態に近い。ゆえに E の役割は内容注入ではなく、detectability を支える検査面の維持にある。

これは §7.7.2 の不特定性 (underspecification) と整合する:
Weber et al. (2026) が示した「ミニマルプロンプトの高分散」= dim Ξ の不足。

**実験的予測 (P41)**:
SKILL.md の C/E/M 3軸を体系的に除去するアブレーションで:
- C 除去: Q が大きく低下 (§7.8.1 と一致)
- E 除去: Q の平均は不変だが**分散が増加** (H₃ の精密化)
- M 除去: Q の平均はやや低下し、**長期整合性が崩壊** (自己参照の喪失)

### 10.7 H₃'' 三条件比較実験計画 ($E_{\text{freq}}$ 非単調性)

§7.7.5b で立てた H₃'' (非単調性仮説) の経験的検定。Adam's Law (Lu et al. 2026) の単調仮定を破る方向に動くため、反証可能性を確保した事前登録設計が必須である。詳細プロトコルは `drafts/infra/experiments/論文XI_H3pp_goldilocks_three_condition_plan.md` を正本とする。本節は本文 reader 向けに要点のみ示す。

**仮説**:

- H₃''-main: $\arg\max_p \mathbb{E}[Q \mid E_{\text{freq}} = p, C \text{ 固定}] \in (p_{\min}, p_{\text{sat}})$
- H₃''-pred: $\mathbb{E}[Q_{\text{gold}}] > \mathbb{E}[Q_{\text{sat}}] \gtrsim \mathbb{E}[Q_{\text{rare}}]$

**三条件**:

| 条件 | corpus 頻度 $p$ | 構成方針 | 例 |
|:---|:---|:---|:---|
| sat | $p > 50\%$ | 超頻出機能語のみ | "the system, the input, the output, do the task..." |
| gold | $p \in [0.4\%, 5\%]$ | 中頻度内容語 / top-k 官能基相当 | 専門領域の中堅語彙、CCL 制御語 |
| rare | $p < 0.1\%$ | 超低頻度語・新造語・hapax 寄り | 専門技術用語、合成語、neologism |

**測定**:

- $Q$: 5 次元盲検評価 (Opus, §7.5 と同設計)
- $\mathrm{Var}[Q]$: 出力分散
- $\hat{\kappa}$: MB 透過性 proxy (§7.10.2)
- $d(C_{\text{intended}}, C_{\text{eff}})$: 制約回収摩擦

**N**: 50/条件、5 トピック ×3 条件 = 750 サンプル。Claude Sonnet 4.6 + Gemini 3.1 Pro Preview の両モデル。

**事前登録 equivalence margin**:

- H₃'' 採択条件: Cohen's $d(\text{gold} - \text{sat}) > 0.4$ かつ $d(\text{gold} - \text{rare}) > 0.4$ (両方向で gold が単独最大)
- H₃'' 棄却条件: $d(\text{gold} - \text{sat}) \in [-0.2, 0.2]$ (TOST 同等性) — 単調 Adam's Law 仮説に降格

**撤回条件 (§7.8.5.4 参照)**:

(a) gold が sat を上回らない場合、H₃'' 棄却 → 公理 H-2Z の根拠喪失。

### 10.8 H-2Z 二層 vs 単層比較実験計画

§7.8.5 で立てた公理 H-2Z の **合成最適性** を検定する。詳細プロトコルは `drafts/infra/experiments/論文XI_H2Z_two_layer_vs_single_plan.md` を正本とする。本節は要点のみ。

**仮説**: $\mathbb{E}[Q \mid P = E_{\text{entry}} \circ E_{\text{control}}] > \max(\mathbb{E}[Q \mid P = E_{\text{entry}}],\, \mathbb{E}[Q \mid P = E_{\text{control}}])$

**四条件**:

| 条件 | 構成 | 例 |
|:---|:---|:---|
| Z-entry | $E_{\text{entry}}$ のみ (機能語) | "Please do the task..." |
| Z-control | $E_{\text{control}}$ のみ (中頻度+構造記法) | CCL 記法のみで構成 |
| Z-2layer | $E_{\text{entry}} \circ E_{\text{control}}$ | 自然語 frontmatter + 構造記法 body (現行 SKILL.md 形式) |
| Z-mix | 入口層と制御層を混在 | 自然語と構造記法を段落内で交互に |

**N**: 30/条件、5 トピック = 600 サンプル。

**事前登録条件**:

- H-2Z 採択条件: Z-2layer が Z-entry, Z-control, Z-mix の **全て** を Cohen's $d > 0.3$ で上回る
- H-2Z 棄却条件 (b): Z-entry または Z-control が Z-2layer 以上の性能を示す → §7.8.5.4 撤回条件 (b) 発動 → 二層分解は経験的経験則に降格 (Lax 化)
- H-2Z 棄却条件 (c): $p_{\min}, p_{\text{sat}}$ の閾値が Claude/Gemini で 2 倍以上ずれる → §7.8.5.4 撤回条件 (c) 発動 → モデル不変性主張を撤回

**Z-mix が想定外に勝った場合**: 公理 H-2Z の「層境界明示」(§7.8.5.3 指針 3) が逆に有害である可能性 → Lax 化のうえで「混在配置の方が有利」を新仮説として再起票。
