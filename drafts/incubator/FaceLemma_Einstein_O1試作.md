# Face Lemma × Einstein — O1 型付け義務の局所 closure 試作

作成日: 2026-04-14  
役割: Paper XIII の `O1: 型付け義務` に対する単独メモ。  
目的: `Face Lemma の 3射` と `GR の接続/計量/方向データ` の関係を、**似ている**から一歩進めて、少なくとも「何を何として対応させるのか」が読める状態にする。  
位置づけ: 局所 closure。`O2 defect`, `O3 closure`, `O4 coupling` は未着手。

## 0. 一文核

`Face Lemma` の 3射は、GR では「三つの独立な物理場」に対応するのではない。  
対応するのは、**方向を選ぶ射 / その方向を計量で比較する射 / その比較が座標変換に耐えるよう保証する接続射** という、三つの**検証役割**である。

この読み替えなら、`第三射は前二者から自明に消えない最小追加情報である` という `O1` の要求に、少なくとも reader-facing には応答できる。

## 1. 何が O1 の障害だったか

Paper XIII の現行草稿では、`D1` の候補として

- `f = ∂/∂x^μ`
- `g = g_{μν}`
- `h = Γ^λ_{μν}`

が置かれている。

このままだと違和感が二つ出る。

1. `f` はベクトル場、`g` は双線形形式、`h` は接続係数であり、**同じ型のものを三つ並べているわけではない**
2. `Γ` は Levi-Civita 条件のもとで `g` から導出されるので、`第三射が独立な追加情報` という Face Lemma の直感と衝突して見える

この衝突を放置すると、`Face Lemma ↔ Einstein` は「3つあるから似ている」という resemblance に落ちる。

したがって O1 の局所 closure は、三射を「三つの field」として数えるのでなく、**三つの役割**として型付け直すところから始める必要がある。

## 3. O1 の局所命題

### O1-local

GR インスタンスにおいて `Face Lemma` の 3射は、次のような**型付き verification triple** として読むのが最も自然である。

1. **方向射 `f`**  
   局所事象 `p` において、接方向を選ぶ射。  
   直感的には `T_p M` からの「どちらへ進むか」の選択。

2. **計量比較射 `g`**  
   選ばれた方向を、容器側のものさし `g_{μν}` で比較可能にする射。  
   直感的には「その方向がどの長さ・角度・因果型を持つか」を測る役割。

3. **接続射 `h`**  
   上の比較が座標系や局所 patch をまたいでも意味を持つようにする整合射。  
   局所座標では `Γ^λ_{μν}` がその担い手になる。

このとき `h` は「metric とは別の ontic field」ではなく、**metric で定義された比較を、局所的に輸送可能な比較へ持ち上げる役割** を持つ。

---

## 4. なぜこれで第三射が自明に消えないのか

ここが局所 closure の核心である。

Levi-Civita 接続は確かに `g` から導出される。  
しかし、Face Lemma が要求しているのは「三つの量が ontologically 独立か」ではない。  
要求しているのは、

> **前二者だけでは外から照合できない比較を、第三射が検証可能にするか**

である。

この基準では、`h = Γ` は十分に第三射である。理由は次の通り。

- `f` だけでは方向はあるが、比較規則がない
- `f + g` だけでは局所点での長さや角度は測れても、**隣接点との比較法** がない
- `h` が入ることで初めて、「その比較をどう運ぶか」が定まり、`g ∘ f` の直接比較と `h` による輸送比較を突き合わせられる

したがって `h` の必要性は、「追加の field がある」ことではなく、**比較を可搬化する最小の witness がある**ことにある。

この意味で `Γ` は `Face Lemma` の第三射に十分近い。

## 5. 提案する型付け

型を乱暴に物理量へ貼るのでなく、次のように**役割空間**を挟んで読む。

| Face Lemma 側 | GR 側の担い手 | 役割 |
|:---|:---|:---|
| `f` | 接方向 `∂/∂x^μ` | 局所方向の選択 |
| `g` | 計量 `g_{μν}` | 容器側の比較規則 |
| `h` | 接続 `Γ^λ_{μν}` | 比較規則の輸送・整合 |

ここで重要なのは、

- `f` は「何が動くか」
- `g` は「どう測るか」
- `h` は「その測り方をどう保つか」

という三役であり、同じクラスの object を三つ並べているわけではない、という点である。

これなら `Face Lemma` の「第三射が前二者から自明に消えない」という要求は、

> **測定対象 / 測定規則 / 測定規則の輸送**

の三役が欠けてはならない、という reader-facing 命題に言い換えられる。

## 6. `B_1^ver` との接続

`Face補題_ホモロジー厳密化.md` が強いのは、`Face Lemma` の主役を `B_1^ver = im ∂_2` に置き換えた点である。

この語彙に引き直すと、GR における O1 の局所 closure は次のように読める。

- `f` が方向を与える
- `g` がその方向の比較規則を与える
- `h` がその比較規則の輸送を与える
- この三者が作る 2-simplex の境界が、**局所比較の照合面** を与える

まだここでは曲率そのものは出ていない。  
曲率は次段 `O2` で、この照合面が patch をまたぐときにどう defect を生むかとして読むべきである。

ゆえに O1 の正しい到達点は、

> `Γ` を「曲率の源」としてではなく、「照合面を成立させる第三射」として固定する

である。

## 7. 何が閉じ、何がまだ閉じていないか

### この試作で閉じたこと

1. `f/g/h` を三つの ontic field として数える誤読を避けた
2. `Γ` を第三射として読む理由を、「metric から導出されるのに第三射でよいのか」という反論に対して reader-facing に説明できるようにした
3. `Face Lemma` の 3射最小性を、GR 側で「方向 / 比較 / 輸送」の三役として言い換えた

### まだ閉じていないこと

1. `O2 defect`: 曲率を `B_1^ver` の defect としてどう読むか
2. `O3 closure`: Bianchi 恒等式を syndrome conservation としてどう読むか
3. `O4 coupling`: なぜ Einstein 方程式がその整合条件としてこの形になるか
4. `P2`: 容器/内容の型付けを、この O1 役割空間とどう接続するか

したがって本メモは **local closure** であり、証明ではない。

## 8. Paper XIII へ戻すと何が変わるか

現行の

> `f = ∂/∂x^μ`, `g = g_{μν}`, `h = Γ^λ_{μν}`

という書き方は、そのままだと「違う型を雑に三つ並べた」ように見える。

O1 を通した後の reader-facing な改善案は、次の一文で十分である。

> Face Lemma の 3射は、GR では「方向の選択」「計量による比較」「その比較の輸送を保証する接続」という三つの検証役割として読むのが自然である。ここで第三射は独立な物理場を追加することではなく、比較を可搬化し外から照合可能にする最小の witness を与える。

これなら、`D1` は resemblance の列挙から一歩出る。

## 9. 次の一手

このメモの次にやるべきは二択である。

1. **O2 へ進む**  
   `Γ` を第三射として固定した上で、`R_{\mu\nu}` や `G_{\mu\nu}` を「照合面の defect / 縮約」として読めるかを試す

2. **P2 へ進む**  
   `方向 / 比較 / 輸送` の三役を踏まえた上で、どこまでが容器でどこからが内容かを再型付けする

順序としては `O1 → P2 → O2` でもよいが、直感的には `O1 → O2` の方が流れは自然である。  
理由は、第三射を固定した直後に「その第三射が作る defect は何か」を問う方が、reader-facing に理解しやすいからである。
