## 0. 用法

1. 原稿を書き始めるとき `§3 原稿×誌マトリクス` で候補を絞る。
2. 候補誌の `§4 確定誌一覧` を読み、料金・語数・scope・査読方式を確認する。
3. `§6 投稿手順` で具体的なステップを追う。
4. policy が改訂されていないかを `§8 鮮度管理` で確認する。6 ヶ月を超えたら再裏取り義務。
5. Aeon・Psyche・PhilArchive・Open Mind・TAC は直接裏取り失敗 (§5)。投稿直前に Tolmetes が公式ページを手動確認する。

## 3. 原稿 × 誌マトリクス

「本命 1 / 保険 2 / 公開面 1」の割付原則。本命で reject された場合の保険と、peer review を待たず DOI を確保する公開面を併走させる。

| 類型 | 本命 | 保険 1 | 保険 2 | 公開面 |
|:---|:---|:---|:---|:---|
| A 圏論中核 | Compositionality | TAC | TheoretiCS (計算寄り) | arXiv `math.CT` + Zenodo |
| B 哲学 | Ergo | Philosophers' Imprint ($) | PhilArchive (プレプリント) | PhilArchive + Zenodo |
| C 認知科学 | Open Mind | PsyArXiv (プレプリント) | arXiv `cs.AI` | arXiv + Zenodo |
| D 公共エッセイ | Aeon | Psyche | Noema / The Point | 自前掲載 (Substack 等) |
| E 物理越境 | Foundations of Physics | SciPost Physics / Quantum | Entropy (MDPI) / JCAP / CQG | arXiv `gr-qc``hep-th``quant-ph` + Zenodo |

### 適用例

- **比較射σの統一定理 (類型 A)**: Compositionality が本命、TAC が保険、arXiv `math.CT` で DOI 確保。投稿運用上の正本は `drafts/standalone/比較射σの統一定理_v0.6.md`、Abstract / 版歴 / claim ledger は `drafts/standalone/比較射σの統一定理_v0.6.meta.md` を参照。
- **論文 I 力としての忘却 (類型 B)**: Ergo が本命、Philosophers' Imprint が保険 (予算確保要)、PhilArchive で即時公開。
- **論文 XII 速度は忘却である (類型 C)**: Open Mind が本命、arXiv `cs.AI` + PsyArXiv で二重公開。物理越境部分は arXiv `physics.gen-ph` 分離投稿も検討。
- **バカをやめたいなら構造を見ろ v2 (類型 D)**: Aeon に pitch、通らなければ Psyche / Noema に連続 pitch。並行して Substack での自前公開準備。
- **論文 XIII 時空は忘却である (類型 E)**: Foundations of Physics が本命 (Chief Editor Carlo Rovelli は relational QM・LQG の当事者で scope 完全一致)、SciPost Physics が保険、arXiv `gr-qc` + Zenodo で DOI 確保。
- **論文 XIV 曲率は忘却の繰り上がりである (類型 E)**: Foundations of Physics 本命、CQG (gravity 特化) or Quantum (量子重力) が保険、arXiv `gr-qc` で公開。

## 6. 投稿手順

主要誌の具体的なステップ。

### 6.1 Compositionality (類型 A 本命)

1. 英訳完了した稿を LaTeX で組み、`arxiv-style.cls` に変換 (Compositionality テンプレあり、受理後に組版)
2. arXiv `math.CT` に投稿 (初回は endorsement 必要 — 既投稿者のエンドースを得る)
3. arXiv ID を取得 (例: 2604.xxxxx)
4. EpiSciences アカウント作成: https://compositionality.episciences.org/
5. 投稿フォームで arXiv ID + 対応著者メール + (任意で) 補足ファイル (コード・データ) を登録
6. Consent to Publish (CC-BY 4.0) に署名
7. 査読待機。受理後に編集委員会のテンプレで組版、arXiv 再アップロード

**チェックリスト**:
- [ ] arXiv endorsement 確保
- [ ] 英訳と LaTeX 組版完了
- [ ] arXiv 投稿 → ID 取得
- [ ] EpiSciences 登録
- [ ] CC-BY 4.0 署名

### 6.1b TAC (類型 A 保険 1 / Compositionality の次点)

1. `tac.cls` スタイルファイルを http://tac.mta.ca/tac/styles/ からダウンロード
2. 稿を **LaTeX 2e + tac.cls + 12pt** で組む:
   - `\textwidth`, `\textheight`, 等は改変しない
   - amsthm パッケージは使わない (journal style と衝突)
   - 各行 72 文字以内推奨
   - PdfLaTeX でコンパイルできることを確認
3. Transmitting Editor を 1 名選ぶ (Managing Editor と TeX Editor は除く):
   - Clemens Berger / Julie Bergner / Richard Blute / John Bourke /
     Maria Manuel Clementino / Valeria de Paiva / Richard Garner /
     Ezra Getzler / Kathryn Hess / Rune Haugseng / Dirk Hofmann / Pieter Hofstra 等
   - 選択基準: 稿の内容が近い分野の editor
4. 選んだ Editor に PDF を email 送付 + **tac@mta.ca (Managing Editor) に CC 必須**
5. 受理通知を待つ (Editor からの acknowledgment)
6. 受理後、TeX source + 最終 PDF を tac@mta.ca へ提出
7. 著者は Consent to Publish Agreement に同意

**チェックリスト**:
- [ ] `tac.cls` ダウンロード
- [ ] LaTeX 2e / 12pt / amsthm 不使用 / 標準マージンで組版
- [ ] PdfLaTeX でコンパイル成功
- [ ] Transmitting Editor 選定 (1 名)
- [ ] email 送付 + tac@mta.ca に CC
- [ ] acknowledgment 受領
- [ ] 同時投稿していないことを確認 (1 論文 1 Editor)

⚠️ **サイト注意**: http://tac.mta.ca (HTTPS ではなく HTTP)。HTTPS は現在壊れている。ブラウザで「安全でない接続」警告が出るが正規。

### 6.2 Ergo (類型 B 本命)

1. https://www.ergosubmissions.org でアカウント登録
2. 稿を **double-blind review 用に匿名化**:
   - 著者名・所属・連絡先を全て除去
   - 自著への参照を「[Author]」「[Author 2024]」等に置換
   - ファイルのメタデータ (PDF 作成者) も削除
   - 謝辞セクションを削除 (あれば)
3. 匿名化 PDF を投稿
4. 査読待機

**チェックリスト**:
- [ ] 原稿を匿名化 (本文・参照・メタデータ)
- [ ] Ergo submissions アカウント
- [ ] PDF 投稿
- [ ] 確認メール受領

### 6.3 Philosophers' Imprint (類型 B 保険 1)

1. https://journals.publishing.umich.edu/phimp/submissions/ でガイド確認
2. **投稿料 $50 または $20 を事前支払い** (permanent post なら $50、それ以外は $20)
3. 支払いレシートの screenshot を保存
4. 稿を **.docx 優先 / .pdf (自前 LaTeX 組版する場合のみ)** で準備
5. **double-blind review 用に匿名化**
6. abstract を plain text で proofread
7. 投稿フォームで PDF + レシート screenshot を提出

**チェックリスト**:
- [ ] 投稿料予算確保 ($50 or $20)
- [ ] 支払い完了 + screenshot
- [ ] .docx 形式 (または自前 LaTeX 可能な PDF)
- [ ] 匿名化
- [ ] 語数 10,000 以下 (超える場合は覚悟)
- [ ] abstract 校正済

### 6.4 PsyArXiv (類型 C プレプリント)

1. OSF アカウント登録 (osf.io)
2. 稿を **AI/LLM 使用について明示**:
   - Claude 等を substantive に使った場合、manuscript 内で disclosure
   - 例: 「本稿の構造化は Claude 4.7 との対話を通じて行われた」
3. scope を psychology / cognitive science に接続する文を intro に入れる
4. PsyArXiv 経由で投稿
5. **pre-moderation 待機** (2025-08-19 以降)

**チェックリスト**:
- [ ] OSF アカウント
- [ ] AI disclosure 明記
- [ ] psychology / cognitive science への接続文
- [ ] プレプリント投稿
- [ ] moderation 通過待機

### 6.5 Zenodo (公開土台 / DOI 取得)

1. https://zenodo.org/ でアカウント登録 (ORCID 推奨)
2. 新規 upload: https://zenodo.org/uploads/new
3. content type を選択 (preprint / publication / dataset 等)
4. 稿ファイルをアップロード + メタデータ (タイトル / 著者 / abstract / keywords / ライセンス)
5. Community (例: Philosophy / Cognitive Science) に submit (任意)
6. 公開 → **DOI 即時発行**
7. バージョン更新時は Edit から新版アップロード (旧版も保持される)

**チェックリスト**:
- [ ] Zenodo + ORCID アカウント
- [ ] ライセンス選択 (CC-BY 4.0 推奨)
- [ ] メタデータ記入
- [ ] DOI 取得確認

### 6.6 HAL (公開土台 / 欧州圏対応)

1. https://hal.science/ でアカウント作成
2. 稿 PDF をアップロード
3. **著者所属を最低 1 件**登録 (HAL は所属なしでは受付不可)
4. メタデータ (タイトル / 著者 / abstract / domain / キーワード) を記入
5. **モデレーション待機** (CCSD が手動チェック、数日)
6. 公開 → HAL ID 発行
7. **削除不可**。修正は版更新で。

**チェックリスト**:
- [ ] HAL アカウント
- [ ] 著者所属 (最低 1 件)
- [ ] メタデータ記入
- [ ] モデレーション待機
- [ ] 削除不可を理解

## 7. 公開戦略パターン

### 7.1 二重公開戦略 (プレプリント + 査読誌) — 論文類型 A/B/C/E のみ

- 先に **arXiv / Zenodo / PhilArchive / PsyArXiv** で DOI 付き preprint 公開 → 引用可能な状態を確保
- 並行して本命誌に査読投稿
- 査読で rejected でも preprint は生きる
- 最も時間を味方にする戦略

⚠️ **類型 D (公共エッセイ) には適用不可**: 商業誌 (Aeon / Psyche / Noema / The Point) は多くが **first-publication rights** を要求する (公知 — 各誌の公式要件は `§5 要再確認`)。Zenodo や Substack で先に公開すると査読誌と違って refuse 理由になる。エッセイは `§7.2` の専用戦略に従う。

### 7.2 公共エッセイ戦略 (類型 D) — 投稿先優先 / 事前公開禁止

原則: **先に投稿、公開はその後**。

- 稿を英訳 (Aeon 系列は英文前提)
- 本命 (Aeon) から連投、rejected ラインを確認してから次
- 同時投稿は誌ごとの可否に従う (The Point は可、Aeon/Psyche/Noema は要公式確認)
- 全敗後に Substack / note で自前公開 (DOI が欲しければ Zenodo も可だが、それは「落ちた後」)

順序例:
1. Aeon に pitch (2-4 週間応答待機)
2. Psyche に pitch (Aeon 系列なので Aeon rejected 後が無難)
3. Noema に full draft 送付 (pitch より採用確率高い)
4. The Point に Submittable 経由 (同時投稿可・有償)
5. 全て落ちたら Substack / note で自前公開

### 7.3 Compositionality 連鎖戦略 (類型 A)

- 本命: Compositionality (arXiv overlay)
- reject 時: TAC へ (arXiv を流用、transmitting editor にメール)
- TAC も reject: TheoretiCS (計算理論寄りなら)
- 全敗時: arXiv preprint として残す + Zenodo で DOI 化

### 7.4 物理越境連鎖戦略 (類型 E)

**原則**: 物理誌は scope が狭く速度も遅いので、arXiv で先に公開して「引用できる状態」を確保してから査読誌に連鎖投入する。

1. **arXiv 先行** (必須): `gr-qc` / `hep-th` / `quant-ph` / `physics.gen-ph` のうち該当カテゴリに投稿 (初回は endorsement 必要)
2. **Foundations of Physics** (本命): review/essay 型なら EIC (Carlo Rovelli) に proposal を事前送付 → 反応を見てから本投稿
3. reject 時の分岐 (稿の性質で):
   - 量子基礎寄り → **Quantum** (arXiv ID 流用)
   - 物理全般 breakthrough 型 → **SciPost Physics** (open peer-witnessed refereeing)
   - 情報理論 / FEP 寄り → **Entropy** (MDPI、APC 要)
   - 宇宙論寄り → **JCAP** (SISSA/IOP、arXiv 統合)
   - 重力・時空寄り → **CQG** (IOP、hybrid)
4. 全敗時: arXiv preprint として残す + Zenodo で DOI 化 + Foundations of Physics の essay 枠に再提案

## 9. Tolmetes への初動提案

**原則**: 類型ごとに動線が違う。エッセイは「先に投稿→落ちたら公開」、論文は「先に preprint→並行で査読誌」。混ぜない。

### 9.1 エッセイ動線 (類型 D) — 最速

`バカをやめたいなら構造を見ろ_v2` は既に完成しているので、**即着手可能**。

1. **v2 を英訳**: Aeon 系列は英文前提。翻訳プロセスで既存の論理を再点検できる。
2. **Aeon に pitch**: Aeon は本命。pitch 形式 (1 段落要約 + 骨格 + 著者プロフィール) で提出。応答待機 2-4 週間。
3. **Rejected 時の連投**: §7.2 に従い Psyche → Noema → The Point。
4. **全敗後のみ** Substack / note で自前公開。Zenodo は最後の手段 (DOI が欲しい場合のみ)。

⚠️ **禁止**: Zenodo / note / Substack に先に上げないこと。商業誌の first-publication rights を壊す。

### 9.2 論文動線 (類型 A/B/C) — 並行

論文側は Zenodo / arXiv 公開と査読誌投稿を **同時並行** で進めるのが標準。

1. **arXiv アカウント整備**: 類型 A/C には必須 (Compositionality / TheoretiCS / arXiv 直公開)。
   - 未所持なら作成、必要なら endorsement を既投稿者に相談。
2. **類型 A から 1 本**: 例 `比較射σの統一定理_v0.6`
   - 英訳 + LaTeX 組版 → arXiv `math.CT` 投稿 → Compositionality へ overlay 投稿。
3. **類型 B から 1 本**: 例 `エッセイ_理解と予測の随伴` or `論文I_力としての忘却`
   - 英訳 + 匿名化 → Ergo へ投稿。並行で PhilArchive に preprint 公開。
4. **Zenodo**: 論文系は各誌投稿と並行して DOI 取得しておくと保険になる。
