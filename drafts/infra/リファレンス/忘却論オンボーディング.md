## 1. 一文要約

忘却論は、「忘却とは欠陥ではなく、構造を選別し、残差・境界・力・知覚・存在・エントロピー・プロンプト・速度・曲率を生む操作である」という系列的理論プログラムである。

もう少し操作的に言えば、忘却論は `U⊣N` を読解軸にして、何を失うかではなく、**何を忘れたから何が見えるようになったか** を追跡する。
`U` は忘却・粗視化・溶解・Explore、`N` は回復・結晶化・固化・Exploit である。系列横断では `U⊣N` を標準とし、局所論文で逆順や別記法が出る場合は、その論文内例外として読む。

## 5. 執筆開始前チェック

新規論文の執筆、既存論文の大規模改稿、または incubator seed の昇格前に、以下を確認する。

### 5.1 番号と公開境界

- `plans/出版計画_忘却論シリーズ.md` で番号と公開層を確認する。
- 既存番号に割り込ませない。
- `standalone` から `series` へ昇格する場合は、依存グラフと公開境界を出版計画へ反映する。

### 5.2 記号と主張水準

- `drafts/リファレンス/統一記号表.md` §0.3 で主張水準を確認する。
- 新記号・新定理 ID・依存関係を導入したら、本文だけでなく統一記号表を更新する。
- 記号衝突がありそうなら、`drafts/リファレンス/系列横断_記号衝突台帳_2026-04-26.md` を読む。

主張水準の最小対応は以下。

| ラベル | 使える条件 |
|:--|:--|
| 定理 | 証明が完結している |
| 命題 | 証明はあるが前提に未解決部を含む、または部分的 |
| 補題 | 他の定理・命題を支援する主張 |
| 推定 | 実験データまたは非形式的論証の裏付けがあるが形式的証明が欠ける |
| 仮説 | 理論的根拠はあるが検証不足 |
| 予想 | 方向性はあるが構成が不足 |
| 実験的知見 | 観測データに基づくが理論導出ではない |

### 5.3 既知批判と failure condition

- `drafts/リファレンス/批判反証レジストリ.md` で横断批判 Z 系と該当論文の個別批判を確認する。
- 応答を書くときは、最強版批判、最強応答、残余問題、failure condition を分ける。
- 応答を本文に入れたら、レジストリも更新する。

### 5.4 `.meta.md`

- 既存論文なら同名 `.meta.md` を先に読む。
- 新規論文なら本文より先に `.meta.md` を作る。
- 核主張を置いたら、`§M1 F⊣G 宣言` と `§M6 虚→実変換面` を埋める。
- `.meta.md` は読者向けではなく、Tolmetes とエージェントの共同台帳である。

### 7.2 α bridge

現在の主面は `α_I / α_III / α_VIII / α_eff / α_CPS` の 5 変種である。

- `α_III → α_VIII` は `η`, `η̄`, 架橋関手 `B` で読む。
- `α_eff` は I / V / VIII を動力学的に束ねる有効忘却強度場である。
- `α_CPS` は XIII で物理側へ昇格する係数であり、`α_III / α_VIII / α_eff` からの昇格写像は未閉鎖である。
- `α_*`, `α^*(p)`, `E(f)`, `sgn(α_III)` は独立の α 種ではなく、制約・橋渡し・読解層として扱う。

### 7.3 速度・加速度・力の橋

Paper XII の速度は粒子速度ではなく、忘却境界または担体境界の front の法線速度である。
加速度を扱うとき、primitive は `D_t v` ではなく、境界が時空内に掃く worldtube `W` の第二基本形式 `II(W)` として置く。

したがって、現時点の読みは以下。

```text
速度 = 忘却境界の一次運動
加速度 = 境界 worldtube の二階曲率
力 = その二階曲率が対象の幾何学的感受性を通じて Level 1 側へ物理量化されたもの
```

「加速度 = 力」とは置かない。力は対象感受性を含む相互作用側であり、加速度は運動学側である。

### 7.4 外部 open question crosswalk

`drafts/infra/open_question_crosswalk/` は related work 収集ではない。外部論文の limitation / future work / unresolved problem を「未解決問センサー」として読み、忘却論側の theorem / proposition / hypothesis / experiment に対応づける内部 workflow である。

運用は以下。

1. external intake: PDF / DOI / local file を読む。alphaXiv と NotebookLM は TAINT。
2. question typing: emergence, boundary failure, representation geometry, context degradation, prompt sensitivity, inverse recovery, scaling anomaly などへ型分けする。
3. Oblivion crosswalk: 忘却論側 anchor と主張水準を明示する。
4. candidate answer: 忘却論ならどう答えるかを短く固定する。
5. risk gate: overfit / vocabulary-only / source-thin / too-internal / premature-contact を確認する。
6. promotion: SOURCE promotion、Gauntlet seed、related work patch、short note、direct contact candidate へ昇格する。

## 11. よくある更新事故

### 11.1 記号表を飛ばして論文だけ直す

**事故内容**: 論文を改稿し、記号の意味や定理の内容を変更したが、`統一記号表.md` を更新しない。

**何が壊れるか**:

- 他の論文が旧定義の記号を参照し続ける。
- 定理索引・依存グラフ・主張水準が本文状態と乖離する。
- 次の執筆者が §5 のチェックを行っても、正本が stale なため誤前提で書き始める。

**復旧先**:

1. `drafts/リファレンス/統一記号表.md` §1 の該当記号を更新する。
2. `drafts/リファレンス/系列横断_記号衝突台帳_2026-04-26.md` で衝突を確認する。
3. 統一記号表 §2 / §3 に影響がある場合は更新する。
4. 批判反証レジストリに関係する批判があれば確認する。

### 11.2 レジストリ未更新で反論を書く

**事故内容**: 外部批判への反論を論文本文に直接書いたが、`批判反証レジストリ.md` にエントリを追加しない。

**何が壊れるか**:

- 同じ批判に別の文脈で再対応し、矛盾する応答が並ぶ。
- 横断批判が個別論文だけで処理され、他論文に伝播しない。
- failure condition が不明なまま解決済みに見える。

**復旧先**:

1. `drafts/リファレンス/批判反証レジストリ.md` §2 に該当論文のエントリを追加する。
2. 横断批判なら §1 の Z 系へ昇格する。
3. §1b のシリーズ共通ルールに照らして、応答の主張水準を確認する。
4. §4 OP 台帳へ追加すべき未解決点がないか確認する。

### 11.3 モノグラフ設計を公開系列の正本と誤認する

**事故内容**: `モノグラフ構成設計.md` の章立て・版番号を公開系列の正本として扱い、それに基づいて論文の順序や版番号を変更する。

**何が壊れるか**:

- 公開番号と内部章立てが混線する。
- 版番号の正本である各論文ヘッダとずれる。
- 出版計画の依存グラフと実体が分離する。

**復旧先**:

1. `plans/出版計画_忘却論シリーズ.md` で番号と公開境界を再確認する。
2. 各論文ファイルのヘッダで版番号を直接確認する。
3. モノグラフ設計は series readiness / 章カードの側へ戻す。

### 11.4 open question crosswalk を related work リストにしてしまう

**事故内容**: 外部論文を「似ている文献」として並べるだけで、open question / limitation / future work と忘却論 anchor の対応を書かない。

**何が壊れるか**:

- 外部 SOURCE が pressure test にならない。
- 忘却論の候補回答が vocabulary-only になる。
- contact candidate や short note へ昇格する基準が消える。

**復旧先**:

1. `drafts/infra/open_question_crosswalk/README.md` の運用フローへ戻る。
2. 外部 quote と local anchor を SOURCE として分ける。
3. risk gate に overfit / vocabulary-only / source-thin を書く。

### 11.5 incubator を常置場にする

**事故内容**: 未成熟な seed を `drafts/incubator/` に置いた後、昇格先・主張水準・失敗条件を書かずに放置する。

**何が壊れるか**:

- seed が論文本体の主張と誤認される。
- proof core / defense line / incubating core の層が混ざる。
- 強い主張を守るための隔離面が、未整理物の置き場になる。

**復旧先**:

1. seed の型を proof seed / bridge note / technical design / legacy に分類する。
2. 昇格先と failure condition を冒頭へ書く。
3. 本文へ戻すか、`リファレンス`, `infra`, `experiments`, `standalone` へ移すか決める。

### 11.6 Paper 0 C5 を一枚の定理として扱う

**事故内容**: L6.4-A-G の分解を飛ばして、「不完全性の幾何学的起源」を証明済みの一文として扱う。

**何が壊れるか**:

- Lawvere 非依存化の本丸である L6.4-C が隠れる。
- E/F が未構成のまま「幾何学的盲点」へ昇格してしまう。
- 失敗時の降格先が消える。

**復旧先**:

1. Paper 0 §6.4 の L6.4-A-G 表へ戻る。
2. A/B/D seed と C/E/F/G の未構成部を分ける。
3. 到達水準を「位相的障害」「幾何学的盲点」「忘却曲率」に分けて書く。

### 11.7 加速度と力を同一視する

**事故内容**: Paper XII の front speed を粒子速度として読み、`D_t v` を primitive にして、加速度をそのまま力と呼ぶ。

**何が壊れるか**:

- Paper XII の level-set front の定義が崩れる。
- Paper I の力が持つ対象感受性の層が消える。
- 「欠如境界の超光速運動」と「信号伝達」を混同する。

**復旧先**:

1. `drafts/incubator/加速度は忘却境界の曲率である_橋梁ノート.md` を読む。
2. primitive を `II(W)` に戻す。
3. 力は二階曲率 + 対象感受性として読む。
