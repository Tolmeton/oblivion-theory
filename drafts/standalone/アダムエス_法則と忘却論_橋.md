# Adam's Law と忘却論 bridge

作成日: 2026-04-24

位置づけ: Paper XI §7.7.5 から切り出した bridge note。これは独立論文ではなく、Paper XI、Paper I §5.9、抽象化草稿、HGK skill 設計へ再利用するための補題面である。

---

## 0. 核心

Adam's Law は、同じ意味の文でも高頻度表現の方が LLM に通りやすいことを示す。忘却論から見ると、これは「高頻度表現ほど低摩擦に忘却が実装される」という言語面の法則である。

この法則は、Paper I §5.9 の「射が密な構造ほど忘却に生き残る」と同じ形を持つ。

```text
Paper I:
  射が多い構造
    -> 忘却圧の下で残りやすい
    -> 抽象が生き残る

Adam's Law:
  出現頻度が高い表現
    -> モデルが制約を回収しやすい
    -> 意図が低摩擦で通る
```

したがって、接続の最小命題は次である。

> 高頻度表現は、言語運用圏における射密度の corpus proxy である。

ただし、これは「頻度 = 抽象」という同一視ではない。頻度は表現面の接続密度であり、抽象は意味面の接続密度である。両者は同じ忘却耐性の式に乗るが、同じ対象ではない。

## 1. SOURCE 面

### 1.1 Adam's Law

Lu et al. (2026) は、意味が同じパラフレーズ同士を比較すると、文レベル頻度の高い表現が数学推論、機械翻訳、常識推論、tool calling で高い性能を出しやすいと報告した。

同論文で形式化されている主張は「高頻度パラフレーズは入力 NLL を低くしやすい」である。タスク性能への橋は経験的仮説として扱う必要がある。

SOURCE:
`Adam's Law: Textual Frequency Law on Large Language Models`, arXiv:2604.02176

### 1.2 Paper XI

Paper XI は prompt を情報追加ではなく、prior の選択的忘却操作として扱う。そこで prompt を次のように分解する。

```text
P = (C, E)
```

`C` は制約、`E` はその制約の符号化である。Adam's Law を受けて、`E` はさらに分かれる。

| 成分 | 役割 | 主な作用 |
|:---|:---|:---|
| `$E_{\text{struct}}$` | 記法、構文、テンプレート | 語彙空間、分散構造、検証可能性を変える |
| `$E_{\text{freq}}$` | 表現が pretraining 地形にどれだけ深く埋まっているか | `$C_{\text{intended}} \to C_{\text{eff}}$` の歪みを変える |

`$C_{\text{intended}}$` は人間が与えたつもりの制約、`$C_{\text{eff}}$` はモデルが実際に回収した制約である。

### 1.3 Paper I §5.9

Paper I §5.9 は、忘却耐性を射の密度で定義する。

```text
R(X) = |Hom(X, -)|
```

射が密な構造ほど忘却されにくい。非自明な忘却関手の下では、普遍的構造は最後まで生き残る。これが「忘却は抽象化を生産する」という定理の芯である。

## 2. Bridge 命題

### 命題 A: 表現頻度は射密度の corpus proxy である

言語表現 `s` の出現頻度が高いとは、`s` が多くの文脈に接続しているということである。忘却論の語彙で言えば、これは `s` から出る射が多い状態に対応する。

```text
frequency(s) 高
  -> s が多くの文脈 Y に接続する
  -> |Hom(s, -)| が大きい
  -> R(s) が高い
```

ここでの `R(s)` は、Paper I の厳密な構造対象 `X` をそのまま移植したものではない。言語運用圏における表現 `s` へ写した近似量である。

### 命題 B: 高頻度表現は制約回収の摩擦を下げる

Adam's Law を Paper XI の記法へ移すと、次の式になる。

```text
E_freq ↑  =>  d(C_intended, C_eff) ↓
```

意味:
人間が同じ制約を与えているつもりでも、低頻度表現ではモデルが別の制約として回収する危険が上がる。高頻度表現では、モデルの pretraining 地形に深い道があるため、意図した制約が歪みにくい。

これは「賢い言葉を使うと賢く答える」の逆である。LLM にとって扱いやすいのは、専門性の高さそのものではなく、事前学習地形に太く刻まれた通路である。

### 命題 C: 抽象と高頻度表現は同じ忘却耐性の式に乗る

抽象が忘却に生き残るのは、多くの具体例に接続しているからである。高頻度表現が LLM に通りやすいのは、多くの文脈に接続しているからである。

対応表:

| 忘却-抽象化 | Adam's Law |
|:---|:---|
| 構造 `X` | 言語表現 `s` |
| `R(X)=|Hom(X,-)|` | `frequency(s)` または corpus 接続数 |
| 忘却圧 `G` | prompt 解釈時の prior / compression 圧 |
| `image(G)` に残る | `$C_{\text{eff}}$` に回収される |
| 抽象が残る | 高頻度表現が通る |

したがって、Tolmetes の直感はかなり正しい。ただし「同型」と言うなら、対象をそろえる必要がある。

正確な言い方:

> Adam's Law は、Paper I §5.9 の忘却耐性定理を、LLM の言語表現面へ写した局所モデルである。

## 3. Paper XI への含意

Adam's Law は H3 を壊さない。壊すのは、`E` を一枚岩として扱う書き方である。

修正後の H3:

```text
C を固定したとき、E_struct の残差的変更は Q の平均を直接押し上げない。
ただし E_freq は C_intended -> C_eff の回収摩擦を下げる。
```

この形なら、次の 2 つが同時に立つ。

1. 記法やテンプレートだけを飾っても平均品質は上がらない。
2. 同じ意味なら、高頻度で平易な表現を選ぶほど、意図した制約が通りやすい。

ここで重要なのは、「平易にする」と「弱くする」は違うという点である。主張の射程を落とすのではなく、制約の回収経路を太くする。

## 4. HGK skill 設計への含意

HGK の skill / prompt 設計では、構造記法と自然語を混ぜている。この bridge からは、役割分担が明確になる。

```text
高頻度自然語:
  LLM に制約を低摩擦で回収させる入口

構造記法:
  人間と agent の制御面、検証面、phase 境界を固定する骨格
```

したがって、SKILL.md の実装原則は次になる。

1. 重要な制約は、まず高頻度自然語で言う。
2. その後に構造記法で固定する。
3. 記法だけに意味を背負わせない。
4. 専門語を使うときは、直前か直後に低摩擦な言い換えを置く。
5. prompt は圧縮ではなく、制約回収の摩擦設計として扱う。

短く言えば:

```text
common wording first, formal control second
```

## 5. 棄却台帳

### 棄却 1: Adam's Law は H3 の反証である

棄却。Adam's Law が示すのは、同じ意味でも表現頻度で制約回収の摩擦が変わることだ。これは `E_freq` の効果であり、`E_struct` の残差的変更が平均品質を押し上げる主張ではない。

### 棄却 2: 頻度と抽象は同じである

棄却。高頻度だが抽象的でない表現は存在する。逆に、抽象的だが低頻度な術語も存在する。両者は同じ対象ではない。

採用するのは次の弱いが強い主張である。

```text
頻度と抽象は、どちらも「接続の多さが忘却耐性を上げる」という式に乗る。
```

### 棄却 3: 難しい言葉を避ければよい

棄却。問題は難しさではなく、制約回収の摩擦である。専門語が必要な場面では使う。ただし、その専門語をモデルに通すための高頻度な橋を置く。

## 6. 実験 seed

### 実験 1: `E_struct / E_freq` 分離

同じ制約 `C`、同じ構造記法、異なる頻度の表現で比較する。

```text
条件 A: 高頻度自然語 + 同一構造記法
条件 B: 低頻度言い換え + 同一構造記法
```

測るもの:

```text
d(C_intended, C_eff)
```

実測 proxy:

1. 制約違反率
2. 回答内で回収された条件数
3. 反復時の出力分散
4. tool calling での schema violation
5. judge による instruction adherence

### 実験 2: 高頻度 wrapper の効果

専門記法だけの prompt と、高頻度自然語 wrapper を付けた prompt を比較する。

```text
条件 A: formal only
条件 B: common wording wrapper + formal control
```

予測:

```text
条件 B は C_eff の回収率を上げる。
ただし E_struct 由来の出力分散や検証可能性は条件 A/B の両方に残る。
```

## 8. 結論

Adam's Law は、忘却論にとって外部から来た単なる prompt engineering 知見ではない。これは、忘却耐性が接続密度に依存するという Paper I の主張が、LLM の言語表現面で観測された事例である。

最小式:

```text
接続が多いものは、忘却圧の下で残る。
```

意味面で接続が多いものを、抽象と呼ぶ。  
表現面で接続が多いものを、高頻度表現と呼ぶ。

両者は同じではない。だが、どちらも忘却に対して強い。
