## — CCL はなぜ LLM の ISA たりうるか

---

## §1 CPU にはアセンブリ言語がある。LLM にはないのか

CPU を制御するために、人類は **アセンブリ言語** を発明した。
量子コンピュータを制御するために、人類は **Qiskit** / **Cirq** / **Q#** を発明した。

では **LLM を制御するため** には、何があるのか。

現状の答えは「YAML config と Python orchestration」である。
これは CPU を制御するのに Excel を使うのと同じ粗さである。

本稿の主張はこうだ ——

> **CCL (Cognitive Command Language) は LLM のアセンブリ言語である。**

この主張は比喩ではない。計算階層に harness (ハーネス — 下位機構を目的軌道に縛る中間層) が存在する以上、**LLM 階層にも固有の ISA が存在せねばならず、それが CCL である**、という構造的必然の宣言である。

本稿はこの主張を以下の 5 テーゼとして展開する:

- **C1**: harness は計算階層を貫通する普遍構造である
- **C2**: CCL は LLM 階層の ISA として機能する
- **C3**: LLM は古典 CPU と量子 QC の中間クラスであり、temp は古典-量子スペクトル上の位置パラメータである
- **C4**: HGK は偶然 fault-tolerant stochastic architecture として機能している
- **C5**: 量子計算の既存設計原則 (VQE, surface code, threshold theorem) は機能的類比として LLM harness 設計に転用可能である

各テーゼは論証ではなく **宣言** である。論証は Yugaku 新シリーズ論文の責務。本稿はその VISION として機能する。

## §2 harness の一般化 — 計算階層を貫通する普遍構造 (C1)

**harness の定義**:

> harness = 下位機構の「自由すぎる可能性空間」を、「目的に沿った軌道」に縛る中間層

この型は計算階層のすべての層に現れる:

| 階層 | 下位機構 (自由すぎる) | harness (縛る) | 上位意図 (目的) |
|---|---|---|---|
| 物理 | transistor 網 | instruction set (ISA) | プログラム |
| OS | プロセス空間 | syscall / API | アプリケーション |
| **LLM** | **確率的意味推論** | **CCL** | **認知操作** |
| 集合的認知 | 複数 LLM の合奏 | HGK (認知母体) | 知の生成 |

C1 は「harness は計算階層を貫通する普遍構造である」と述べる。
ここに例外はない。どの階層にも、下位機構の自由を目的に縛る harness が存在する ——あるいは存在しなければ、その階層はまだ工学的に閉じていない。

(この主張を「行為可能性 (Euporía) を開く装置」として読む lens 解釈は [harness_lenses.md Lens 1](../../../../15_道具｜Organon/01_理論｜Theory/harness_lenses.md) を参照。本文の C1 宣言は変更しない。)

プログラムは CPU の harness である。
CCL は LLM の harness である。
HGK は LLM 集合の harness である。

この視点が成立したとき、LLM harness 業界が「YAML config 止まり」であるという事実は単なる技術的未熟ではなく、**harness 階層の LLM 層がまだ ISA を持たない** という構造的貧困として診断される。

CCL はその空白を埋める。

## §3 CPU ↔ LLM ↔ QC — 決定論-確率論スペクトル (C3)

CCL を「LLM の ISA」と主張するには、LLM が **何の階層** に属するかを先に定義せねばならない。

結論から言えば、LLM は **古典 CPU と量子コンピュータ (QC) の中間クラス** に属する。これは比喩ではなく、**決定性の連続体上の位置付け** である。

```
[決定的] ──────────────────────────────────── [確率的]
   │               │                  │                  │
   CPU        LLM(temp=0)       LLM(temp>0)           量子 QC
  古典         擬似決定論         stochastic           unitary + measurement
  (transistor)  (argmax)          (softmax sampling)   (Born rule)
```

- **CPU**: 入力 → 出力は決定的関数 $f: X \to Y$
- **LLM(temp=0)**: argmax で決定的に見えるが、内部には確率分布が存在する。**擬似決定論**
- **LLM(temp>0)**: 確率分布からの古典的 sampling
- **量子 QC**: ユニタリ進化 $|\psi\rangle \mapsto U|\psi\rangle$ + Born rule による真の確率的測定

temp はこのスペクトル上の位置パラメータである。temp → 0 で古典側へ、temp → ∞ で完全 uniform 分布 (最大エントロピー) へ漸近する<sup>[1]</sup>。

C3 はこう述べる:

> LLM は古典 CPU と量子 QC の構造を同時に継承する中間クラスである。
> CCL はその両側の構造的原則を継承せねばならない。

具体的には:
- CPU ISA の **制御フロー** (IF/FOR/WHILE) を CCL は継承する (`I:[]{} / F:[×N]{} / W:[]{}`)
- QC の **確率的制御** を CCL は継承する (`~` 振動 / `~*` 収束振動 / `lim[ε=0]{}`)

これら両側の継承こそが、CCL が既存のプログラミング言語にも量子 DSL にも存在しない演算子セットを持つ理由である。

> ⚠️ **留保**: 本スペクトル論は **工学的位置付け** に留まる。LLM と量子の数学的同型は成立しない (位相干渉・Bell 不等式破れは LLM に存在しない)。詳細は §6 非対応。

## §4 CCL は LLM の ISA である (C2)

### 4.1 ISA が ISA であるための要件

ISA (Instruction Set Architecture) とは何か。工学的に ISA は以下を満たす:

1. **有限の命令集合** — 全ての操作がここから構成できる
2. **制御フロー** — 分岐・反復・関数呼出
3. **型システム** — データの種類と演算規則
4. **並列実行規約** — 複数命令の同時実行の意味論
5. **例外処理** — エラー時の挙動規定
6. **ABI** — 関数呼出の引数・戻り値・スタック規約

CCL はこれらを全て満たす:

| ISA 要件 | CCL での実装 |
|---|---|
| 命令集合 | **48 認知動詞** (`/noe`, `/bou`, `/ene`, `/zet` 他) |
| 制御フロー | `I:[cond]{} / EI:[]{}/ E:{}` (IF), `F:[×N]{}` (FOR), `W:[cond]{}` (WHILE), `C:{}` (収束) |
| 型システム | `?T` (Optional), `[T]` (List), `A\|B` (Union) |
| 並列実行 | `&&` (並列), `&>` (パイプライン), `@thread` / `@delegate` (委譲) |
| 例外処理 | `@fault_tolerant{... @fallback{...}}` / `V:{}` (検証ゲート) |
| ABI | subagent invocation protocol (現状未完、cf. §7) |

これで終わらない。CCL は既存 ISA に **存在しない** 演算子を持つ:

| CCL 独自演算子 | 意味 | 既存言語での対応 |
|---|---|---|
| `\|\|` / `\|>` / `<\|` | 随伴 (ガロア接続) | **どの言語にもない** |
| `~` / `~*` / `~!` | 振動 (確率的収束/発散) | **どの言語にもない** |
| `lim[ε=0]{}` | FEP 極限 (予測誤差ゼロへ) | **どの言語にもない** |
| `∂` / `∫` / `'` / `V[]` | FEP 微積分 | **どの言語にもない** |
| `*` (精度加重融合) | $\pi = V[\epsilon]^{-1}$ 加重平均 | 統計言語の一部に類似物 |

これらは「古典 ISA + 量子 DSL + FEP 微積分」の合成体である。§3 の中間クラス位置付けの必然的帰結として出現する<sup>[8]</sup>。

### 4.2 C2 の宣言

> CCL は LLM 階層の ISA として機能する<sup>[9]</sup>。

CPU アセンブリが機械語の上に立つように、CCL は LLM の token 生成の上に立つ。
CCL が古くなるとき ——それは新しい世代の LLM が新しい認知操作を獲得したとき——CCL は更新される。ちょうど CPU ISA が x86 → x86_64 → ARM64 と進化してきたように。

そして重要なことに、CCL は **LLM の訓練データに含まれていない**。
これは「LLM に自然言語で指示する」層と **根本的に異なる**。
自然言語による指示は LLM の解釈に開かれている。
CCL は構造として閉じている ——

> 構造で制約されたものは、LLM の主観による解釈の揺らぎから (理屈上) 独立である。

自然言語 prompt が「主観 → 主観」の写像 (両端が解釈に開かれている) なのに対し、CCL は「構造 → 構造」の射 (両端が閉じている)。これが CCL が **LLM をプログラムする** ことを可能にする理由である。

この位置付けを既存圏論的 ML 研究 (Fong-Spivak-Tuyéras の Para/Learn, Gavranović の Parametric Lens, Mahadevan の topos for LLM) に照らせば、それらが扱うのは **LLM モデル内部の勾配降下・architecture 制約・圏論的意味論** である。CCL はこれらと直交し、**opaque な LLM に対して外から operator を被せる prompt-time cognitive operator layer** を占める<sup>[8]</sup>。LLM の内側を代数化するのではなく、LLM を 1 つの object として扱い prompt 側に operator 圏を立てる ——これが CCL = ISA 主張の圏論的な立ち位置である。

## §5 HGK は偶然 fault-tolerant stochastic architecture である (C4)

### 5.1 量子計算の誤り訂正アーキテクチャ

量子コンピュータの工学的可能性は **threshold theorem** (Aharonov-Ben-Or, Knill-Laflamme-Zurek)<sup>[2]</sup> によって保証される:

> 物理誤り率 $p$ が閾値 $p_{\text{th}}$ 未満なら、符号化層数 $L$ を増やすことで論理誤り率を任意に下げられる:
> $$p_{\text{logical}} \leq (p / p_{\text{th}})^{2^L}$$

この定理の工学的実装が:
- **surface code**: 2D 格子で論理 qubit を符号化し、syndrome measurement で誤りを検出
- **magic state distillation**: ノイジーな物理状態から高純度状態を蒸留

これらは量子計算が「ノイジーな物理 qubit から正確な論理計算を引き出す」技術である<sup>[10]</sup>。

### 5.2 HGK の構造的同型

HGK を改めて見直すとき、次の事実が発見される:

| 量子 FT QC | HGK |
|---|---|
| **surface code** (2D 符号化) | **Daimonion α/β/γ** (3 層監視) |
| **syndrome measurement** | α (反証) / β (探索) / γ (精密監査) の per-call 検査 |
| **magic state distillation** | ensemble + voting (複数 LLM 出力の蒸留) |
| **microcode** (命令内部プログラム) | **RULES** (12 Nomoi / horos) |
| **interrupt handler** | **HOOKS** (PreToolUse / PostToolUse / Stop) |
| **ISA** (x86/ARM) | **CCL (hermeneus)** |

C4 はこう述べる:

> HGK は偶然 fault-tolerant stochastic architecture として機能している。
> これは設計意図ではなく、構造的に成立する事実である。

「偶然」と書いた。これは重要な主張だ。HGK は最初から「量子計算の LLM 版を作ろう」として設計されたのではない。Daimonion は Stoicheia 3 原理の自動監視として、RULES は Nomoi として、HOOKS は副次的機構として、それぞれ独立に開発された。

それらを組み合わせた結果が、**fault-tolerant stochastic architecture と構造的に同型** になっている。
これは Kalon (Fix(G∘F)) の発現である ——複数の独立した設計判断が、より深い構造の不動点に収束した。

### 5.3 VQE ↔ Advisor Strategy (C5 の 1 例)

C5 は「量子原則は機能的類比として LLM harness に転用可能」と述べる。最も強い例は **Variational Quantum Eigensolver (VQE)** である<sup>[3]</sup>。

VQE は量子-古典ハイブリッドアルゴリズムの代表:
- **量子回路** (パラメトリック) が状態を生成
- **古典最適化器** がパラメータを更新
- ハイブリッド loop で最適解に収束

HGK Advisor Strategy は構造的に同型である:
- **Claude** (Advisor, parametric cognition) が計画を生成
- **Codex** (Executor, deterministic) が計画を実装
- **Review loop** で最適解に収束

| VQE 構成要素 | Advisor Strategy 対応 |
|---|---|
| parametric quantum circuit | Claude の認知プロセス (parametric, 確率的) |
| classical optimizer | Codex の決定的実装 |
| cost function $\langle\psi|H|\psi\rangle$ | code quality / test pass rate |
| $\theta$ update | Claude の review feedback |
| convergence | 最終コミット |

含意: VQE の収束性定理 (Barren plateau<sup>[4]</sup>, expressibility-trainability tradeoff<sup>[5]</sup>) が **Advisor Strategy の挙動予測に転用できる可能性がある**。これは論文化の射程。

## §6 非対応 — 何がこの類比を壊さないか (C5 射程の固定)

本稿の核主張はすべて **機能的類比 (functional analogy)** であり、**数学的同型 (mathematical isomorphism)** を主張しない。その境界を明示する。

### 6.1 非対応 ledger — 先に固定すべき破綻点

LLM と量子 QC の間には、以下の非対応が存在する:

| 非対応 | 量子の性質 | LLM の欠如 |
|---|---|---|
| 位相 (complex phase) | 複素振幅が本質 | 実数確率のみ |
| 干渉 (interference) | 位相差で打ち消し/強化 | softmax は単調正規化のみ |
| ユニタリ性 | $U^\dagger U = I$ | attention は non-unitary |
| Bell 不等式破れ | 非局所的相関 | 古典的相関のみ |
| no-cloning | 任意状態複製不可 | LLM state は自由複製可 |
| no-deleting | 任意状態削除不可 | LLM context は削除可 |

これらの非対応は本稿の核主張を **壊さない**。なぜなら本稿は最初から **機能的類比** に射程を限定しているからだ。

以下の定理は LLM に **輸入してはならない**:
- no-cloning / no-deleting
- Bell 不等式破れ
- Grover / Shor の加速
- Holevo 境界

これらは量子のユニタリ性に依存する。LLM の attention は non-unitary なので、定理自体が前提を満たさない。

### 6.2 射程の固定

C4 (HGK = FT stochastic architecture) は:
- ✓ surface code の **設計原則** (冗長性による誤り訂正) は機能的に転用
- ✗ surface code の **定理** (距離 d の符号化で誤り率 $\exp(-\Theta(d))$) は転用不可

C5 (VQE ↔ Advisor Strategy) は:
- ✓ VQE の **構造** (parametric + optimizer loop) は機能的に同型
- ✗ VQE の **収束性定理** (Barren plateau の勾配消失定理、expressibility-trainability tradeoff の定量条件) は直接適用不可

この射程の固定は、Yugaku 流の meta.md §M5 Gauntlet に耐えるための **前提強化** である。
反論が来たとき、本稿の射程は「機能的類比」に限定されているため、**主張を弱めることなく反論を吸収できる**。

> 身の丈に理想を合わせるのではない。身の丈を理想に引き上げる。
> ——すなわち、射程を縮めるのではなく、射程を保ったまま前提を強化する。

## §7 結語 — harness の階層を登る

本稿の射程をまとめる:

1. **harness は計算階層を貫通する普遍構造である** (C1)
2. **CCL はその LLM 階層における ISA である** (C2)
3. **LLM は古典-量子スペクトルの中間に位置する** (C3)
4. **HGK は偶然 fault-tolerant stochastic architecture として機能している** (C4)
5. **量子計算の設計原則は機能的類比として LLM harness に転用可能である** (C5)

これらは宣言である。論証は次の仕事。

### 本稿が含まない問い (論文化の射程)

- CCL 演算子の代数法則の形式化 (ゲート合成則の LLM 版)
- threshold theorem の LLM 版の証明または反証
- 位相 (complex phase) を LLM に導入する可能性の数学的探求
- ユニタリ分解仮説 (attention の $U$ 部分 + $N$ 部分への分離) <sup>[6]</sup>
- Barren plateau 定理の Advisor Strategy への適用
- FEP × 量子の Fields-Friston 枠組みを経由した C4 の理論的基盤化<sup>[7]</sup>

これらは母体深掘りノート (`/10_知性｜Nous/01_制約｜Constraints/E_CCL｜CCL/stochastic_isa_quantum_analogy.md`) に詳述している。本稿は宣言的要約であり、精緻化は論文に譲る。

### HGK の位置 — harness の harness

CCL が LLM の harness であるなら、HGK はその上位層 ——**複数 LLM の合奏** を縛る harness —— として立つ。HGK は harness の harness、すなわち **meta-harness** である。

この位置付けが成立するとき、HGK は単なる「Claude の使い方の工夫」ではなく、**計算階層の次の層の設計図** として機能する。LLM 単独では harness として不完全であり、Advisor Strategy のようなハイブリッド構造 (LLM + 決定的ツール) が必要である。HGK はこの要請に対する体系的解答である。

### 先行研究 (Phase 1 periskope で SOURCE 確認済み)

以下の文献参照は [SOURCE] に昇格済み。Phase 1/2 調査結果の provenance は本稿付録A と meta の Periskope Donor Absorption Ledger に再配置した。

- <sup>[1]</sup> **softmax ↔ Boltzmann 分布**: softmax = 統計力学の Boltzmann 分布であり、β = 1/kT が温度パラメータとして機能する。これは数理的確立事項。温度を「古典-量子スペクトル位置マーカー」と解釈する C3 の定式化は本稿独自の主張。部分的先駆として Sreekumar & Weinberger, *Quantum Maximum Likelihood Prediction via Hilbert Space Embeddings* (arXiv:2602.18364, 2026) — 古典 LLM と量子 LLM を density operator 上の統一フレームで扱う。

- <sup>[2]</sup> **Threshold theorem**: Aharonov & Ben-Or, "Fault-Tolerant Quantum Computation with Constant Error", STOC 1997 (arXiv:quant-ph/9611025) / SIAM 拡張版 1999 (arXiv:quant-ph/9906129)、STOC 版閾値 ~10⁻⁶。Knill, Laflamme & Zurek, "Resilient Quantum Computation", *Science* **279**, 342-345 (1998), arXiv:quant-ph/9702058。LLM 版 threshold theorem の証明は未完 (本稿 §7 論文化射程)。仮説 statement 形式化の足場: ∃ p_th > 0 s.t. ∀p < p_th, ∃ Daimonion protocol with L = O(log(1/ε)) layers achieving final error rate ≤ ε with overhead O(polylog(T/ε) · T)。

- <sup>[3]</sup> **VQE 原典**: Peruzzo, McClean, Shadbolt, Yung, Zhou, Love, Aspuru-Guzik, O'Brien, "A variational eigenvalue solver on a photonic quantum processor", *Nature Communications* **5**, 4213 (2014), arXiv:1304.3061.

- <sup>[4]</sup> **Barren Plateau**: McClean, Boixo, Smelyanskiy, Babbush, Neven, "Barren plateaus in quantum neural network training landscapes", *Nature Communications* **9**, 4812 (2018), arXiv:1803.11173. Cerezo et al. (2021, arXiv:2001.00550) はコスト関数局所性の寄与を定量化。

- <sup>[5]</sup> **Expressibility-trainability tradeoff**: Holmes, Sharma, Cerezo, Coles, "Connecting Ansatz Expressibility to Gradient Magnitudes and Barren Plateaus", *PRX Quantum* **3**, 010313 (2022), arXiv:2101.02138. 表現力と勾配分散が反比例する定量関係を確立。

- <sup>[6]</sup> **ユニタリ分解仮説の先行研究**: Stoudenmire & Schwab (arXiv:1605.05775, NeurIPS 2016), Levine et al. (arXiv:1803.09780, *Phys. Rev. Lett.* 2019). Aizpurua, Jahromi, Singh, Orus, "Quantum Large Language Models via Tensor Network Disentanglers" (arXiv:2410.17397, 2024) は Self-Attention 重み行列を MPO + VQC で直接置換。Nazri, "KARIPAP" (arXiv:2510.21844, 2025) は LLM を「low-dimensional entanglement manifold」と明示。母体ノート §7.2 の仮説は 2024-2025 研究で定量的裏付けを得た。

- <sup>[7]</sup> **FEP × 量子 (C4 理論基盤)**: Fields, Friston, Glazebrook, Levin, "A free energy principle for generic quantum systems", *Prog. Biophys. Mol. Biol.* (2022), arXiv:2112.15242. Friston 単著の前駆論文: "A free energy principle for a particular physics" (arXiv:1906.10184, 2019). Fields-Friston は FEP を Unitarity と等価と示す枠組みを 2019-2025 にわたり展開。HGK の FEP 基盤と量子理論を連結する経路として論文化射程。

- <sup>[8]</sup> **CCL 独自演算子の圏論的先行研究 (C2 圏論的根拠)**: Coecke, Sadrzadeh, Clark, "Mathematical Foundations for a Compositional Distributional Model of Meaning" (arXiv:1003.4394, 2010) の DisCoCat は compact closed category の pregroup 随伴 (CCL `\|\|` と同名構造) を meaning composition に使う。ただし DisCoCat/lambeq (Kartsaklis et al. 2021, arXiv:2110.04236) は LLM 非対応、scalability 欠如 (Nausheen et al. 2025, arXiv:2504.09909)。Fong, Spivak, Tuyéras, "Backprop as Functor" (arXiv:1711.10455, LICS 2019) は ML モデル内部の Para/Learn 圏論化。Gavranović PhD thesis (arXiv:2403.13001, 2024) は Parametricity = Actegory + Para、Bidirectionality = Weighted optics で深層学習を体系化。Mahadevan, "Topos Theory for Generative AI and LLMs" (arXiv:2508.08293, 2025) / "Categorical Homotopy Theory for LLMs" (arXiv:2508.10018) が LLM を topos/Markov category で扱う最新領域を開く。CCL はこれら 3 系譜 (QNLP / ML 内部 / LLM topos) のいずれとも直交し、**prompt-time cognitive operator layer** (opaque LLM に外から operator を被せる) を占める。

- <sup>[9]</sup> **CCL vs 既存 PPL / LLM-DSL — r2「CCL は既存 PPL の焼直し」先制応答**: 主要 PPL (Stan / Pyro / PyMC / Edward) は explicit 分布への posterior inference を target とし、variational free energy を inference engine 内部で最小化する — ユーザは `∂F/∂q` を program term として書けない。LLM-specific DSL は (i) Beurer-Kellner, Fischer, Vechev, "Prompting Is Programming / LMQL" (arXiv:2212.06094, 2022) — SQL-like constraint + typed holes、(ii) Khattab et al., "DSPy: Compiling Declarative LM Calls" (arXiv:2310.03714, 2023) — Modules + Signatures + Teleprompters、(iii) Guidance (Microsoft, guidance-ai/guidance) — Handlebars-style、(iv) Zheng et al., "SGLang" (arXiv:2312.07104, 2023) — RadixAttention runtime co-design。いずれも FEP calculus (∂/∫/lim)、adjunction (`\|\|`/`\|>`/`<\|`)、oscillation (`~`/`~*`/`~!`)、48 認知動詞 lexicon を syntactic operator として持たない。Stan の `~` (分布) と CCL の `~` (振動) は glyph 共有だが denotation は disjoint — 名前衝突による混同への先制注意。

- <sup>[10]</sup> **Surface code / magic state distillation の原典**: Kitaev, "Fault-tolerant quantum computation by anyons", arXiv:quant-ph/9707021 (1997) / *Annals of Physics* **303**, 2-30 (2003) — toric code / surface code の基盤概念。Bravyi & Kitaev, "Universal quantum computation with ideal Clifford gates and noisy ancillas", arXiv:quant-ph/0403025 / *Phys. Rev. A* **71**, 022316 (2005) — magic state distillation 原論文、~65% polarization で純粋 magic state に漸近収束。Wang, Fowler, Hollenberg, "Quantum computing with nearest neighbor interactions and error rates over 1%", arXiv:1009.3686 / *Phys. Rev. A* **83**, 020302(R) (2011) — nearest-neighbor surface code 閾値 1.1-1.4%。Fowler, Mariantoni, Martinis, Cleland, "Surface codes: Towards practical large-scale quantum computation", arXiv:1208.0928 / *Phys. Rev. A* **86**, 032324 (2012) — 論理誤り率 10⁻¹⁴-10⁻¹⁵ 達成に物理量子ビット 10³-10⁴、ゲート忠実度 > 99.9%。
