## 1. 概要

JetBrains の SWE-agent fork の `history_processors.py` を改造し、
observation の種類に基づく**差分的 masking**を実装する。

均一 masking (Ξ≈0) と LLM-summary (Ξ>0) の間に、
中間的な不均一度を持つ条件を作り、Ξ を操作変数として介入する。

## 2. SWE-agent の observation 分類

SWE-agent で発生する observation の種類 (SWE-bench 文脈):

| 種類 | 典型的内容 | 情報密度 | 保持方針 |
|:--|:--|:--|:--|
| `file_read` | ソースコード表示 | **高** | 保持 (構造情報) |
| `search_result` | grep/find の結果 | 中 | 保持 (探索結果) |
| `test_output` | test suite のログ | **低** (冗長) | **破棄** (結果のみ保持) |
| `directory_listing` | ls/find の出力 | 低 | **破棄** |
| `edit_result` | ファイル編集の確認 | 低 | **破棄** (action に含まれる) |
| `error_trace` | stack trace | **高** (診断) | 保持 |
| `command_output` | 汎用コマンド出力 | 中 | 条件付き保持 |

## 3. 構造的 masking のルール

```python
def structured_mask(observation: str, action: str, turn_age: int, window_m: int) -> str:
    """
    Type-aware observation masking
    
    均一 masking との差分:
    - 均一: turn_age >= M → 全マスク
    - 構造的: turn_age >= M → 種類に応じてマスク/保持
    """
    if turn_age < window_m:
        return observation  # 最新 M turn は全保持 (均一 masking と同じ)
    
    obs_type = classify_observation(observation, action)
    
    # 保持する種類 (高情報密度)
    if obs_type in ("file_read", "error_trace", "search_result"):
        return truncate_if_long(observation, max_chars=500)  # 先頭500字だけ保持
    
    # 破棄する種類 (低情報密度)
    if obs_type in ("test_output", "directory_listing", "edit_result"):
        return f"[{obs_type}: {len(observation)} chars omitted]"
    
    # 汎用: 先頭100字のみ保持
    return observation[:100] + f"\n[... {len(observation)-100} chars omitted]"
```

## 4. observation の分類ロジック

```python
def classify_observation(observation: str, action: str) -> str:
    """SWE-agent の action コマンドから observation の種類を推定"""
    action_lower = action.lower()
    
    # ファイル読取
    if any(cmd in action_lower for cmd in ["cat ", "open ", "view ", "str_replace_editor"]):
        if "str_replace" in action_lower:
            return "edit_result"
        return "file_read"
    
    # テスト実行
    if any(cmd in action_lower for cmd in ["pytest", "python -m test", "make test", "npm test"]):
        return "test_output"
    
    # 検索
    if any(cmd in action_lower for cmd in ["grep ", "find ", "rg ", "ag "]):
        return "search_result"
    
    # ディレクトリ一覧
    if any(cmd in action_lower for cmd in ["ls ", "tree ", "find . -type"]):
        return "directory_listing"
    
    # エラー (observation 内容から判定)
    if "traceback" in observation.lower() or "error:" in observation.lower():
        return "error_trace"
    
    return "command_output"
```

## 5. Ξ の理論的予測

| 条件 | 保持パターン | Ξ (予測) |
|:--|:--|:--|
| **Raw** | 全保持 (retention=1.0 ∀t) | 0 |
| **均一 masking** | binary step (0 or 1) | ≈ 0 (step function の Var) |
| **構造的 masking** | type-dependent (0, 0.5, 1.0) | **0 < Ξ < Ξ_summary** |
| **LLM-Summary** | 要約品質依存 (continuous) | > 0 |

構造的 masking の Ξ が均一 masking より大きくなる理由:
- 均一: 全 old turns が同じ retention (0) → Var = 0
- 構造的: old turns の retention が種類依存 (0, 0.5, 1.0) → Var > 0
