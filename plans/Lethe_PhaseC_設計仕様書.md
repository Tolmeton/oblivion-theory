## §0. 設計原則

### 核心命題 (Source: /kat v2.2)

> Think Anywhere (TA) は、推論の忘却（不選択）における非一様性 (∇Φ ≠ 0) を利用して、
> 有限なスペクトル効率 ρ の非一様配分により局所的な推論力を極大化する力（構造）を生み出している。

### 設計公理

1. **∇Φ 原理**: 忘却は非一様であるとき力を生む。一様な忘却（全位置で同一処理）は力を生まない
2. **ρ 有限性**: ρ（推論の知識密度）は有限。RLVR による微視的増加 (δρ > 0) があっても、有限 ρ の非一様配分の優位性は変わらない
3. **cell レベル非依存性**: ∇Φ ≠ 0 は 0-cell（位置）、1-cell（経路）、2-cell（戦略）のいずれのレベルでも成立する
4. **制御可能性制約**: 0-cell 忘却（推論する位置の選択）は LLM の内部アテンション機構に依存し、外部から直接制御できない。1-cell と 2-cell に焦点を絞る

## §1. Cell レベル別 CCL 操作対応表

忘却理論の cell レベル構造を CCL 演算子体系に転写する。

### 1.1 概念的対応

| Cell レベル | 忘却の意味 | CCL 演算子 | CCL における発現 | 制御可能性 |
|:--|:--|:--|:--|:--|
| **0-cell** (対象) | 推論を**どの位置で**行うか/行わないかの選択 | `{}` 直積、`\|` 選択 | WF の選択・省略 | ❌ 外部制御不能 (LLM内部) |
| **1-cell** (射) | 推論の**どの経路を**辿るか/省くかの選択 | `>>` 合成、`_` シーケンス | WF 合成順序の選択 | ✅ CCL で直接指定可能 |
| **2-cell** (自然変換) | 推論の**どの戦略で**処理するか/しないかの選択 | `~` 振動、`.d`/`.h`/`.x` | 推論戦略の動的切替 | ✅ CCL で直接指定可能 |

### 1.2 1-cell 忘却の CCL 実装: 経路選択（>> 合成順序の忘却スコア依存選択）

**原理**: CCL の `>>` 演算子は射の合成（構造的変換）を表す。忘却スコアは、合成の「**順序**」と「**省略**」を動的に制御する。

**メカニズム**:

```
通常の CCL 合成:
  /noe+ >> /dia+ >> /ene+     (認識 → 判定 → 実行)

忘却スコア依存の合成 (∇Φ ≠ 0 の実装):
  CCL に忘却スコア φ(WF) を注入し、φ が閾値以下の WF を自動省略
  
  φ(/noe+) = 0.9  (高: この文脈で認識は重要)
  φ(/dia+) = 0.2  (低: この文脈で判定は不要)
  φ(/ene+) = 0.8  (高: 実行は重要)
  
  → /noe+ >> /ene+ に自動圧縮 (/dia+ は忘却)
```

**実装設計**:

| 要素 | 仕様 |
|:--|:--|
| **忘却スコア φ(WF, ctx)** | 各 WF の文脈依存の重要度。φ ∈ [0, 1]。0 = 完全忘却、1 = 完全保持 |
| **閾値 θ** | φ < θ の WF を自動省略。θ = 0.3 (初期設定、BBH 実験で較正) |
| **注入点** | CCL パーサーの `>>` 演算子評価時に φ を参照 |
| **φ の計算** | タスクの BBH カテゴリ（ logical_deduction, tracking_shuffled_objects 等）に基づく LUT (Look-Up Table) から初期値を取得。将来的に Semantic Scholar のタスク分類で動的計算 |

**CCL 構文拡張案**:

```ccl
# 明示的な忘却スコア指定
/noe+[φ=0.9] >> /dia+[φ=0.2] >> /ene+[φ=0.8]

# 忘却閾値の指定
@oblivion[θ=0.3]{/noe+ >> /dia+ >> /ene+}

# 自動忘却モード（φ をコンテキストから推論）
@oblivion{/t}    # Telos 族全体に忘却スコアを適用
```

### 1.3 2-cell 忘却の CCL 実装: 戦略選択（~ 自然変換による推論戦略の動的切替）

**原理**: CCL の `~` 演算子は動的な往復（振動）を表す。忘却スコアは、振動の「**深さ**」と「**方向**」を制御する。

**メカニズム**:

```
通常の振動:
  /noe ~* /dia     (認識と判定を収束まで往復)

忘却スコア依存の振動 (∇Φ ≠ 0 の実装):
  φ が低い側の振動を早期に打ち切り、φ が高い側にリソースを集中
  
  φ(/noe) = 0.9, φ(/dia) = 0.4
  → /noe が支配的な振動: /noe の深化に3ターン、/dia は1ターンで打ち切り
  
  φ(/noe) = 0.3, φ(/dia) = 0.8
  → /dia が支配的な振動: /dia の深化に3ターン、/noe は1ターンで打ち切り
```

**実装設計**:

| 要素 | 仕様 |
|:--|:--|
| **振動の非対称性** | φ(A)/φ(B) の比率で A 側と B 側のターン配分を決定 |
| **最小ターン** | φ が低い側でも最低1ターンは実行（完全忘却の防止） |
| **打ち切り条件** | 非対称振動では低 φ 側を先に打ち切り、高 φ 側に残りリソースを集中 |
| **注入点** | `~*` / `~!` の振動ループ内でターン制御に φ を参照 |

**CCL 構文拡張案**:

```ccl
# 非対称振動の明示的指定
/noe[φ=0.9] ~* /dia[φ=0.4]     # noe 支配的収束振動

# 忘却関手の適用（族レベル）
@oblivion{/noe ~ /dia}          # コンテキストから自動で φ を推論

# 戦略切替の忘却注入
/ene[φ=0.7].d ~ /ene[φ=0.3].h  # 随伴振動で d 側を優先
```

### 1.4 0-cell 忘却（後回し — 将来設計メモ）

| 要素 | 状況 |
|:--|:--|
| **概念** | 推論をどの位置（=WF）で実行するかの選択自体 |
| **制御困難性** | LLM のアテンション機構が推論位置を決定。外部からの制御は間接的にしかできない |
| **間接制御の案** | システムプロンプトに「この位置では推論を省略せよ」と指示。効果は非決定的 |
| **Phase C での扱い** | 制御放棄。1-cell/2-cell の成果を確認後に再検討 |

## §2. 忘却スコア φ の設計

### 2.1 φ の定義

```
φ(WF, ctx) : WF × Context → [0, 1]

where:
  WF   = CCL の動詞（/noe, /dia, /ene 等の認知操作）
  ctx  = タスクの文脈（BBH カテゴリ、入力の特性、セッション状態）
  φ    = 0 は完全忘却（WF は省略）、1 は完全保持（WF は全力実行）
```

### 2.2 φ の計算方法（Phase C 初期）

**静的 LUT アプローチ** (M1-M2):

```
BBH タスク × CCL WF → φ 値のテーブル

例:
| BBH タスク | /noe | /dia | /ene | /lys | /ele |
|:--|:--|:--|:--|:--|:--|
| logical_deduction | 0.9 | 0.8 | 0.3 | 0.7 | 0.9 |
| tracking_shuffled_objects | 0.4 | 0.3 | 0.9 | 0.2 | 0.5 |
| causal_judgment | 0.8 | 0.9 | 0.4 | 0.9 | 0.7 |
```

**動的計算アプローチ** (M3 以降 — 将来):

```
φ(WF, ctx) = σ(w · embed(ctx) + b_WF)

where:
  embed(ctx) = タスクの埋め込みベクトル
  w          = 学習済みの重み
  b_WF       = WF ごとのバイアス
  σ          = sigmoid 関数
```

### 2.3 φ と ∇Φ の関係

```
Φ(x) = Σ_WF φ(WF, ctx_x)     # 位置 x における忘却ポテンシャル

∇Φ = dΦ/dx                     # 忘却の空間的勾配

∇Φ ≠ 0 ⟺ φ の分布が非一様    # 核心命題の CCL 実装
```

**ρ との接続**:

```
r_local(x) ≤ √(ρ / (K(x) + 1))    # Paper IV 天井公式

ここで K(x) は位置 x における忘却された WF の数:
  K(x) = |{WF : φ(WF, ctx_x) < θ}|

φ が非一様 (∇Φ ≠ 0) ならば:
  高 φ 位置: K 小 → r_local 大（推論力が高い）
  低 φ 位置: K 大 → r_local 小（推論力が低い、リソース節約）
  
  → 有限 ρ の非一様配分 = 忘却スコアによる CCL の自動圧縮
```

## §3. BBH 実験プロトコル

### 3.1 ρ₂ の操作的定義

```
ρ₂ = BBH スコア (忘却スコアあり) - BBH スコア (忘却スコアなし)

成功基準: p < 0.05 かつ Cohen's d > 0.3 (中程度の効果)
```

### 3.2 実験設計

| 項目 | 仕様 |
|:--|:--|
| **条件 A (Control)** | 標準 CCL パイプライン: `@oblivion` なし |
| **条件 B (Treatment)** | 忘却スコア注入 CCL: `@oblivion[θ=0.3]{...}` |
| **BBH タスク** | 最低 3 タスク: logical_deduction, tracking_shuffled_objects, causal_judgment |
| **試行数** | 各タスクにつき N=30 問 × 3 シード = 90 データポイント |
| **測定指標** | 正解率 (accuracy)、推論トークン数 (efficiency)、応答時間 |
| **統計手法** | Bootstrap 95% CI + Permutation test (p 値) + Cohen's d |

### 3.3 BBH タスクの φ 値初期設定（静的 LUT）

| BBH タスク | 仮説 | 期待される ∇Φ パターン |
|:--|:--|:--|
| **logical_deduction** | 演繹推論は /noe (認識) と /ele (吟味) が支配的。/ene (実行) は低 φ | /noe↑ /ele↑ /ene↓ |
| **tracking_shuffled_objects** | 状態追跡は /ene (実行) が支配的。/noe は低 φ（高次認識は不要） | /ene↑ /noe↓ /dia↓ |
| **causal_judgment** | 因果推論は /dia (判定) と /lys (分析) が支配的 | /dia↑ /lys↑ /ene↓ |

### 3.4 撤退条件 (from /pai)

| 条件 | 基準 | 判定 |
|:--|:--|:--|
| **効果なし** | Cohen's d < 0.1 が 3 タスク以上で再現 | 数値的・機械的 |
| **時間超過** | M1 (2026-04-17) までに本仕様書の Creator レビュー未通過 | 絶対時刻 |
| **技術不能** | CCL パーサーが忘却スコアを受け付けない | Boolean |

## §5. Hermeneus パーサー拡張設計

### 5.1 新しい CCL 構文

```
@oblivion[θ=<float>]{<CCL expression>}

パラメータ:
  θ (0.0-1.0): 忘却閾値。φ < θ の WF を省略。デフォルト 0.3
```

### 5.2 パーサー変更の影響範囲

| モジュール | 変更内容 | 影響度 |
|:--|:--|:--|
| `ccl_parser.py` | `@oblivion` マクロの追加 | 中 (既存 @ マクロパターンに準拠) |
| `operator_loader.py` | φ パラメータの読み込み | 低 |
| `workflow_executor.py` | φ 依存の WF 省略ロジック | 中 |
| `hermeneus_run` | context にφ値を含めた全量渡し | 低 |

### 5.3 AST 拡張

```
既存 AST:
  Sequence([Verb("noe", "+"), Op(">>"), Verb("dia", "+"), Op(">>"), Verb("ene", "+")])

拡張 AST:
  Oblivion(
    threshold=0.3,
    body=Sequence([
      Verb("noe", "+", phi=0.9),
      Op(">>"),
      Verb("dia", "+", phi=0.2),
      Op(">>"),
      Verb("ene", "+", phi=0.8)
    ])
  )
  
  → 評価時に phi < threshold の Verb をスキップ:
  Sequence([Verb("noe", "+"), Op(">>"), Verb("ene", "+")])
```

## §6. 理論的接続

### ∇Φ ≠ 0 と CCL の圏論的対応

```
忘却理論                CCL 実装                    圏論
─────────────────────────────────────────────────────────────
∇Φ ≠ 0               φ の非一様分布              射の選択的合成
ρ 有限                 WF ポイント予算              対象数の有限性
非一様忘却             @oblivion による WF 省略     忘却関手 U の作用
力の生成               高 φ WF への集中             左随伴 F の局所極大
天井公式               K(x) = |{φ < θ}|           余極限の計算
```

### U-series 忘却関手との対応

| U パターン | φ における発現 | 防御 |
|:--|:--|:--|
| U_compose (過剰一般化) | 全 WF の φ を均一に設定 → ∇Φ = 0 | φ のタスク依存性を強制 |
| U_depth (表層化) | θ を高く設定しすぎて大半の WF を省略 | θ の下限を 0.1 に固定 |
| U_causal (因果の忘却) | φ 値の根拠なき設定 | BBH 実験でφを較正 |
